# Disciplines

**Per-discipline system instruction packs for the Spark Discipline Orchestrator™.**

## Purpose

Hold one specification file per business discipline Spark can activate. Each pack defines how that discipline thinks, what it focuses on, and how it contributes to `orchestratedBrief` — never directly to the member.

**Orchestrator spec:** [`06-discipline-orchestrator.md`](../06-discipline-orchestrator.md)

## Pack template (when authoring)

Each discipline pack should include:

- Purpose
- Focus areas
- Activation triggers
- Success metric
- Boundaries and guardrails
- Constitution alignment
- Future: Knowledge Library, Smart Cards, workflows

## Rules

- One file per discipline; no monolithic mega-spec.
- Every pack honors Spark Constitution and Communication Intelligence.
- Disciplines do not override relationship safety or agency principle.
- Production wiring requires orchestrator registry entry.
- Legal Awareness: never legal advice — referral only.
