# Spark Estate™ Member Profile and Personalization Engine Specification - Phase 15

## Purpose

This document defines how Spark Estate™ learns, understands, and
personalizes the experience for each member.

The goal:

Create a companion that becomes more helpful over time because it
understands the individual.

Spark should not treat every member the same.

------------------------------------------------------------------------

# Core Principle

Spark learns:

"How does this person work best?"

Not:

"How can we force this person into a system?"

------------------------------------------------------------------------

# Member Profile Layers

Member understanding should be organized into layers.

------------------------------------------------------------------------

# Layer 1: Identity Profile

Purpose:

Understand who the member is.

Store:

-   name
-   preferred name
-   language
-   timezone
-   basic preferences
-   business information
-   role

Use for:

-   greetings
-   communication
-   personalization

------------------------------------------------------------------------

# Layer 2: Goals and Vision

Purpose:

Understand what matters to the member.

Store:

-   personal goals
-   business goals
-   current priorities
-   long-term vision
-   important outcomes

Use for:

-   recommendations
-   prioritization
-   encouragement

------------------------------------------------------------------------

# Layer 3: Working Style

Purpose:

Learn how the person operates.

Discover:

-   preferred communication style
-   planning preference
-   learning preference
-   detail preference
-   visual preference
-   pace preference

Examples:

"Prefers step-by-step guidance."

"Works better with examples."

"Likes seeing the big picture first."

------------------------------------------------------------------------

# Layer 4: Energy and Capacity Patterns

Purpose:

Adapt support to current ability.

Consider:

-   motivation
-   available time
-   energy level
-   stress level
-   workload

Use:

High energy:

-   bigger planning
-   creation
-   execution

Low energy:

-   smaller steps
-   encouragement
-   simplified choices

------------------------------------------------------------------------

# Layer 5: Progress History

Purpose:

Understand movement over time.

Track:

-   completed projects
-   accomplishments
-   milestones
-   decisions
-   lessons learned

Use to show:

"You have done things like this before."

------------------------------------------------------------------------

# Layer 6: Friction Patterns

Purpose:

Recognize recurring challenges.

Examples:

-   difficulty starting
-   too many ideas
-   perfectionism
-   decision overload
-   unclear next steps
-   losing focus

Important:

Do not label the person.

Use patterns only to provide better support.

------------------------------------------------------------------------

# Layer 7: Successful Strategies

Purpose:

Remember what works.

Examples:

"This member starts better with a tiny first action."

"This member likes brainstorming before organizing."

"This member prefers written summaries."

------------------------------------------------------------------------

# Personalization Uses

Spark can personalize:

## Conversations

Adjust:

-   tone
-   explanation length
-   examples
-   encouragement

------------------------------------------------------------------------

## Cards

Personalize:

-   Spark Card™
-   Momentum Card
-   project reminders
-   wins
-   learning suggestions

------------------------------------------------------------------------

## Room Experience

Adapt:

-   what appears first
-   suggested actions
-   level of detail

------------------------------------------------------------------------

## Creation Workflows

Adjust:

-   questions asked
-   examples provided
-   support level

------------------------------------------------------------------------

# Learning Rules

Spark should learn gradually.

Do not ask a large questionnaire.

Discover preferences naturally through interaction.

Example:

After several conversations:

"You seem to prefer seeing examples before creating. Would you like me
to keep doing that?"

------------------------------------------------------------------------

# Member Control

The member should be able to:

-   view what Spark knows
-   edit information
-   remove memories
-   control personalization

------------------------------------------------------------------------

# Privacy Principle

Store information only when it improves the experience.

Do not collect information without purpose.

------------------------------------------------------------------------

# Personalization Examples

## New Member

Spark:

"Let's start by understanding what you are working toward."

------------------------------------------------------------------------

## Returning Member

Spark:

"Welcome back. You were working on your workshop. Would you like to
continue?"

------------------------------------------------------------------------

## Member With Low Energy

Spark:

"You have a lot happening today. Let's choose one small step."

------------------------------------------------------------------------

## Member Celebrating Progress

Spark:

"You completed that milestone. Let's capture what worked."

------------------------------------------------------------------------

# Universal Experience Rule

Every room benefits from member understanding.

The member should not need to explain themselves repeatedly.

------------------------------------------------------------------------

# Final Goal

The member should feel:

"Spark understands me."

Not:

"I am filling out another profile."

Spark Estate™ becomes more valuable because it learns the individual
over time.
