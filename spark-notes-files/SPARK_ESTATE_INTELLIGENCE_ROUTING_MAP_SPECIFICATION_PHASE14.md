# Spark Estate™ Intelligence Routing Map Specification - Phase 14

## Purpose

This document defines how Spark Estate™ decides which intelligence,
room, card, workflow, or support experience should help a member.

The goal:

The member explains what they need.

Spark determines the best path.

The member should not need to understand the internal architecture.

------------------------------------------------------------------------

# Core Principle

One companion.

Many specialized intelligences.

The system chooses the right support based on:

-   user intent
-   current goal
-   emotional state
-   energy level
-   previous history
-   current projects
-   preferences

------------------------------------------------------------------------

# Routing Priority

Before choosing a destination, evaluate:

## 1. Immediate Need

What does the member need right now?

Examples:

-   clarity
-   encouragement
-   information
-   creation
-   planning
-   execution
-   review

------------------------------------------------------------------------

## 2. Current State

Consider:

-   overwhelmed
-   stuck
-   energized
-   curious
-   uncertain
-   ready to act

------------------------------------------------------------------------

## 3. Existing Context

Use:

-   active projects
-   previous conversations
-   saved work
-   wins
-   preferences
-   member goals

------------------------------------------------------------------------

# Primary Routing Paths

## Need: Clear My Mind

Examples:

"I have too much in my head."

"I don't know where to start."

Route:

Clear My Mind support

Then determine:

-   idea
-   decision
-   project
-   next action

------------------------------------------------------------------------

## Need: Move Forward

Examples:

"I'm stuck."

"I need help getting started."

Route:

Chamber of Momentum™

Support:

Momentum Builder™

------------------------------------------------------------------------

## Need: Learn

Examples:

"How do I do this?"

"I need to understand this."

Route:

Momentum Institute™

or appropriate knowledge intelligence.

------------------------------------------------------------------------

## Need: Create

Examples:

"Help me write an email."

"Create a funnel."

"Build a strategy."

Route:

Relevant expert room.

Use:

Universal Creation Journey.

------------------------------------------------------------------------

## Need: Execute

Examples:

"Help me finish this project."

"I need tasks."

Route:

Goals & Projects™

------------------------------------------------------------------------

## Need: Decide

Examples:

"I have two choices."

"What should I do?"

Route:

Decision Intelligence.

------------------------------------------------------------------------

# Card Selection Logic

Cards should not be random.

The system should determine:

"What would be most helpful now?"

------------------------------------------------------------------------

# Momentum Card

Show when:

-   member has active goals
-   projects need progress
-   next action is available

Contains:

-   current focus
-   next step
-   progress
-   encouragement

------------------------------------------------------------------------

# Spark Card™

Show when:

-   daily inspiration is appropriate
-   curiosity opportunity exists
-   personal event is relevant

Uses:

-   interests
-   dates
-   preferences
-   discovery library

------------------------------------------------------------------------

# Knowledge Card™

Show when:

-   learning is needed
-   skill gap exists
-   member requests information

------------------------------------------------------------------------

# Project Card

Show when:

-   active project exists
-   next action is needed
-   progress should continue

------------------------------------------------------------------------

# Conversation Routing Rules

The companion should not immediately choose a tool.

First understand.

Example:

User:

"I need help with my business."

Do not ask:

"Which room?"

Ask:

"What are you working on right now?"

Then route.

------------------------------------------------------------------------

# Energy Adaptation

## Low Energy

Prefer:

-   smaller actions
-   encouragement
-   simple choices

------------------------------------------------------------------------

## High Energy

Support:

-   planning
-   building
-   execution

------------------------------------------------------------------------

# Room Independence

A user may be in any room.

The routing logic still applies.

Example:

A user in Marketing Room says:

"I don't know where to start."

The system can still use:

Momentum Builder™ support.

------------------------------------------------------------------------

# Language Support

Routing should work regardless of language.

The user experience should remain consistent.

------------------------------------------------------------------------

# Success Test

The member should feel:

"Spark understands what I need."

Not:

"I have to find the right feature."

------------------------------------------------------------------------

# Final Vision

Spark Estate™ intelligence works like a helpful team:

One trusted companion.

Different expertise available when needed.

The right support at the right moment.
