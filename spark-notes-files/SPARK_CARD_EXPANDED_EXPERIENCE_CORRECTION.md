# Spark Card™ Expanded Experience Correction

## Purpose

Correct the expanded Spark Card™ experience.

The current implementation behaves like a large article panel.

Spark Card™ should remain a lightweight discovery experience.

------------------------------------------------------------------------

# Identity

When opened, Spark Card™ should feel like:

-   opening a note
-   reading a thoughtful card
-   discovering a small idea

It should not feel like:

-   a webpage
-   a dashboard
-   a long article

------------------------------------------------------------------------

# Layout

## Header

Include:

-   Spark flame
-   Today's Spark
-   category
-   title

------------------------------------------------------------------------

## Content

### What Happened?

Short story.

### Why It Matters

Short impact explanation.

### Spark It

One reflection question.

------------------------------------------------------------------------

# Actions

Keep simple:

❤️ Loved it

😊 Made me smile

💡 Gave me an idea

⭐ Save Spark

Optional:

📓 Add to Journal

🔗 Connect to Project

------------------------------------------------------------------------

# Routing

Save Spark:

→ My Sparks

Add to Journal:

→ Journal

Gave Me An Idea:

→ Idea Vault or Chamber of Momentum™

Connect to Project:

→ Momentum project connection

------------------------------------------------------------------------

# Do Not

Do not:

-   create a full-screen article
-   require long scrolling
-   display every possible action at once
-   overwhelm the user

------------------------------------------------------------------------

# Final Experience

Spark Card™ should create:

Curiosity → Connection → Optional Action

The user should think:

"I found something interesting."

Not:

"I opened another dashboard."
