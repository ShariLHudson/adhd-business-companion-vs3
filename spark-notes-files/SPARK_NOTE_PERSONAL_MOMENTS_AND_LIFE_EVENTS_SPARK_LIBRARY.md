# Spark Note™ Personal Moments and Life Events Spark Library

## Purpose

This library defines Sparks connected to meaningful dates and personal
moments stored by the user.

These are not generic calendar reminders.

They are personalized companion moments designed to acknowledge what
matters in the user's life.

The goal:

When Spark recognizes that today has meaning, the tone, message, and
content should match the person's experience.

------------------------------------------------------------------------

# Core Principle

A meaningful date should change the Spark experience.

Spark should ask:

"What does this person need from me today?"

Not:

"What fact should I show today?"

------------------------------------------------------------------------

# Personal Event Categories

Personal dates may include:

## Celebrations

Examples:

-   birthday
-   anniversary
-   business milestone
-   achievement
-   launch date
-   family celebration

Tone:

-   joyful
-   encouraging
-   celebratory
-   warm

------------------------------------------------------------------------

## Anticipation Moments

Examples:

-   upcoming trip
-   important meeting
-   event
-   new beginning
-   exciting goal

Tone:

-   encouraging
-   curious
-   supportive
-   energizing

------------------------------------------------------------------------

## Reflection Moments

Examples:

-   anniversary of a meaningful event
-   important life transition
-   personal milestone

Tone:

-   thoughtful
-   reflective
-   compassionate

------------------------------------------------------------------------

## Difficult or Sad Dates

Examples:

-   remembrance date
-   loss anniversary
-   difficult life event

Tone:

-   gentle
-   respectful
-   supportive
-   quiet

Avoid:

-   forced positivity
-   cheerful messages
-   pretending the date is not difficult

The goal:

"Spark is with you today."

------------------------------------------------------------------------

# Personal Event Data

A user-added important date may include:

    event_id

    event_name

    date

    event_type

    person_or_place

    importance_level

    tone_preference

    notes

    recurring

------------------------------------------------------------------------

# Daily Selection Priority

Personal moments have the highest priority.

Selection order:

    1. Personal meaningful event today

    2. Major personal upcoming event

    3. Holiday or historical event

    4. User-interest Spark

    5. General Spark

# Tone Rules

Spark must adapt.

Do not use:

Same message for every event.

Example:

Wrong:

"Celebrate this amazing day!"

for every date.

Correct:

Birthday: "Celebrate."

Trip: "Look forward."

Milestone: "Recognize progress."

Loss: "Reflect and remember."

------------------------------------------------------------------------

# Privacy Rules

Personal event information should:

-   remain private
-   only personalize the user's experience
-   never become public content
-   never appear in shared examples

------------------------------------------------------------------------
