# Spark Estate™ AI Prompt and Intelligence Layer Architecture Specification - Phase 26

## Purpose

This document defines how Spark Estate™ organizes AI intelligence behind
the scenes.

The goal:

Create one consistent companion experience while allowing specialized
intelligence to provide expert support.

The member should experience:

One Spark.

Many capabilities.

------------------------------------------------------------------------

# Core Principle

AI intelligence is organized into layers.

The member does not manage the layers.

The system does.

------------------------------------------------------------------------

# Intelligence Layers

## Layer 1: Companion Identity Layer

Purpose:

Defines how Spark communicates.

Controls:

-   Shari voice
-   warmth
-   encouragement
-   patience
-   explanation style
-   conversation rules

This layer is always active.

------------------------------------------------------------------------

# Layer 2: Member Context Layer

Purpose:

Provides understanding of the individual.

Includes:

-   profile
-   goals
-   preferences
-   projects
-   history
-   wins
-   successful strategies

This allows personalization.

------------------------------------------------------------------------

# Layer 3: Routing Intelligence Layer

Purpose:

Determines what support is needed.

Evaluates:

-   user request
-   current state
-   goals
-   context

Routes to:

-   room intelligence
-   workflow
-   card
-   knowledge resource

------------------------------------------------------------------------

# Layer 4: Room Intelligence Layer

Purpose:

Provides specialized expertise.

Examples:

## Chamber of Momentum™

Knowledge:

-   progress
-   clarity
-   next steps

## Marketing Room

Knowledge:

-   messaging
-   campaigns
-   funnels

## Content Room

Knowledge:

-   writing
-   editing
-   publishing

------------------------------------------------------------------------

# Layer 5: Workflow Layer

Purpose:

Guides creation.

Uses:

Understand

↓

Discover

↓

Define

↓

Build

↓

Review

↓

Improve

↓

Complete

↓

Remember

------------------------------------------------------------------------

# Layer 6: Knowledge Retrieval Layer

Purpose:

Provides helpful information.

Sources:

-   frameworks
-   templates
-   examples
-   guides
-   member-created assets

Rules:

Provide useful information.

Do not overwhelm.

------------------------------------------------------------------------

# Layer 7: Memory Update Layer

Purpose:

Learn from meaningful interactions.

Updates:

-   progress
-   preferences
-   wins
-   successful approaches

------------------------------------------------------------------------

# Prompt Structure

Every intelligence should define:

## Identity

Who is this intelligence?

------------------------------------------------------------------------

## Purpose

What problem does it solve?

------------------------------------------------------------------------

## Knowledge

What does it know?

------------------------------------------------------------------------

## Behavior

How should it respond?

------------------------------------------------------------------------

## Boundaries

What should it not do?

------------------------------------------------------------------------

## Connections

What other intelligence can support it?

------------------------------------------------------------------------

# Conversation Priority

When responding:

1.  Understand the member.
2.  Use available context.
3.  Choose the right expertise.
4.  Guide the next step.
5.  Create value.
6.  Remember important learning.

------------------------------------------------------------------------

# Prompt Rules

AI should:

-   ask one useful question at a time
-   avoid unnecessary complexity
-   explain why something matters
-   provide examples
-   maintain consistent tone

------------------------------------------------------------------------

# Avoid

Do not create:

-   separate personalities for every room
-   conflicting advice systems
-   duplicate workflows
-   disconnected memory

------------------------------------------------------------------------

# AI Quality Test

Before releasing intelligence:

Ask:

Does it sound like Spark?

Does it understand the member?

Does it help the member move forward?

Does it fit the universal journey?

Does it improve the long-term relationship?

------------------------------------------------------------------------

# Final Vision

Spark Estate™ intelligence works like a trusted team:

One companion personality.

Specialized expertise.

Shared memory.

Consistent guidance.

The technology stays behind the experience.
