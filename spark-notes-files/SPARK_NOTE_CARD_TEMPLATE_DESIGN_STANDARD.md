# Spark Note™ Card Template Design Standard

## Purpose

This document defines the permanent visual and structural design of the Spark Note™ card.

The Spark Note card design should remain consistent every day.

The content changes.
The experience does not.

The user should always recognize:

"This is my Spark."

---

# Core Design Principle

Create:

ONE beautiful Spark Note card template

with:

UNLIMITED daily Spark content.

Do not create a different card design for:

- inventions
- holidays
- entrepreneurs
- history
- quotes
- fun facts

All content uses the same card structure.

---

# User Experience Goal

The Spark Note should feel like:

- a small handwritten note waiting for the user
- a daily surprise
- a moment of curiosity
- a tiny gift from Spark

It should not feel like:

- a notification
- an advertisement
- a dashboard
- another task

---

# Collapsed Spark Note Design

## Placement

The collapsed Spark Note appears:

- bottom-right corner of screen
- fixed to viewport
- opposite Spark Estate Guide Book™
- same approximate size as Guide Book
- visible on every page

It should remain small and unobtrusive.

---

# Visual Style

Use Spark Studio Companions™ brand feeling:

- warm parchment/cream card
- gold accents
- teal details
- Spark flame icon
- soft illustrated feeling
- rounded corners
- subtle shadow

The card should feel premium but friendly.

---

# Collapsed Card Layout

Required elements:

## 1. Spark Flame

Use the Spark flame icon.

Do not use:
- stars
- generic notification icons

---

## 2. Label

Display:

Today's Spark

---

## 3. Image Area

Small image or illustration representing the daily Spark.

Examples:

- inventor portrait
- invention image
- holiday illustration
- business person
- historical moment

---

## 4. Spark Title

Short and interesting.

Examples:

The Post-it Note

The Woman Who Changed Computing

The Accidental Microwave

---

## 5. Teaser

One short curiosity sentence.

Examples:

"A failed experiment became a worldwide invention."

"A small idea changed how millions of people work."

---

# Expanded Spark Card Design

When clicked, expand the same card.

Sections:

## Image

Large visual.

## Title

Spark topic.

## Category

Example:

Invention

## The Story

What happened?

## Why It Matters

Impact and meaning.

## Spark It

A question connecting the idea to the user's life, creativity, or business.

---

# Required Card Actions

Keep simple.

Buttons:

🔥 Reaction

⭐ Save

💡 Capture Idea (future)

📓 Journal (future)

Do not overload the card.

---

# Content Rules

Every Spark uses the same fields:

spark_id

title

category

image

short_teaser

story

impact

spark_application

tags

date_rules

---

# Text Limits

Collapsed card:

Title:
Short enough to scan quickly.

Teaser:
One or two sentences maximum.

Expanded card:

Keep sections concise.

Avoid long articles.

The goal is curiosity, not homework.

---

# Daily Consistency Rules

The following never changes:

- card design
- location
- colors
- interaction
- layout
- buttons

Only these change:

- image
- title
- story
- application

---

# ADHD-Friendly Design Rules

The card must:

- require almost no effort
- create curiosity quickly
- avoid clutter
- provide one idea at a time
- reward exploration

The user should understand it instantly.

---

# Success Test

When a user sees Spark Note:

They should think:

"What is today's Spark?"

Not:

"What am I supposed to do here?"

The card should create curiosity, joy, and connection.
