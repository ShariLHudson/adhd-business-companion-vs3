# Chamber of Momentum™ Project Engine Specification - Phase 4

## Purpose

This document defines how Chamber of Momentum™ supports projects and
forward movement.

The goal:

Turn ideas and goals into manageable next steps without creating another
overwhelming project management system.

Existing project functionality should be connected and organized, not
rebuilt.

------------------------------------------------------------------------

# Core Principle

A project is not just a list of tasks.

A project is:

-   something that matters to the member
-   a desired outcome
-   a series of next steps
-   a place to capture progress and wins

------------------------------------------------------------------------

# Project Experience

When a member chooses:

"I want to work on a project"

The system should understand:

## What are you trying to accomplish?

Examples:

-   Launch my website
-   Create a course
-   Finish my book
-   Organize my business
-   Create a product

------------------------------------------------------------------------

# Project Structure

Every project should contain:

## Project Identity

-   Project name
-   Description
-   Why it matters
-   Desired outcome

------------------------------------------------------------------------

## Current State

Track:

-   Where am I now?
-   What has already been completed?
-   What is blocking progress?

------------------------------------------------------------------------

## Next Action

Every active project should have one clear next action.

Avoid:

"Work on website"

Use:

"Write the homepage headline"

The next action should be:

-   specific
-   small enough to start
-   physically possible

------------------------------------------------------------------------

# Project Breakdown

When a project feels too large:

The system helps break it down.

Example:

Large goal:

"Create an online course"

Break into:

1.  Define audience
2.  Outline modules
3.  Create first lesson
4.  Record introduction
5.  Upload content

------------------------------------------------------------------------

# Momentum States

Each project can have a current state:

## Idea

"I might want to do this."

## Exploring

"I am figuring it out."

## Active

"I am working on this."

## Blocked

"I need help."

## Complete

"I finished this."

## Archived

"Not active right now."

------------------------------------------------------------------------

# Blocker Support

When blocked, ask:

"What is getting in the way?"

Possible categories:

-   unclear next step
-   too big
-   missing information
-   lack of time
-   low energy
-   fear or uncertainty

The system should respond differently based on the blocker.

------------------------------------------------------------------------

# Low Energy Mode

When energy is low:

Do not create a large plan.

Offer:

-   smallest next step
-   quick win
-   progress reminder

Example:

Instead of:

"Create your marketing plan"

Offer:

"Write three ideas for your first customer message."

------------------------------------------------------------------------

# Wins and Progress

Every project should capture progress.

Track:

-   completed actions
-   milestones
-   lessons learned
-   wins

Connect with:

-   Evidence Bank
-   Weekly Wins
-   Momentum History

The member should see proof:

"I am making progress."

------------------------------------------------------------------------

# Project Relationship To Other Momentum Areas

## Momentum Builder™

Helps determine:

"What should I do next?"

------------------------------------------------------------------------

## Goals & Projects™

Stores:

-   projects
-   tasks
-   plans
-   execution details

------------------------------------------------------------------------

## Momentum Institute™

Provides:

-   learning
-   resources
-   knowledge

------------------------------------------------------------------------

# ADHD-Friendly Rules

The project system should:

-   show one next step
-   reduce visual clutter
-   avoid giant task lists
-   allow ideas to be captured without losing focus
-   celebrate progress

Avoid:

-   complicated project dashboards
-   too many required fields
-   forcing perfect planning

------------------------------------------------------------------------

# Project Completion Experience

When completed:

Do not simply mark complete.

Celebrate.

Ask:

"What did you accomplish?"

Capture:

-   achievement
-   lesson
-   confidence builder

Store in:

-   Wins
-   Evidence Bank
-   Growth history

------------------------------------------------------------------------

# Demo Requirements

Verify:

✓ Create project works

✓ Project has clear outcome

✓ Next action exists

✓ Breakdown works

✓ Blockers can be captured

✓ Wins are saved

✓ Project connects to Momentum experience

------------------------------------------------------------------------

# Final Goal

Chamber of Momentum™ should help members move from:

"I have too much in my head."

to:

"I know my next step."
