# Spark Estate™ Spark Card™ Visual Design Refinement and Daily Generation Specification

## Purpose

This document defines the approved visual experience and behavior rules
for the Spark Card™.

The goal:

Create a small, meaningful daily companion moment that feels personal,
beautiful, and connected to Spark Estate™.

The Spark Card™ should not feel like a dashboard widget or notification.

It should feel like:

"A thoughtful message from Spark."

------------------------------------------------------------------------

# Core Design Principle

The Spark Card™ is a companion experience.

It should create a moment of:

-   encouragement
-   gratitude
-   meaning
-   reflection
-   discovery

The card is not designed to manage tasks.

It is designed to connect.

------------------------------------------------------------------------

# Visual Design Requirements

## Size

The Spark Card™ should be:

-   compact
-   elegant
-   easy to open
-   similar in footprint to the guidebook card

It should not dominate the room.

The room environment remains the primary experience.

------------------------------------------------------------------------

# Placement

The Spark Card™ should appear:

-   in the lower right area of the room
-   as a small accessible companion element

It should visually belong with the room design.

------------------------------------------------------------------------

# Card Appearance

The card should use the approved Spark Estate™ visual style:

-   clean design
-   warm feeling
-   premium but approachable
-   consistent branding

Avoid:

-   dashboard styling
-   crowded layouts
-   excessive buttons
-   notification appearance

------------------------------------------------------------------------

# Card Content Structure

## Header

Display:

Spark Card™

Small branded identifier.

------------------------------------------------------------------------

## Main Message

The focus of the card.

Content may include:

-   gratitude
-   meaning
-   encouragement
-   reflection
-   curiosity
-   inspiration

The message should be concise but meaningful.

------------------------------------------------------------------------

## Personal Reflection Area

The card may adapt based on member information.

Examples:

## Birthday

Tone:

Celebration and appreciation.

------------------------------------------------------------------------

## Anniversary

Tone:

Recognition and meaning.

------------------------------------------------------------------------

## Upcoming Trip

Tone:

Excitement and anticipation.

------------------------------------------------------------------------

## Difficult or Sad Date

Tone:

Compassionate and reflective.

------------------------------------------------------------------------

# Card Actions

Keep actions simple.

Possible actions:

## How Did This Land?

Purpose:

Allow the member to reflect.

------------------------------------------------------------------------

## Save to Journal

Purpose:

Store meaningful reflections.

------------------------------------------------------------------------

## My Sparks

Purpose:

Allow the member to revisit saved Spark moments.

------------------------------------------------------------------------

# Daily Generation Rule

## One Spark Card™ Per Day

The Spark Card™ should generate once per calendar day.

It should not change every time the member opens the app.

------------------------------------------------------------------------

# Daily Card Lifecycle

At the start of a new day:

Spark evaluates:

-   date
-   member preferences
-   interests
-   important dates
-   gratitude and meaning library
-   recent relevant context

Then creates:

one daily Spark Card™.

------------------------------------------------------------------------

# Same Day Behavior

The same card remains visible throughout the day.

It remains unchanged when the member:

-   closes the app
-   reopens the app
-   changes rooms
-   returns later

------------------------------------------------------------------------

# Regeneration Exceptions

A new card may only be created when:

-   a new day begins
-   the member specifically requests a new Spark
-   a significant personal setting changes

Default behavior:

One card. One day.

------------------------------------------------------------------------

# Memory Connection

The system may remember:

-   saved Spark Cards™
-   reflections
-   meaningful responses

Use these to improve future personalization.

------------------------------------------------------------------------

# Language Support

The Spark Card™ must support member language preferences.

Translation should preserve:

-   meaning
-   warmth
-   emotional tone

------------------------------------------------------------------------

# Quality Test

The Spark Card™ should make the member feel:

"I received something meaningful today."

Not:

"I received another notification."

------------------------------------------------------------------------

# Final Vision

The Spark Card™ becomes a small daily ritual inside Spark Estate™.

One thoughtful moment.

One meaningful message.

One connection point with Spark each day.
