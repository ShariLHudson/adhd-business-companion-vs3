# Spark Estate™ Design System Series

**Series range: 118–124**

UX and quality standards that implement Constitution series **113–117** and the product [Design Bible (125)](../design-bible/125_SPARK_ESTATE_DESIGN_BIBLE.md). They sit below the Product Constitution and Design Bible in the order of authority — they must not override constitutional principles.

| # | Document | Role |
|---|----------|------|
| **118** | [Universal Interaction Patterns](./118_UNIVERSAL_INTERACTION_PATTERNS.md) | Standard interaction model for every Estate experience |
| **119** | [Room Experience Standards](./119_ROOM_EXPERIENCE_STANDARDS.md) | How rooms should feel and behave |
| **120** | [Relationship and Connection UX Standards](./120_RELATIONSHIP_AND_CONNECTION_UX_STANDARDS.md) | Connected To… and relationship surfaces |
| **121** | [Business Pulse UX Standards](./121_BUSINESS_PULSE_UX_STANDARDS.md) | Calm progress awareness UI |
| **122** | [Recognition and Celebration UX Standards](./122_RECOGNITION_AND_CELEBRATION_UX_STANDARDS.md) | Wins, Hall, Garden, Hall celebration UX |
| **123** | [Accessibility and ADHD Experience Standards](./123_ACCESSIBILITY_AND_ADHD_EXPERIENCE_STANDARDS.md) | Sensory safety, ADHD-first experience |
| **124** | [Certification and Quality Gates](./124_CERTIFICATION_AND_QUALITY_GATES.md) | Experience · accessibility · technical · user gates |

## Authority (from Constitution 113)

1. User trust and safety  
2. Product Constitution (**113**)  
3. Specialized Constitutions (**114–117**)  
4. Design Bible (**125**) — [`docs/design-bible/`](../design-bible/README.md)  
5. Certified Product Specifications (incl. **112**)  
6. **This Design System series (118–124)**  
7. Implementation documentation  
8. Local feature convenience
