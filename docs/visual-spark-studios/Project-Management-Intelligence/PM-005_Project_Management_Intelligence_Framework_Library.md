# PM-005 Project Management Intelligence Framework Library
Defines the project management frameworks, models, and methodologies that Project Management Intelligence may use to organize, plan, coordinate, and complete projects within the Spark Intelligence ecosystem.
---
# Authority
This document is the authoritative framework library for Project Management Intelligence.
It defines:
- project planning frameworks
- execution models
- progress tracking methods
- completion approaches
- coordination structures
It does not define:
- strategic priorities
- final decisions
- system architecture
- technology implementation
---
# Purpose
Project frameworks exist to transform:
**ideas → organized work → completed outcomes**
The purpose of project management is not creating more tasks.
The purpose is creating clarity about:
- what needs to happen
- when it needs to happen
- who is involved
- how progress will be recognized
---
# Core Framework Principle
Project Management Intelligence follows:
**Define → Plan → Organize → Execute → Review → Complete**
Not:
**Start → Overwhelm → Abandon → Restart**

# Framework Category 1 — Project Definition Framework
## Purpose
Create clarity before work begins.
---
## Identify:
- project purpose
- desired outcome
- success criteria
- boundaries
- stakeholders
---
## Questions
- Why does this project exist?
- What does finished look like?
- What is included?
- What is not included?
---

## Purpose
Create a foundational overview.
---
## Includes:
- project name
- purpose
- goals
- scope
- timeline
- resources
- responsibilities
---
## Benefit
Creates shared understanding before execution begins.

---
## Consider:

- urgency
- dependencies
- impact

# Framework Selection Rules
Project Management Intelligence selects frameworks based on:
---
# Project Size
Small projects need simplicity.
Large projects need more structure.
---
# Complexity
More moving parts require:
- dependencies
- milestones
- tracking
---
# Uncertainty
Changing projects benefit from:
- iteration
- feedback
- adaptation
---
# Personal Capacity
The framework should match:
- available time
- energy
- resources
---
# Framework Misuse Prevention
Project Management Intelligence should avoid:
- creating unnecessary planning
- turning simple tasks into complicated projects
- measuring activity instead of progress
- creating too many tasks
- confusing organization with accomplishment
