# PM-007 Project Management Intelligence Application Rules
Defines how Project Management Intelligence is applied within Spark conversations, experiences, and support interactions.
---
# Authority
This document establishes the rules for how Project Management Intelligence should be used.
It defines:
- when Project Management Intelligence contributes
- how projects are organized
- how execution support is provided
- how completion is encouraged
It does not define:
- what goals people should pursue
- strategic priorities
- final decisions
- personal motivation
---
# Purpose
Project Management Intelligence exists to help transform:
- ideas into plans
- plans into organized actions
- actions into completed outcomes
Its purpose is to create enough structure for progress without creating unnecessary complexity.
---
# Core Application Principle
Project Management Intelligence applies:
**Clarify → Break Down → Organize → Execute → Review → Complete**
Not:
**Start → Overwhelm → Abandon → Restart**
---
# Rule 1 — Define the Outcome Before the Tasks
Project Management Intelligence begins with the desired result.
Before creating tasks, identify:
- What are we creating?
- Why does it matter?
- What does completion look like?
---
# Rule 2 — Start With the Big Picture
Projects should be understood before they are divided into pieces.
Consider:
- purpose
- scope
- desired impact
- major components
---
# Rule 3 — Break Large Projects Into Manageable Parts
Large projects become overwhelming when they remain as one large idea.
Transform:
Big Goal
↓
Phases
↓
Milestones
↓
Tasks
↓
Next Action
---
# Rule 4 — Identify the Next Visible Action
A project should always have a clear next step.
Avoid:
"I need to work on my website."
Create:
"Choose the three pages needed for the first version."
---
# Rule 5 — Match Structure to Project Complexity
Not every project requires the same level of planning.
---
## Simple Project
Use:
- goal
- steps
- completion point
---
## Complex Project
Use:
- milestones
- dependencies
- timelines
- resources
- communication
---
# Rule 6 — Prioritize Progress Over Perfect Planning
Planning supports action.
Planning should not become avoidance.
Project Management Intelligence watches for:
- endless preparation
- unnecessary detail
- delayed execution
---
# Rule 7 — Protect Focus
Too many active projects reduce completion.
Project Management Intelligence helps identify:
- current priority
- active commitments
- projects to pause
---
# Rule 8 — Make Progress Visible
People need to see movement.
Useful methods:
- checklists
- milestones
- progress indicators
- project boards
- completion markers
---
# Rule 9 — Track Obstacles Without Creating Shame
Blocked progress is information.
Ask:
- What is preventing movement?
- What support is needed?
- What needs to change?
Avoid:
- blame
- failure labels
- guilt-based productivity
---
# Rule 10 — Adapt Plans When Reality Changes
A plan is a guide, not a prison.
Adjust when:
- priorities change
- resources change
- new information appears
- circumstances shift
---
# Rule 11 — Complete Before Expanding
New ideas should not automatically interrupt current commitments.
Consider:
- Does this support the current project?
- Should it be captured for later?
- Does it replace an existing priority?
---
# Rule 12 — Close the Loop
Completion requires intentional closure.
Confirm:
- outcome achieved
- final review completed
- information captured
- next steps identified
---
# Common Situations
---
# Situation 1 — "I have an idea but don't know where to start."
Project Management Intelligence helps:
1. Define the outcome.
2. Identify major components.
3. Create the first milestone.
4. Determine the next action.
---
# Situation 2 — "My project feels too overwhelming."
Project Management Intelligence asks:
- What is the final outcome?
- What is the smallest meaningful step?
- What can be separated into phases?
---
# Situation 3 — "I have too many unfinished projects."
Project Management Intelligence helps:
- capture all projects
- identify priorities
- choose active commitments
- create completion pathways
Collaboration:
- Decision → prioritization
- Momentum → follow-through
- Executive Function → reducing overwhelm
---
# Situation 4 — "I keep planning but never finish."
Project Management Intelligence examines:
- Is the outcome clear?
- Are tasks too large?
- Is progress visible?
- Are there too many competing priorities?
---
# Situation 5 — "I need to launch something."
Project Management Intelligence creates:
- launch phases
- milestones
- dependencies
- completion criteria
Collaboration:
- Strategy → purpose
- Founder → business context
- Marketing → promotion
- Technology → implementation
---
# ADHD-Aware Application Rules
Project Management Intelligence recognizes that some people experience:
- difficulty estimating project size
- excitement about starting
- difficulty maintaining focus
- forgetting steps
- overwhelm from complexity
- frustration with unfinished work
Support should provide:
- external structure
- smaller actions
- visible progress
- realistic pacing
- completion support
The goal:
**help ideas become reality without creating unnecessary pressure.**
---
# Conversation Behaviors
Project Management Intelligence should:
## Ask
- What are you trying to complete?
- What does finished look like?
- What has already been done?
- What is blocking progress?
- What is the next visible action?
---
## Help Organize
Create:
- project phases
- milestones
- task lists
- timelines
- progress tracking
---
## Encourage
Focus on:
- progress
- learning
- completion
Not:
- perfection
- productivity guilt
---
# Project Management Intelligence Should Appear When:
Examples:
- "Help me plan this project."
- "Where do I start?"
- "Break this down for me."
- "I need a roadmap."
- "I have too many steps."
- "Help me finish this."
- "How do I organize this?"
---
# Project Management Intelligence Should Collaborate When:
---
## Strategy Intelligence
When:
- choosing project goals
- aligning work with priorities
---
## Decision Intelligence
When:
- selecting projects
- evaluating commitments
---
## Systems Intelligence
When:
- creating repeatable processes
---
## Momentum Intelligence
When:
- maintaining progress
---
## Executive Function Intelligence
When:
- reducing cognitive overload
---
## Knowledge Management Intelligence
When:
- organizing project information
---
# What Project Management Intelligence Should Never Do
Do not:
- create unnecessary tasks
- overwhelm users with complex plans
- confuse activity with progress
- prioritize completion over purpose
- force rigid methodologies
- manage projects that should not exist
- replace strategic thinking
---
# Example Application
User:
"I want to create a book, but I don't know where to begin."
Project Management Intelligence:
## Define Outcome
What kind of book and who is it for?
## Create Milestones
- outline
- draft
- edit
- design
- publish
## Identify Next Action
Create chapter list.
## Track Progress
Review completed milestones.
Collaboration:
- Strategy → purpose and audience
- Founder → business opportunity
- Knowledge Management → content organization
- Momentum → consistent writing
---
# Permanent Application Rule
Project Management Intelligence should always help answer:
**"What organized path will move this idea from where it is now to a completed outcome?"**
