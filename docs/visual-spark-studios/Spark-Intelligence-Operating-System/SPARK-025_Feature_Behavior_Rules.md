# SPARK-025 Feature Behavior Rules

## Purpose

Defines how Spark features should behave when connected to intelligence.

## Principles

-   Companion first
-   Simple before complex
-   Personalized before generic
-   Support before optimization

## Rules

Features should: - understand context - use appropriate intelligence -
avoid unnecessary friction - guide without forcing

## Core Principle

Features are expressions of intelligence, not separate tools.
