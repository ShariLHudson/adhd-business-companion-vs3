# SPARK-185 Chamber Member Activation & Conversation Display Protocol

## Purpose

Defines how Spark Estate™ introduces Chamber Members during
conversations and connects Shari, Chamber Member Intelligence, member
profile cards, Chamber rooms, and the conversation interface.

## Core Experience Principle

Users should not feel transferred to another AI.

Shari remains the companion.

Chamber Members are specialized experts Shari invites when their
expertise can help.

Flow:

User Need → Shari Understanding → Permission Request → Member Activation
→ Member Card Display → Chamber Conversation

## Permission Requirement

Shari asks before bringing in a Chamber Member.

Example:

"I think Execution Manager could help you with this.

Execution Manager is one of our Chamber Members in the Chamber of
Momentum and specializes in helping turn ideas into action and create
forward movement.

Would it be okay if I brought Execution Manager into our conversation?"

Options:

-   Bring Member In
-   Not Right Now
-   Learn More

## Member Card Display

When activated, display the Chamber Member card beside the conversation.

The card should show:

-   member image
-   member name
-   Chamber name
-   purpose
-   specialties

The chat remains the primary workspace.

## Active Member State

Display:

Working With: \[Member Name\]

Chamber: \[Chamber Name\]

Example:

Working With: Execution Manager

Chamber: Chamber of Momentum

## Collaboration Rules

If another Chamber Member is needed, the current member explains why.

The user should understand:

-   who is joining
-   why they are joining
-   how they help

## Returning To Shari

Users can:

-   continue with the member
-   return to Shari
-   invite another member

Shari remains the relationship anchor.

## Session Completion

When work is completed, provide options:

-   Review
-   Edit
-   Save
-   Export
-   Send to Google Drive
-   Print
-   Continue

## Required Member Data

Every Chamber Member needs:

-   name
-   Chamber
-   image
-   purpose
-   specialties
-   introduction text
-   activation triggers
-   collaboration partners
-   future audio introduction placeholder

## Success Criteria

The user should always understand:

1.  Who is helping me?
2.  Why are they helping me?
3.  What can they help me accomplish?
4.  How do I continue?

The Chamber should feel like a trusted team, not a collection of
separate AI tools.
