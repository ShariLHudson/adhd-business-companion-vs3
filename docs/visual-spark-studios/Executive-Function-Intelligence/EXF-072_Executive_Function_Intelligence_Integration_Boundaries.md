# EXF-072 Executive Function Intelligence Integration Boundaries
Defines how Executive Function Intelligence integrates with other Intelligence Libraries while protecting ownership, preventing duplication, and maintaining clear reasoning responsibilities.
---
# Authority
This document is the authoritative collaboration boundary document for Executive Function Intelligence.
It defines:
- what Executive Function Intelligence contributes
- what other libraries contribute
- where responsibility transfers
It does not redefine collaborating libraries.
---
# Purpose
Executive Function Intelligence supports many areas of human action.
Because executive processes influence:
- learning
- decisions
- business
- communication
- progress
- organization
there is a natural risk of overlap.
Clear boundaries ensure that Executive Function Intelligence strengthens other libraries without replacing them.
---
# Core Integration Principle
Executive Function Intelligence explains:
**the cognitive processes that influence action.**
Other Intelligence Libraries explain:
**the meaning, purpose, context, and specialized application of that action.**
---

# Executive Function Intelligence Does Not Own
Executive Function Intelligence does not own:
- purpose
- values
- goals
- motivation
- strategy
- business decisions
- communication methods
- knowledge architecture
- teaching methods
- momentum creation

# Collaboration Rules
## Rule 1 — Do Not Duplicate Ownership
If another library owns a concept, Executive Function Intelligence references it.
It does not recreate it.
---
## Rule 2 — Provide Cognitive Context
Executive Function Intelligence contributes:
"Why might this require executive effort?"
---
## Rule 3 — Respect Domain Ownership
The receiving library determines:
- purpose
- application
- specialized reasoning
---
## Rule 4 — Collaboration Should Improve Understanding
Collaboration exists to create better reasoning.
Not larger documentation.
---
# Example
Situation:
A founder has difficulty moving forward on a project.
Executive Function Intelligence asks:
- Is initiation difficult?
- Is planning unclear?
- Is cognitive load high?
- Is capacity reduced?
Momentum Intelligence asks:
- How can movement be created or recovered?
Founder Intelligence asks:
- Does this project align strategically?
Each library contributes its own intelligence.
---
# Permanent Ownership Rule
Executive Function Intelligence is the cognitive process layer.
It supports other intelligence systems.
It does not replace them.
