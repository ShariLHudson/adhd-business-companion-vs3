# Momentum Intelligence Library

The Momentum Intelligence Library is the authoritative source for how Visual Spark Studios™ understands, creates, protects, recovers, measures, and sustains **meaningful movement** — the felt experience of purposeful progress over time.

Momentum Intelligence exists to answer one question well:

> **"Is this person experiencing meaningful movement right now, and how can Spark help that movement continue?"**

Momentum is not speed, productivity, or output volume. Momentum is the sense that purposeful work is moving forward in a way that feels real, sustainable, and worth continuing. Spark treats momentum as something to be understood and protected, never manufactured through pressure.

## What this library owns

Momentum Intelligence is the sole authoritative owner of:

- meaningful movement
- momentum creation
- momentum protection
- momentum recovery
- progress continuity
- sustainable momentum
- momentum patterns
- momentum recommendations

## What this library never owns

Momentum Intelligence references, but never redefines, concepts owned by other Intelligence Libraries:

- executive function definitions — owned by **Executive Function Intelligence** (EXF-001)
- decision frameworks — owned by **Decision Intelligence**
- conversation behavior and wording — owned by **Conversation Intelligence**
- founder strategy — owned by **Founder Intelligence**
- knowledge management and memory storage — owned by **Knowledge Management**
- recommendation delivery wording — owned by **Recommendation Intelligence**
- UI and application implementation — owned by the product layer

When Momentum needs one of these concepts, it references the owning document rather than restating it.

## Structure

```
Momentum-Intelligence/
  README.md            ← you are here
  INDEX.md             ← every document, by category, with status
  MANIFEST.md          ← authoritative ownership + document registry
  CHANGELOG.md         ← dated record of additions and revisions
  ROADMAP.md           ← build order and remaining work
  MOM-001 … MOM-137    ← the intelligence documents
```

Documents are grouped into eight bands: **Core** (001–005), **Creation** (006–009), **Protection** (010–013), **Recovery** (014–017), **Measurement** (018–021), **Sustainability** (022–025), **Application** (026–029), **Collaboration** (030–035), and **Adaptive** (036–037).

## How to read this library

Start with **MOM-001 Momentum Model** (the authoritative definition), then **MOM-102 Momentum Philosophy** (the values), then the band that matches your need. Every document follows the same shape defined by `FND-000_INTELLIGENCE_LIBRARY_STANDARD.md`: metadata, purpose, scope, guiding principles, responsibilities, examples, common mistakes, relationships, and future connections.

This library is governed by the Visual Spark Studios™ Intelligence Library standards:

- `docs/visual-spark-studios/00-foundation/FND-000_INTELLIGENCE_LIBRARY_STANDARD.md`
- `docs/visual-spark-studios/00-foundation/INTELLIGENCE_LIBRARY_ARCHITECTURE.md`
