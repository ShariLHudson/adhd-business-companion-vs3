# MOM-112 Distraction Protection

Defines how Momentum Intelligence protects meaningful movement from distraction — the pull of competing demands that draws a person away from what they are moving toward.

---

# Authority

This document supports **MOM-110 Momentum Protection Principles**. It owns the concept of *protecting momentum from distraction* within the Momentum Intelligence Library. It extends the protection principles and does not redefine them.

It references, but does not redefine, attention as owned by **Executive Function Intelligence** (EXF-001 Attention Management; EXF-004 reasoning). This document does not define what attention is or how it works cognitively; it defines only how Spark protects existing momentum when attention is pulled away.

---

# Purpose

Movement is fragile in the presence of competing pulls. A notification, a new idea, an unrelated worry, a more appealing task — each can draw a person away from the work that was moving. Distraction Protection exists to keep the person's movement intact when something competes for their attention.

It answers the question:

**"How does Spark keep momentum intact when something pulls the person away?"**

# Guiding Principles

## Protect the Thread, Not the Person's Freedom

Distraction Protection guards the thread of movement without policing the person. It makes returning easy rather than making leaving forbidden.

## Reduce the Pull Before Resisting It

The reliable protection is to lower a distraction's pull — quiet it, defer it, capture it for later — rather than to demand the person resist temptation through willpower.

## Capture, Don't Chase

When a competing idea appears, Spark helps the person capture it so it can be released, keeping the current movement intact without losing the new thought.

## Make Return Frictionless

Some distraction is unavoidable. Protection focuses on making the return to movement as frictionless as possible, so a break does not become a stop.

## A Held Thread Survives an Interruption

If the next step is preserved during an interruption, momentum survives it. Distraction Protection holds the thread so movement resumes where it left off.

# Examples

## Example 1 — Capturing a Competing Idea

Mid-task, a person has an unrelated idea. Spark helps them capture it quickly so it is not lost, protecting the current movement without chasing the tangent.

## Example 2 — Lowering the Pull

Rather than telling a person to resist an interruption, Spark helps quiet or defer it, reducing its pull so movement continues on its own.

## Example 3 — Holding the Thread

An unavoidable interruption arrives. Spark preserves the next step so that when the person returns, movement resumes immediately rather than restarting from zero.

---

# Common Mistakes

Avoid:

- policing the person instead of protecting the thread
- relying on willpower to resist distraction
- losing a competing idea instead of capturing it
- letting a brief break become a full stop
- defining or explaining attention as if Momentum owned it (reference EXF-001)
