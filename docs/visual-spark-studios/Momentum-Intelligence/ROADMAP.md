# Momentum Intelligence Library — Roadmap

Build order and remaining work for the Momentum Intelligence Library. Foundational documents are long-form and production quality; supporting documents are complete but proportionate to their topic. No short placeholders.

## Build principles

1. **Core before application.** The model and philosophy anchor every later document.
2. **Principles before supporting docs.** Each band opens with a principles document that the supporting documents reference.
3. **Reference, never redefine.** Concepts owned by other libraries are referenced through their owning document.
4. **One document, one concept.** Overlap is resolved by ownership, not duplication.
