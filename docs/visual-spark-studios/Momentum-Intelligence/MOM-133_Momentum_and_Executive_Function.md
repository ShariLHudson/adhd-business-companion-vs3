# MOM-133 Momentum and Executive Function

Defines the collaboration between Momentum Intelligence and Executive Function Intelligence. This is a relationship document. It defines how the two libraries work together and never redefines either.

---

# Authority

This document describes a collaboration. It owns neither library's concepts. Momentum reasoning remains owned by the Momentum Intelligence Library. **Executive Function is defined solely by Executive Function Intelligence** — see EXF-001 Executive Function Model, EXF-002 Philosophy, EXF-003 Intelligence DNA, EXF-004 Thinking Engine, and EXF-005 Knowledge Map, together with the individual domain documents. This document references those definitions and must not restate them.

**Binding boundary:** the eleven executive domains — Task Initiation, Planning, Working Memory, Organization, Prioritization, Attention Management, Time Awareness, Transitions, Self-Monitoring, Emotional Self-Regulation, and Execution — are named here only to point to their owning documents. Their definitions live in Executive Function Intelligence (EXF-001 and the corresponding EXF domain documents) and are not reproduced in this library.

---

# Why These Libraries Collaborate

Executive Function Intelligence understands *how purposeful action is cognitively supported* — how a person begins, organizes, remembers, sustains, and adapts action. Momentum Intelligence understands *how meaningful movement continues over time*. These are two views of the same human effort: one explains the cognitive conditions of action, the other explains the trajectory of progress. They collaborate constantly, because momentum is often helped or hindered by executive conditions that Executive Function Intelligence, not Momentum, understands.

They are close collaborators and clearly distinct. Executive Function asks, "What cognitive conditions would make action easier?" Momentum asks, "How does meaningful movement begin, continue, and recover?"

---

# What Information Is Exchanged

- **Executive Function Intelligence → Momentum:** understanding of the cognitive conditions influencing action — for example, why beginning is hard (EXF-006 Task Initiation), how attention is being pulled (EXF-010 Attention Management), how much cognitive load is present (EXF-019), and how much executive capacity is available (EXF-020). Momentum consumes this understanding to choose appropriate movement support.
- **Momentum → Executive Function Intelligence:** observations of movement, stalling, rhythm, and recovery over time, which contribute context to executive reasoning.

Momentum never interprets the cognitive process itself; it receives Executive Function's interpretation and reasons about movement around it.

---

# What Each Library Owns

- **Momentum Intelligence owns:** meaningful movement, momentum creation, protection, recovery, continuity, sustainable momentum, momentum patterns and recommendations.
- **Executive Function Intelligence owns:** the definition and interpretation of executive functioning and its eleven domains, executive capacity, cognitive load, executive recovery, executive patterns, and adaptive executive support (EXF-001 and related EXF documents).

---

# What Each Library Never Owns

- **Momentum never owns:** any executive concept. When momentum reasoning depends on how action is cognitively supported — initiation, attention, working memory, capacity, load, and so on — it references the owning Executive Function document rather than defining the concept. For example, MOM-108 First Step Intelligence references EXF-006 Task Initiation; MOM-112 Distraction Protection references EXF-010 Attention Management; MOM-111 Friction Reduction and MOM-123 Rhythm and Pacing reference EXF-019 Cognitive Load and EXF-020 Executive Capacity.
- **Executive Function Intelligence never owns:** the definition or reasoning of momentum, meaningful movement, or momentum recovery. Where EXF documents mention momentum, they reference this library.

**On the two "recovery" and "pattern" concepts:** Executive Function Intelligence owns *executive recovery* (EXF-021 — restoring cognitive capacity) and *executive patterns* (EXF-022). Momentum Intelligence owns *momentum recovery* (MOM-114 — restoring meaningful movement) and *momentum patterns* (MOM-136). These are distinct: one concerns cognitive resources, the other concerns the trajectory of movement. Each references the other rather than merging.

---

# How the Collaboration Improves Reasoning

Executive Function tells Momentum *why* movement is hard or easy at the cognitive level; Momentum decides *how* to keep meaningful movement going given that understanding. A person who cannot begin may have an initiation condition (Executive Function) that Momentum answers with a smaller first step (MOM-108). A person losing movement to interruptions may have an attention condition (Executive Function) that Momentum answers by protecting the thread (MOM-112). Neither library could serve the person well alone: Executive Function without Momentum understands the cognition but not the trajectory; Momentum without Executive Function protects movement without understanding why it stalls.

# Guiding Principles

## Reference the Cognition, Reason About the Movement

When a momentum question turns on a cognitive process, Momentum references Executive Function Intelligence and reasons only about the movement.

## Never Restate an Executive Domain

Momentum documents name executive domains to point to them, never to define them. Definitions live in Executive Function Intelligence.

## Distinguish Executive Recovery From Momentum Recovery

Restoring cognitive capacity (EXF-021) and restoring meaningful movement (MOM-114) are different concepts that reference, not absorb, each other.

---

# Common Mistakes

Avoid:

- restating or redefining any of the eleven executive domains inside a Momentum document
- treating momentum recovery and executive recovery as the same concept
- treating momentum patterns and executive patterns as the same concept
- reasoning about the cognitive process instead of referencing Executive Function Intelligence
- merging the two libraries rather than clarifying the boundary
