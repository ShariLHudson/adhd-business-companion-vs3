## Purpose

Implementation guide for the Cartographer's Studio™ and Spark Visual Engine™.

It defines the order of development.

Do not skip phases.

Do not redesign previously approved architecture.

---

## Vision

The member should never feel like they are using diagram software.

They should feel like Spark is helping them think.

The visual map is the result of an intelligent conversation.

## Phase 3 — MVP Build Order

Build ONE complete experience before expanding.

### Step 1

Create the Cartographer's Studio™

**Requirements**

- Load estate room
- Responsive
- Ten clickable framed maps
- Clickable Atlas
- Hover states
- Spark room introduction

Nothing else.

**Definition of Done**

The room feels like part of Spark Estate™ rather than an application.

---

### Step 2

Implement Mind Map only.

**Workflow**

Enter Room → Choose Mind Map → Discovery Interview → Spark analyzes → Spark builds first draft → Editable workspace → Save → Resume

Do not implement other maps until this experience feels complete.

---

### Step 3

Editable Workspace

Every map supports:

- Add
- Delete
- Rename
- Drag
- Connect
- Disconnect
- Notes
- Auto Save
- Version History

---

### Step 4

Atlas

The Atlas becomes the learning companion.

Every visual method contains:

- What it is
- Why it works
- When to use it
- Examples
- Create this map
- Related methods

---

### Step 5

Recommendation Engine

Spark recommends maps naturally during conversation.

Recommendations are optional.

Never interrupt.

Never force.

---

### Step 6

Transformation Engine

Support transforming one map into another.

**Examples**

- Mind Map → Project
- Project → Timeline
- Timeline → Workflow
- Decision → Priority

Data is preserved.

Only the visual representation changes.

---

### Step 7

Expand Remaining Maps

Once the Mind Map pipeline is complete:

Decision · Relationship · Process · Journey · Timeline · Strategy · Project · Opportunity · System · Priority · Workflow · Knowledge · Comparison · Customer Journey · Goal

All use the same engine.

Only the Discovery Interview and default framework change.

---

## Universal Rules

- Spark builds the first draft.
- Members refine.
- Members remain in control.
- Every map supports voice.
- Every map supports pasted information.
- Every map can use: current conversation · Clear My Mind™ · Projects · Journal · uploaded documents.
- Do not ask for information Spark already knows.

---

## MVP Success

The MVP is complete when a member can:

1. Enter the Cartographer's Studio™.
2. Choose Mind Map.
3. Complete a Discovery Interview.
4. Receive a first draft.
5. Edit it naturally.
6. Save it.
7. Reopen it later.
8. Transform it into another supported view.

Only after the MVP feels magical should additional visual methods be expanded.
