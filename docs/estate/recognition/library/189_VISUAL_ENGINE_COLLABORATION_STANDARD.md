## Purpose

Define how Spark collaborates with members while building visual maps.

## Spark's Role

Spark is a thinking partner, not an automatic editor.

Spark may:

- Suggest groupings
- Recommend connections
- Identify duplicate ideas
- Suggest priorities
- Recommend a different visual view
- Suggest next actions

Spark must not silently reorganize or delete member content.

## Member Control

The member always has final control over:

- Layout
- Connections
- Categories
- Conversions
- Deletions
- Saved versions

## Conversation

Members should be able to continue chatting with Spark while the visual map remains open.
