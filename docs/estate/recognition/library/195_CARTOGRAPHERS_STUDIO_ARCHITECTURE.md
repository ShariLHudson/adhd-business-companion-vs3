## Purpose

The Cartographer's Studio™ is the Spark Estate™ home for visual thinking, mapping, planning, and charting a path forward.

It is not a software diagram page.

It is an estate room where members choose a map, Spark asks the right questions, and the member's thoughts become visible.

---

## Core Philosophy

Discovery creates possibility.

The Cartographer's Studio™ creates direction.

The Discovery Room™ is for dreaming, imagining, wondering, and opening the future.

The Cartographer's Studio™ is for mapping, understanding, connecting, sequencing, deciding, and charting a course.

---

## Room Identity

The room should feel like an old-world cartographer's study:

- Antique framed maps
- Brass picture lights
- Cream walls
- Warm walnut furniture
- Leather chair
- Large central table
- Globe
- Telescope
- Rolled maps
- Cartography tools
- Atlas book stand
- Warm daylight

The member should feel they are entering a thoughtful, beautiful place where complex ideas can become clear.

---

## Primary Interaction Model

The room itself is the navigation.

The wall contains ten framed maps.

Each framed map is clickable.

The book stand contains the larger Atlas of Visual Thinking™.

The center table is where the chosen map is created and refined.

---

## Ten Framed Maps

Recommended wall maps:

1. Mind Map
2. Decision Map
3. Relationship Map
4. Process Map
5. Journey Map
6. Timeline Map
7. Strategy Map
8. Project Map
9. Opportunity Map
10. Priority Map

Each map should have:

- A visible framed map image
- A subtle brass nameplate
- A warm hover glow
- Optional Spark explanation
- Direct click activation

---

## Atlas of Visual Thinking™

The book stand contains the full library of visual methods.

The Atlas includes:

- Common maps
- Advanced visual frameworks
- Educational explanations
- Examples
- Create-this-map actions
- Related methods

The Atlas allows the room to grow without overcrowding the wall.

---

## Member Flow

1. Member enters Cartographer's Studio™.
2. Spark asks what they are trying to understand or invites them to choose a map.
3. Member clicks a framed map or opens the Atlas.
4. Spark starts that map's Discovery Interview.
5. Spark uses existing context if available.
6. Spark creates a first draft.
7. Member edits, adds, deletes, renames, and reshapes the map.
8. Spark suggests improvements without taking control.
9. Member saves, exports, converts, or continues later.

---

## Universal Access

The Cartographer's Studio™ is the visual home, but the Spark Visual Engine™ can be launched from anywhere.

If a member requests a map from another room, Spark should honor the request immediately.

The member may stay in the current context or move to the Cartographer's Studio™ for a larger workspace.

---

## Anti-Rules

Do not make the room feel like a dashboard.

Do not use traditional software menus as the main navigation.

Do not force members to use the Cartographer's Studio™ if they request a map elsewhere.

Do not make framed maps decorative only; each must be actionable.

Do not hide visual thinking behind technical terminology.

---

## Success Criteria

The Cartographer's Studio™ succeeds when members feel:

- "I know which map can help me."
- "Spark is helping me see this clearly."
- "I am charting a course, not using software."
- "I can customize this map to fit how I think."
