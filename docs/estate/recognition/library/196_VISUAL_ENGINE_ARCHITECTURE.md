## Purpose

The Spark Visual Engine™ is the shared system that powers visual maps throughout Spark Estate™.

It is not one feature.

It is a reusable engine available from any room, conversation, project, journal, decision, or thinking experience.

---

## Core Philosophy

The goal is not to create diagrams.

The goal is to help members think.

A visual map is the visible expression of a thinking process.

---

## Engine Responsibilities

The Spark Visual Engine™ owns:

- Map type selection
- Direct map command recognition
- Context intake
- Discovery interviews
- Framework structure
- First-draft map creation
- Node actions
- Section customization
- Map transformations
- Save and history
- Export and conversion
- Integration with other estate systems

---

## What the Engine Does Not Own

The Visual Engine does not own:

- Final project data
- Calendar data
- Journal data
- Evidence Vault™ data
- Destination Gallery™ data
- Google documents
- Member identity
- Permanent room navigation

It visualizes, transforms, and returns updates to the owning system.

---

## Activation Paths

### Direct Request

Examples:

- Create a mind map.
- Make a decision tree.
- Show this as a timeline.
- Build a workflow.
- Map this out.

Direct requests should launch the requested visual view immediately.

### Spark Suggestion

Spark may recommend a visual view when the information would be easier to understand visually.

### Clickable Map

Members may choose a framed map inside the Cartographer's Studio™.

### Atlas Selection

Members may choose an advanced method from the Atlas of Visual Thinking™.

---

## Core Flow

1. Receive map request.
2. Determine requested map type.
3. Gather existing context.
4. Ask only the missing questions.
5. Build first draft.
6. Allow member customization.
7. Save automatically.
8. Provide next actions.

---

## Map Transformations

The same information may be transformed into different views.

Examples:

- Thought Map → Project Map
- Project Map → Timeline
- Decision Map → Priority Map
- Process Map → Workflow
- Opportunity Map → Strategy Map

Data should not be lost when the view changes.

---

## Chamber Member Integration

Chamber Members may suggest visual thinking when appropriate.

Suggestions should be helpful, not intrusive.

---

## Member Control

Spark may recommend, organize, and suggest.

The member controls what is added, deleted, renamed, connected, saved, and converted.

---

## Success Criteria

The Visual Engine succeeds when members feel:

- Spark understood what they were trying to think through.
- The map helped them see something more clearly.
- The map was useful, not decorative.
- They could shape the map to fit their real situation.
