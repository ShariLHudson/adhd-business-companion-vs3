# EST-001 ŌĆö The Room Is the InterfaceŌäó

## Core Principle

> **Members should feel like they are entering a placeŌĆönot opening a feature.**

Every room inside Spark EstateŌäó should feel like a real destination. The software fades into the background while the estate becomes the experience.

---

# Design Hierarchy

## Layer 1 ŌĆö The Place

The room comes first.

Design for:

- Architecture
- Lighting
- Atmosphere
- Sound
- Beauty
- Emotion
- Purpose

The member should immediately feel where they are and why the room exists.

---

## Layer 2 ŌĆö Spark

Spark is the guide.

Spark should:

- Welcome the member.
- Explain the room's purpose naturally.
- Notice meaningful context.
- Recommend helpful actions.
- Stay conversational.

Spark never feels like a wizard or setup screen.

---

## Layer 3 ŌĆö The Workspace

Only after the member understands the room should tools appear.

Tools should feel integrated into the environment rather than floating application controls.

Examples:

- Discovery Table
- Atlas
- Journal
- Evidence displays
- Parchment maps

Avoid exposing technical controls unless they are needed.

---

# Estate-Wide Rules

## The Room Fills the Screen

Rooms should occupy the available browser space.

Avoid:

- Large white margins
- Nested application windows
- Secondary dashboards
- Competing toolbars

The room itself is the interface.

---

## Remove Software Chrome

Hide development-era UI such as:

- View selectors
- Workspace modes
- Chat focus toggles
- Save menus
- Auto Saved indicators
- Internal labels
- Legacy navigation

Replace software language with natural estate language.

Example:

Instead of:

> Auto Saved

Spark simply preserves the work.

---

## Navigation

Navigation should be subtle.

Preferred:

- ŌåÉ Return to Estate
- Help
- Resume Previous Work
- Exit

No large application navigation bars inside rooms.

---

## Camera Before Pages

Whenever possible:

Move the camera.

Do not switch to another page.

Examples:

### Cartographer's StudioŌäó

Camera moves toward the Discovery Table.

The parchment opens.

The visual map appears.

### Evidence VaultŌäó

Spark opens the archival vault.

Selected evidence appears naturally.

### ConservatoryŌäó

Thoughts appear on the Discovery Table instead of inside software cards.

---

# Emotional Design

Every room should answer five questions before any interface is built.

1. What room does this belong in?
2. What should the member feel when they arrive?
3. How does Spark welcome them?
4. What naturally draws their attention?
5. How can the tools stay in the background until they are needed?

If these questions cannot be answered, the room is not ready for implementation.
