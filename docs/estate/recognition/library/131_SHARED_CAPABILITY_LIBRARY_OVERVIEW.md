# 131_SHARED_CAPABILITY_LIBRARY_OVERVIEW

# Spark Estate™
## Shared Capability Library Overview

Purpose:
Define the reusable capability system created from GPT navigation breakdowns.

Core idea:
Capabilities are not trapped inside one GPT, chat, member type, or room.

A capability can support many members, rooms, workflows, and future companions.

Examples:
- Decision making
- Planning
- Brainstorming
- Reflection
- Problem solving
- Research
- Strategy
- Content creation
- Organization
- Celebration
- Learning
- Communication

Spark should compose capabilities dynamically based on the member's context.

The user should experience one companion, not a collection of separate GPTs.
