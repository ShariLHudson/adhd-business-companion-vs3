---

## Purpose

Define what members can do with any item, node, card, step, or visual element inside the Spark Visual Engine™.

A visual map should not be decorative.

Every node should be useful.

---

## Core Principle

Every visual node can become action.

A thought can become a task.

A task can become a project.

A project can become a timeline.

A discovery can become an Evidence Vault™ item.

A finished piece can go to the Destination Gallery™.

---

## Required Node Actions

Every node should support:

- Edit
- Rename
- Move
- Delete
- Connect
- Disconnect
- Add child node
- Add sibling node
- Add note
- Mark important
- Mark complete
- Change color/category
- Add tag

---

## Conversion Actions

A node may convert into:

- Task
- Project
- Calendar item
- Reminder
- Journal entry
- Evidence Vault™ item
- Destination Gallery™ item
- Google Doc
- Google Drive item
- Canva / Create item
- Spark Social Media item
- Decision Compass™ item
- Clear My Mind™ item

---

## Work Actions

A node may trigger:

- Start Focus Timer
- Schedule work block
- Open related project
- Open related journal entry
- Open related document
- Send to Visual Engine view
- Add to waiting list
- Add to someday list

---

## Relationship Actions

Nodes may connect to show:

- Depends on
- Related to
- Part of
- Blocks
- Supports
- Comes before
- Comes after
- Inspires
- Belongs with

---

## Member Control

Spark may suggest actions, but the member controls final placement.

Example:

Spark may say:

“This looks like a project. Would you like to turn it into one?”

The member may accept, ignore, or choose another action.

---

## Save Behavior

All node changes should persist.

Saved maps should preserve:

- Nodes
- Positions
- Connections
- Labels
- Notes
- Categories
- View type
- Source context
- Related project/session

---

## Success Criteria

The member should not feel like the visual map is a dead image.

They should feel like every part of the map can help them move forward.
