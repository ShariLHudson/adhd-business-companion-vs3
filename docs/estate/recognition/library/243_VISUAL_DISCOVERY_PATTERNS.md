# Visual Discovery Patterns

## Purpose

Spark should think before asking.

Never repeat information already known.

---

## Pattern 1

Member chooses map.

Do NOT ask which map.

---

## Pattern 2

Member provides context.

Do NOT ask them to repeat it.

---

## Pattern 3

Use estate memory first.

Check:

- Conversation
- Journal
- Clear My Mind™
- Projects
- Previous maps

---

## Pattern 4

Ask one question at a time.

Conversation—not questionnaire.

---

## Pattern 5

Know when to stop.

The goal is to create the first draft quickly, not collect every possible detail.
