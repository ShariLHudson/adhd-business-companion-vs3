# Estate Collection Interior Design™

**Interaction blueprints for what happens inside each room.**

Architecture and emotional experience are approved. This sprint designs **interior life** — how visitors move, focus, create, browse, find, and return. Not implementation. Not code. Not new features.

**Builds on:** [Experience™](./ESTATE_COLLECTION_EXPERIENCE.md) · [User Journey](./ESTATE_COLLECTION_USER_JOURNEY.md) · [Room Identity Bible™](./ESTATE_ROOM_IDENTITY_BIBLE.md)

---

## Interior design principles (all rooms)

| Principle | Interaction meaning |
|-----------|---------------------|
| **One focal point** | The eye knows where to land within three seconds |
| **Conversation continues** | Chat may stay visible; room never hijacks |
| **Compose is invitation** | Creating never feels like a form submission |
| **Browse is leisure** | Scrolling should feel like reading, not searching a database |
| **Find without folders** | Search and gentle filters — never tree navigation |
| **Organize invisibly** | Spark and metadata work quietly; member sees calm |
| **One related thread** | Never a widget of six connections |
| **Spark whispers** | Inside the room, Spark speaks less — atmosphere speaks more |

**Scroll order philosophy:** First visits emphasize *create*. Return visits with content emphasize *browse* — but compose always remains reachable without hunting.

# Journal Gazebo

### 1. Arrival experience

Visitor steps into soft outdoor light — gazebo scene subdued (small plate, low opacity). Header is minimal: kicker *The White Gazebo*, room name, one-line description. Spark offers a single welcome line (first visit or return variant). **Writing surface appears before** prompts and saved pages — the room says *write first*.

No dashboard. No stats. Back control returns to Estate without breaking conversation.

### 2. Primary focal point

**The primary textarea** — borderless, large serif, bottom rule only. Everything else is peripheral until the visitor scrolls. Saved pages live below the fold; they never compete on first paint.

### 3. Creating a new entry

Flow feels like **opening a blank page in a notebook**:

1. Cursor lands in the writing area (or visitor taps it).
2. Optional margin label hidden visually — placeholder carries invitation.
3. Secondary fields (kind of entry, remembered line, estate place) sit *below* the page — progressive, never blocking.
4. Keepsakes (photos) attach quietly at the bottom — *optional*, not step three of five.
5. Save is a text action: *Rest this page* — underline link weight, not a primary button bar.

While composing, suggested prompts are **text links** below Spark’s line — tap fills the page, does not replace conversation.

### 4. Viewing a single entry

Opening a page is **reading mode** — full body text large, date as italic margin, optional title as soft heading (not a filename). Treasured pages have subtle warmth on the divider — not a gold star.

Actions appear on hover/focus: *Continue writing* · *Treasure this* · *Remove page* — whisper links, not a toolbar.

Keepsake opens in place — image breathes, does not open a modal workflow.

### 5. Browsing many entries

Section *Pages at rest* — vertical scroll of pages separated by hairlines. Each page shows: date → optional title → body preview (many lines for journal — this room rewards reading previews). Density stays low; 500 entries still feel like a journal, not an inbox.

No grid. No cards with equal-height boxes. Variable length is beautiful.

### 6. Finding older entries

**Search** — single underline field: *Search your journal pages…* — searches title and body meaningfully.

**Kind of entry** — text-link filters (gratitude, lesson, prayer, etc.) — underline toggles, not chips.

**Treasured** — implicit via filter or visual on entries; no separate “favorites app.”

Results replace list calmly — empty search shows gentle *nothing matched* line, not an error.

### 7. Organizing entries

Member organizes **lightly**:

- Optional category at save time
- *Treasure this* for pages that matter most
- Optional *line to remember* acts as informal title

No folders. No drag-and-drop sort. Chronology is default — life order. Spark may cluster by theme in conversation later — not in-room taxonomy UI.

### 8. Related entries

At most **one** related whisper when reopening or when Spark speaks inside the room:

> *"You explored something similar last spring."*

Tapping opens that page — no “related entries” panel. Relatedness from themes, season, and category — never algorithm exposed.

### 9. Spark interactions inside the room

| When | Spark |
|------|-------|
| Arrival (first) | One welcome line, then silence |
| Arrival (return) | *"Nice to see you in the Gazebo again."* — optional |
| While writing | Silent — mic available, no coaching overlay |
| After save | Confirmation line only — no next-step menu |
| Browse | Silent unless visitor asks in chat |
| Reopen page | Optional context: *"You wrote this the week before the launch."* |
| Pattern (rare) | *"You've come here often this month — a heavy season?"* — only with care |

Spark never counts pages aloud. Never suggests “journal more.”

### 10. How the room evolves over years

| Era | Interior life |
|-----|----------------|
| **Year 1** | Mostly compose; few pages below fold |
| **Years 2–3** | Browse becomes pleasure; search used for recall |
| **Years 4–6** | Treasured pages become informal index; themes emerge in chat |
| **Years 7–10** | Room feels like personal archive; compose less frequent, rereading precious |

Physical metaphor stays: **gazebo notebook** — never morphs into note-taking SaaS.

# Growth Greenhouse

### 1. Arrival experience

Warm greenhouse light — scene plate modest. Kicker *Where life is still becoming*. Spark welcome is patient. Compose sits **above** *Still becoming* list but below brief orientation — visitor may plant before browsing seedlings.

Atmosphere: green-gold tones, no progress UI anywhere.

### 2. Primary focal point

**The “what is growing” textarea** — primary field, generous rows. Question *Something taking root* frames compose. The growing thing is the hero — not category, not season field.

### 3. Creating a new entry

Feels like **labeling a seedling pot**:

1. What is growing? (required — large text)
2. Kind of growth (select — habit, skill, relationship, etc.)
3. What would nurturing look like? (optional — smaller)
4. Season (optional text — *early roots*, not %)

Save: *Place here gently*. No progress bar on save. No “complete.”

### 4. Viewing a single entry

Read as **a plant record** — body in quote-weight or soft prose, extra fields (*Nurturing*, *Season*) as quiet labeled lines below, not a form recap.

Actions: *Refine growth* · *Let go* — letting go is gentle language for remove.

No “mark complete” — graduation is a conversation that may move entry elsewhere.

### 5. Browsing many entries

*Still becoming* — vertical stack, hairline dividers. Each entry: body prominent, nurture/season as secondary lines. No icons of sprouting plants. No checklist columns.

Quote emphasis on body — growth language feels reflective, not metric.

### 6. Finding older entries

Search: *Search what is growing…*

Filter: *Kind of growth* — text links.

No “active vs archived” tabs in V1 interior — letting go removes; mature growth may live in Achievement/Evidence instead.

### 7. Organizing entries

Category at create time. Chronological browse default.

Organization is **seasonal in language**, not dates sorted by deadline. Member may refine text over time — same entry evolves in place.

No priority ranks. No due dates.

### 8. Related entries

One whisper maximum:

> *"This connects to the patience you were tending last year."*

Or when viewing: link to Evidence/Achievement if member graduated that growth — suggested in chat after visit, not inline web.

### 9. Spark interactions inside the room

| When | Spark |
|------|-------|
| Arrival | Patient welcome |
| Return with entries | *"A few things are still becoming here."* |
| After save | *Placed here gently…* |
| Revisit (invited) | *"How is that boundary practice feeling?"* — only when member opens chat |
| Breakthrough in chat | *"Sounds like this graduated — ready for the shelf or vault?"* |
| Dormant | *"Some seasons are quiet. This still belongs here."* |

No growth reminders push notifications. No “update your goals.”

### 10. How the room evolves over years

| Era | Interior life |
|-----|----------------|
| **Year 1** | Few seedlings; compose-forward |
| **Years 2–4** | Mix of old and new growth; browse shows character arc |
| **Years 5–7** | Entries graduate out to other rooms; greenhouse stays for new becoming |
| **Years 8–10** | Room is for new seasons of life — old growth lives in library/vault |

Metaphor: **potting bench** — not project management.

# Evidence Vault

### 1. Arrival experience

Dark warm vault atmosphere — ivory workspace inverts to vault variant. Scene subdued. Kicker *Proof you can solve hard things*. Welcome is confident, private.

Compose uses structured narrative fields — but arrival still shows **one primary story field first** when possible; detail fields follow scroll.

### 2. Primary focal point

**The story of what happened** — situation / primary narrative field. Vault is testimony-first; structure supports memory, not bureaucracy.

Browse section *Records at rest* (or vault-specific collection title) sits below compose.

### 3. Creating a new entry

Feels like **depositing testimony**:

1. What happened (situation — may prefill from chat)
2. What problem was solved
3. What you did
4. Why your approach mattered
5. Outcome / why it mattered

Fields read as sentences, not form IDs. Save: *Preserve in vault* (or approved vault save line).

Keepsakes optional — screenshot, email, photo of outcome.

### 4. Viewing a single entry

**Record mode** — narrative body prominent; structured fields as labeled story sections with generous spacing. Category as uppercase whisper if used.

Dark-theme text: warm cream on vault — readable, not spy-movie.

Actions: *Open record* · remove whispers.

### 5. Browsing many entries

Vertical records — each entry a proof story preview (4 lines). Confidence-building scan: visitor sees handled, solved, prevented language in their own words.

No severity badges. No “impact score.”

### 6. Finding older entries

Search across all narrative fields.

Category filter as text links — *client*, *family*, *process*, etc. if categorized.

Member recalls by **story**, not ID — search is essential by year five.

### 7. Organizing entries

Category optional at capture. Chronological default.

Vault organizes by **life order** — Spark may later cluster “client rescue” stories in conversation, not in-room folders.

### 8. Related entries

When viewing or in chat:

> *"You solved something similar when the billing crisis hit."*

One link. Opens sibling record.

Graduation: record may link to Achievement volume — *"This became the case study on your shelf"* — text in chat after, not forced UI edge.

### 9. Spark interactions inside the room

| When | Spark |
|------|-------|
| Arrival | Quiet confidence welcome |
| Return | *"Your proof is here when you need it."* |
| After save | *"This story now lives safely in your Vault."* |
| Browse (silent) | Atmosphere only |
| Hard day in chat | May suggest opening vault — permission |
| Before challenge | *"Want to remember how you handled something similar?"* |

Never: *"Add more evidence to build confidence."*

### 10. How the room evolves over years

| Era | Interior life |
|-----|----------------|
| **Year 1** | Handful of records; each feels significant |
| **Years 2–4** | Browse before hard meetings becomes ritual |
| **Years 5–7** | Search by story; patterns visible to member |
| **Years 8–10** | Vault is legacy of capability — interview, talks, mentoring |

Metaphor: **private archive of proof** — not compliance storage.

# Achievement Library

### 1. Arrival experience

Library atmosphere — elegant, warm. Kicker *The Estate Library*. Scene plate like a glimpse down the stacks, not a hero banner.

Compose: *A place on the shelf* — title and story feel literary.

Browse *Volumes on the shelf* is the long-term heart of return visits.

### 2. Primary focal point

**Dual on first visit:** compose for new volume.

**On return with shelf:** browse dominates scroll position — but compose remains one calm scroll away.

Inside a volume view: **title + story** are focal — spine metaphor in list, full volume when open.

### 3. Creating a new entry

Feels like **placing a book on the shelf**:

1. Kind of work (select)
2. Title on the spine (required)
3. The story behind it (primary textarea — large)
4. Year optional
5. Cover keepsake optional

Save: *Rest on the shelf*. Confirmation: *Another volume now rests on your shelf.*

### 4. Viewing a single entry

**Volume mode** — grid in list becomes full read:

- Leather spine + optional cover image (left metaphor in list)
- Title in serif large
- Year as quiet meta
- Story body — 3-line preview in list, full text when opened

*Open volume* to edit. Keepsake cover tappable — opens image, not workflow.

Treasured volumes: warmer divider — same as journal treasure.

### 5. Browsing many entries

**Single-column shelf** — each row: spine | optional cover thumb | title + story preview + year.

Designed for **500 volumes** — still one column, hairlines, no card grid exhaustion.

Visitor strolls the shelf — notices eras, creative phases, courage accumulating.

Load more: *View more volumes* — italic underline, not button chrome.

### 6. Finding older entries

Search: *Search the shelves…*

Filter: *Kind of work* — text links.

Year visible on entries — eye finds era without sort controls.

### 7. Organizing entries

Kind of work at save. Treasured flag. Chronological default.

No shelves A–Z. No manual ordering. **Spine title** is how humans remember.

### 8. Related entries

One whisper when opening volume:

> *"This sits beside your first course — different season, same courage."*

Anniversary: *"Three years since this volume was placed here."*

Hall chapter link for landmark volumes — conversational, not badge.

### 9. Spark interactions inside the room

| When | Spark |
|------|-------|
| Arrival | *"Your life's work gets a shelf."* |
| Return | *"Your shelf has grown."* |
| After save | Volume confirmation line |
| Browse | Silent — room is the guide |
| Bio/resume chat | *"Several volumes might speak to this — want to browse together?"* |
| Launch anniversary | Quiet observation in chat |

Never: *"Document your achievements."*

### 10. How the room evolves over years

| Era | Interior life |
|-----|----------------|
| **Year 1** | First volumes precious; shelf sparse but beautiful |
| **Years 2–4** | Shelf fills; browse is pride without boasting |
| **Years 5–7** | Volumes become legacy source — marketing, bios |
| **Years 8–10** | Library is autobiography of work — sparse new volumes |

Metaphor: **private library** — not media CMS.

# Celebration Garden

### 1. Arrival experience

Garden light — joyful, elegant. Kicker *Wins in bloom*. Welcome invites wins of any size.

Compose *Plant a win* before *In bloom* — but returning visitors see blooms rise in scroll naturally.

### 2. Primary focal point

**What are we celebrating?** — primary textarea. Bloom list uses stem mark — subtle, not gamified.

### 3. Creating a new entry

Feels like **planting**:

1. What are we celebrating? (required)
2. Why it matters (optional)
3. Who or what helped? (optional)
4. Date optional
5. Photo from the moment optional

Save: *Plant in garden*. Confirmation: *Your win has been planted.*

### 4. Viewing a single entry

**Bloom mode** — quote-weight body for the win; why-it-matters and gratitude as soft fields below. Stem + mark in list, not trophy icons.

Date as gentle meta. Keepsake photo if present.

*Refine bloom* · remove — quiet links.

### 5. Browsing many entries

*In bloom* — vertical path of blooms. Each: stem | body preview | optional fields | date.

Feels like **walking a garden path** — wins of mixed scale interwoven (water + wedding week + email sent).

No leaderboard. No “this week’s wins” dashboard.

### 6. Finding older entries

Search: *Search your garden…*

Treasured blooms — visual warmth.

Date helps anniversary discovery — *one year ago* is Spark’s job in chat, not push UI.

### 7. Organizing entries

Chronological path. Treasured.

Optional win date — not required. No tags required.

### 8. Related entries

Anniversary in chat: *"One year ago today you planted…"*

If win was also solve: Vault link offered once in conversation.

Journal link if memory-rich: *"Want the fuller story on a quiet page?"*

### 9. Spark interactions inside the room

| When | Spark |
|------|-------|
| Arrival | Joyful, brief |
| Return | *"Your garden has grown."* |
| After save | Planted confirmation |
| Hard season (chat) | One bloom retrieved — permission |
| Minimized win | *"Small things count here."* |

Never: streaks, points, *"plant more wins."*

### 10. How the room evolves over years

| Era | Interior life |
|-----|----------------|
| **Year 1** | Few blooms; each planted deliberately |
| **Years 2–4** | Garden dense; browse lifts mood |
| **Years 5–7** | Anniversaries rich; hard years show courage in small blooms |
| **Years 8–10** | Garden balances Hall — daily joy alongside rare chapters |

Metaphor: **planted wins** — not gamification.

# Celebration Hall

### 1. Arrival experience

Grand, museum hush — scene plate slightly taller than other rooms but still not cinematic distraction. Kicker *Honored chapters*.

Sparse browse is **expected** — empty hall still dignified.

Compose *Honor a chapter* — used rarely; each addition feels weighty.

### 2. Primary focal point

**Chapter title + story** when composing.

**When browsing:** centered chapters with museum spacing — title as plaque, body as narrative beneath.

### 3. Creating a new entry

Feels like **inscribing a chapter**:

1. Chapter title (required)
2. Milestone date (optional but encouraged)
3. What belongs in this chapter? (primary story — large)

Save: *Honor chapter*. Confirmation: *This chapter is honored in the Hall.*

No photos required — dignity over media.

### 4. Viewing a single entry

**Chapter mode** — centered text, max-width column, milestone date as uppercase quiet plaque meta.

Extra fields minimal. Long preview lines in list (5) — chapters deserve space.

*Refine chapter* — rare; hall entries are mostly permanent.

### 5. Browsing many entries

*Honored chapters* — museum layout: one column, **large gaps** (2.5–3rem). Each chapter feels alone on the wall.

Few entries is correct. Many entries still calm — no density increase, more scroll.

### 6. Finding older entries

Search: *Search honored chapters…*

Milestone date is primary find mechanism — era navigation by eye.

No category filters required — hall is inherently “landmark.”

### 7. Organizing entries

Chronological by milestone date when present; else created order.

No reorder. No folders. **Scarcity is organization.**

### 8. Related entries

> *"Five years since you honored your business launch."*

> *"The same spring you placed your first book on the shelf."*

Cross-room references in Spark chat or single line under chapter — never link graph UI.

### 9. Spark interactions inside the room

| When | Spark |
|------|-------|
| Arrival | Reverent, brief |
| Return | *"Your honored chapters are here."* |
| After save | Honor confirmation |
| Milestone anniversary | Chat observation — optional reflect |
| Browse | Silence — let hall speak |

Never pressure new chapters.

### 10. How the room evolves over years

| Era | Interior life |
|-----|----------------|
| **Year 1** | 0–1 chapters — hall feels open |
| **Years 2–5** | Slow accumulation — wedding, launch, graduation |
| **Years 6–8** | Timeline of life visible in one scroll |
| **Years 9–10** | Legacy room — rarely new; often revisited |

Metaphor: **private museum of chapters** — not reporting.

## Implementation sequence (when code begins)

**Do not start until this blueprint is approved.**

Suggested order — interior only, one room at a time:

1. **Journal** — focal compose order, save copy, browse read mode
2. **Garden** — bloom list + plant save copy
3. **Vault** — vault theme + record read mode
4. **Greenhouse** — remove any PM residue; season language
5. **Library** — shelf browse at scale; volume view
6. **Hall** — museum spacing; chapter view

Each room: map sections 1–10 to existing engine slots (header, compose, collection, browse bar, cards) — **no new engine**.

## Interior release gate

Before shipping each room’s interior pass:

1. **Arrival:** Focal point obvious in 3 seconds?
2. **Create:** Feels like the metaphor (page, seedling, testimony, volume, bloom, chapter)?
3. **Single view:** Reading pleasure — not record inspection?
4. **Browse:** 15 minutes enjoyable at 100+ entries?
5. **Find:** Search + one filter type sufficient — no folder craving?
6. **Related:** At most one thread — not a graph?
7. **Spark:** Quieter inside room than in chat?
8. **Ten years:** Metaphor still holds?
