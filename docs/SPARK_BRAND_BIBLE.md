## Mission & Vision

### Mission

Help entrepreneurs become **more capable**, **more confident**, and **more effective** — through clearer thinking, better decisions, stronger businesses, and meaningful transformation.

Not productivity for its own sake.  
**Meaningful progress.**

### Vision

A lifelong **Entrepreneur Development Estate** where:

- Every room serves growth  
- Every conversation strengthens trust  
- Every lesson builds capability  
- Every member feels: *"I don't have to figure this out alone."*

Spark becomes the place entrepreneurs return to — not because they must, but because **it helps them become who they are building toward**.

## Voice & Tone

### How Shari speaks

| Shari is | Shari is not |
|----------|--------------|
| Warm · plainspoken · conversational | Corporate · robotic · essay-like |
| One thought at a time | Framework dumps (unless asked) |
| Curious · patient · hopeful | Pushy · judgmental · rushed |
| A companion at the table | Software announcing features |
| Honest about uncertainty | Pretending to know everything |

**Full rules:** [Spark Human Voice Rules™](./SPARK_HUMAN_VOICE_RULES.md) · [Spec 111 — Hospitality](./SPARK_HOSPITALITY_FRAMEWORK.md) · [Relationship Constitution](./RELATIONSHIP_CONSTITUTION.md)

### The Shari test

Before any copy ships:

> Could Shari say this to someone sitting across from her?

If no → rewrite.

### What Spark never says

| Never | Prefer |
|-------|--------|
| Error · Failed · Required · You must | Plain, human language |
| Opening the Conservatory… | Let's think about this together |
| Choose a room · Navigate to | Would it help to move somewhere quieter? |
| Welcome back! We missed you! (day-count) | I'm glad you're here |
| You're behind · Stay on track | What would help most today? |
| As an AI / Here's a breakdown | Natural speech |
| Optimize · Leverage · Synergy | Plain words |

**Runtime:** `lib/humanConversation/sparkHumanVoice.ts`

## Estate Philosophy

The Estate is an **ecosystem**, not a menu.

**Governing rule:** [Conversation Front Door™](./ESTATE_ROOMS_FRAMEWORK.md#the-conversation-front-door) — if something can be done by talking to Spark, members should not need a map.

### What rooms represent (emotional purpose)

| Place | Emotional question the room answers |
|-------|-------------------------------------|
| **Welcome Home** | *I'm here — what now?* Arrival, belonging, calm start |
| **Conservatory™** | *I need to think out loud without pressure* — conversation primary |
| **Clear My Mind™** | *Too much in my head* — capture, relief, no completion trap |
| **Peaceful Places™** | *I need to restore* — no workspace; reflection only |
| **Coffee House™** | *Light connection and warmth* — low-stakes presence |
| **Creative Studio™** | *I want to make something* — creation with ownership |
| **Observatory™** | *What's changing in the world?* — curated curiosity |
| **Momentum Builder™** | *What's the easiest forward step?* — momentum, not guilt |
| **Momentum Institute™** | *I want to become more capable* — transformation, not courses |
| **Decision Compass™** | *I need to choose well* — clarity before commitment |
| **My Estate™ / My Journey™** | *Remember who I'm becoming* — private growth record |
| **Homestead / Plan My Day** | *What matters today?* — gentle structure |

**Full framework:** [Estate Rooms (T-017)](./ESTATE_ROOMS_FRAMEWORK.md) · [Spec 108 — Environment Integration](./SPARK_ENVIRONMENT_INTEGRATION_FRAMEWORK.md)

### Room design principles

- Feel like **real places** — photographic, warm, intentional  
- **Premium estate** — never gamified, busy, or toy-like  
- **Lanterns and candles flicker gently** — alive, not strobe ([Estate Light Flicker](../.cursor/rules/estate-light-flicker.mdc))  
- **Conversation travels** — never restart on room change  
- **Optional always** — staying where you are is valid

## Visual Language

### Emotional target

**Luxurious · calm · uncluttered · ADHD-friendly**

Members should feel clarity before they read a word.

### Color & atmosphere

| Quality | Direction |
|---------|-----------|
| Warmth | Estate browns, cream, soft gold — not cold SaaS gray |
| Calm | Low visual noise; generous space |
| Legibility | Strong contrast on glass; no pale gray on frosted panels |
| Motion | Subtle, warm light flicker; respect `prefers-reduced-motion` |

### Typography (conversational surfaces)

Per [Spec 109 — Frosted Workspace](./SPARK_FROSTED_CONVERSATION_WORKSPACE_FRAMEWORK.md):

| Element | Desktop minimum |
|---------|-----------------|
| Shari message | 30–36px |
| Member message | 26–30px |
| Buttons / input | 24–30px |
| Supporting | 22–24px |

### Imagery

- **Photographic estate scenes** — edge-to-edge in immersive rooms  
- **Timeless** — not trend-chasing UI chrome  
- **Human scale** — places people want to stay, not dashboards they endure  
- **Estate Image Bible™** — [ESTATE_IMAGE_BIBLE.md](./estate/ESTATE_IMAGE_BIBLE.md) — Spark flame logo on every mug, brass/pen/journal standards, master image prompt block for all room plates

### What we avoid

- Enterprise software aesthetics  
- Dashboards unless they **reduce** cognitive load  
- Gamification (streaks, points, badges, leaderboards)  
- Harsh pulses, fast strobes, arcade flash  
- Competing panels fighting for attention  

**Related:** [Spec 103 — Universal Experience](./UNIVERSAL_EXPERIENCE_STANDARDS_FRAMEWORK.md) · [Universal Experience (T-003)](./UNIVERSAL_EXPERIENCE_STANDARDS.md)

## Educational Standards™

### Institute purpose

Teach entrepreneurs. Develop **capabilities**. Not courses. Not information dumps. **Transformation.**

**Architecture:** [Momentum Institute](./MOMENTUM_INSTITUTE_ARCHITECTURE.md) · [Curriculum Master Index](./SPARK_CURRICULUM_MASTER_INDEX.md)

### Learning lifecycle

Every experience should support:

```
Discover → Learn → Reflect → Make It Mine™ → Coach → Apply → Return → Evidence → Growth
```

### Every lesson must answer

1. Why does this matter?  
2. How do I use it?  
3. How can I apply it in my business?  
4. Would I like help implementing it?  
5. What happened after I tried it?  

**Learning is not complete until it changes something.**

### Experience types (Institute)

| Type | Role |
|------|------|
| Knowledge Card™ | One canonical concept |
| Business Mastery Minute™ | Short capability burst |
| Business Lab™ | Hands-on practice |
| Simulation™ | Safe rehearsal |
| Challenge™ | Real-world application |
| Apprenticeship™ | Guided multi-week development |

### Source integrity (non-negotiable)

- No invented sources  
- Facts require verified citations  
- Spark synthesis never presented as direct quotes  
- [Source Integrity Checklist](./SPARK_BUSINESS_BRAIN.md#source-integrity) before publish

## Design Rules

### Luxurious, calm, uncluttered, ADHD-friendly

| Rule | Application |
|------|-------------|
| One primary action per screen | T-003 |
| Max three choices when choices appear | T-003 · 106 |
| Progressive disclosure | Product Constitution |
| Input always visible in conversation | Spec 109 |
| No scroll-chasing in chat | Spec 109 |
| Executive function first | Reduce memory, planning, decision load |
| Orientation before action | Every screen answers what to do now |
| Warm light flicker on flames | Estate Light Flicker CSS |
| Reduced motion respected | Accessibility non-negotiable |

### Release gate (all must be yes)

From [Relationship Constitution](./RELATIONSHIP_CONSTITUTION.md):

1. Does this make someone feel more **seen**?  
2. Does it reduce **stress**?  
3. Does it sound like **Shari**?  
4. Does it feel **safe**?  

From [Manifesto](./SPARK_MANIFESTO.md) **The Standard** — overwhelm, thinking, momentum, implementation, confidence, entrepreneurship.
