# Spark Card™ Collapsed Visual Design Fix

## Purpose

Correct the collapsed Spark Card™ implementation.

The current version appears as a notification banner. This is incorrect.

Spark Card™ is a companion object, similar in visual importance to the
Spark Estate Guide Book™.

The feeling should be:

"A little Spark is waiting for me."

Not:

"I have a notification."

------------------------------------------------------------------------

# Placement

Spark Card™ belongs:

-   bottom-right corner of the screen
-   opposite the Spark Estate Guide Book™
-   visible but unobtrusive

------------------------------------------------------------------------

# Shape Requirement

Do NOT create:

-   horizontal notification bars
-   pill-shaped alerts
-   banners
-   long rectangles
-   message strips

Create:

-   a small card object
-   compact proportions
-   similar visual weight to the Guide Book

------------------------------------------------------------------------

# Collapsed Content

Show only:

-   Spark flame icon
-   Spark Card™ identity
-   optional small indication of today's Spark

Do not show:

-   full story
-   long teaser
-   multiple buttons
-   article content

The purpose is curiosity.

------------------------------------------------------------------------

# Icon Rule

Use the approved Spark flame.

Do not use:

-   duplicate flame icons
-   generic notification symbols
-   unrelated icons

------------------------------------------------------------------------

# Click Behavior

The entire card should open the expanded Spark Card™ experience.

It should feel like opening a card, not opening an alert.

------------------------------------------------------------------------

# Success Test

The user should think:

"What is inside?"

Not:

"What notification did I receive?"

Spark Card™ is a companion object.
