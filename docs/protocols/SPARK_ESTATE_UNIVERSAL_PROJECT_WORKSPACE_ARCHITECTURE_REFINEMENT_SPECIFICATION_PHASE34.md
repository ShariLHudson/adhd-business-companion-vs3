# Spark Estate™ Universal Project Workspace Architecture Refinement Specification

## Purpose

This document defines the corrected project experience for Spark
Estate™.

The goal:

Projects, SOPs, funnels, templates, strategies, and other long-term
creations should become living workspaces that follow the member
throughout the entire app.

A project is not owned by a room.

A project belongs to the member.

------------------------------------------------------------------------

# Core Principle

The Chamber of Momentum™ is a place where projects can be created and
organized.

It is not the only place projects exist.

The project follows the member.

------------------------------------------------------------------------

# Remove Old Workspace Model

The previous workspace concept should be replaced.

Do not create:

-   isolated project areas
-   projects that only exist inside one room
-   disconnected workspaces
-   duplicate copies of the same project

------------------------------------------------------------------------

# New Universal Project Model

A project is a living object available throughout Spark Estate™.

Flow:

Member Project

↓

Available Everywhere

↓

Room / Conversation / Feature

↓

Project Context Available

↓

Continue Progress

------------------------------------------------------------------------

# Project Entry Points

A project can begin from anywhere.

Examples:

## Chamber of Momentum™

"I need to create a sales funnel."

## Marketing Room

"I need a campaign strategy."

## Email Area

"I need a client email sequence."

## Conversation

"I have an idea I want to build."

Spark recognizes when something should become a project.

------------------------------------------------------------------------

# Project Library

Members need a central place to find their work.

The library should include:

-   active projects
-   completed projects
-   drafts
-   archived projects
-   recently updated work

Search should include:

-   projects
-   SOPs
-   templates
-   funnels
-   strategies
-   documents

------------------------------------------------------------------------

# Universal Project Access

The member should be able to access an active project from anywhere.

Examples:

## Marketing Room

Continue marketing work connected to the project.

## Email Area

Create emails connected to the project.

## Content Room

Create content connected to the project.

## Chamber of Momentum™

See overall progress and next steps.

------------------------------------------------------------------------

# Current Project Indicator

The app should show the member's active project context.

Example:

"Working on: Sales Funnel Project"

This should be accessible throughout the app.

Selecting it opens the project workspace.

------------------------------------------------------------------------

# Spark Continuity

When returning to a project, Spark should know:

-   what the project is
-   what has been completed
-   what remains
-   where the member stopped
-   the next helpful action

Example:

"Welcome back. Your sales funnel project is ready to continue. Last time
we completed the audience and offer sections. Would you like to work on
the email sequence next?"

# ADHD-Friendly Requirements

The system should reduce:

-   searching
-   remembering locations
-   restarting work
-   losing ideas

The member should never wonder:

"Where did I save that?"

"Which room was I using?"

"How do I continue?"

Spark should provide continuity.

------------------------------------------------------------------------

# Final Vision

Spark Estate™ organizes around the member's work, not around app
features.

Rooms provide expertise.

Projects provide continuity.

Spark provides guidance.

The member can work anywhere and their progress follows them.
