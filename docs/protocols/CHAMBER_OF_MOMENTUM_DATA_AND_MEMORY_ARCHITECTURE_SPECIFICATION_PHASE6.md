# Chamber of Momentum™ Data and Memory Architecture Specification - Phase 6

## Purpose

This document defines what Chamber of Momentum™ remembers about the
member and how those memories support future progress.

The goal:

Transform Momentum from a one-time helper into a long-term companion.

The system should learn:

"What helps this person move forward?"

------------------------------------------------------------------------

# Core Principle

Momentum remembers patterns, not just information.

The purpose of memory is not to collect everything.

The purpose is to reduce friction.

------------------------------------------------------------------------

# Memory Categories

Momentum memory should be organized into clear categories.

------------------------------------------------------------------------

# 1. Project Memory

Purpose:

Remember active and completed work.

Store:

-   project name
-   desired outcome
-   status
-   next action
-   milestones
-   blockers
-   completion date
-   lessons learned

Examples:

"Website launch"

"Create coaching program"

"Develop new product"

------------------------------------------------------------------------

# 2. Momentum History

Purpose:

Remember movement and progress.

Store:

-   actions completed
-   milestones reached
-   breakthroughs
-   resets
-   completed steps

This answers:

"What progress has this person already made?"

------------------------------------------------------------------------

# 3. Evidence Bank

Purpose:

Store proof of capability.

Examples:

-   completed projects
-   customer wins
-   positive feedback
-   skills developed
-   personal accomplishments

Use when the member needs confidence.

Example:

"You have successfully completed projects like this before."

------------------------------------------------------------------------

# 4. Wins Memory

Purpose:

Celebrate progress.

Store:

-   weekly wins
-   accomplishments
-   small victories
-   resilience moments

Small wins matter.

Momentum should recognize movement, not only completion.

------------------------------------------------------------------------

# 5. Pattern Memory

Purpose:

Learn how the person works best.

Possible patterns:

-   best work time
-   preferred planning style
-   common blockers
-   energy patterns
-   successful strategies
-   preferred level of detail

Example:

"This person completes tasks better when the first step is very small."

------------------------------------------------------------------------

# 6. Blocker Memory

Purpose:

Recognize repeated obstacles.

Examples:

-   perfectionism
-   unclear next steps
-   too many ideas
-   difficulty starting
-   low energy periods
-   fear of making decisions

Memory should help support, not label.

------------------------------------------------------------------------

# 7. Preference Memory

Purpose:

Remember how the person prefers to interact.

Examples:

-   communication style
-   language
-   preferred encouragement
-   visual preferences
-   planning preferences

------------------------------------------------------------------------

# What Momentum Should NOT Store

Do not store unnecessary personal information.

Do not collect:

-   private details unrelated to helping progress
-   sensitive information without purpose
-   temporary emotional moments without value

Memory should serve the member.

------------------------------------------------------------------------

# Memory Connections

The ecosystem should connect:

    Spark Card™

          ↓

    Reflection / Idea

          ↓

    Idea Vault

          ↓

    Chamber of Momentum™

          ↓

    Project

          ↓

    Progress

          ↓

    Win

          ↓

    Evidence Bank

          ↓

    Future Support

------------------------------------------------------------------------

# Using Memory During Conversations

Momentum may use memory to:

-   suggest next steps
-   recognize patterns
-   celebrate progress
-   reduce repeated explanations

Example:

"I remember you prefer starting with a small first step. Would that help
here?"

------------------------------------------------------------------------

# Learning What Works

Momentum should track successful approaches.

Examples:

A member responds well to:

-   breaking tasks into tiny steps
-   visual planning
-   deadlines
-   encouragement
-   examples

Future guidance can adapt.

------------------------------------------------------------------------

# Memory Safety Rules

Memory should be:

-   helpful
-   transparent
-   editable
-   respectful

The member should understand:

"What does Spark remember about me?"

------------------------------------------------------------------------

# Demo Requirements

Verify:

✓ Projects can connect to progress

✓ Wins can become evidence

✓ Previous progress can be referenced

✓ Patterns can improve future guidance

✓ Memory does not create clutter

------------------------------------------------------------------------

# Final Goal

The member should feel:

"Spark understands how I work."

Not:

"Spark is storing information about me."

Momentum memory exists to help people move forward with less friction.
