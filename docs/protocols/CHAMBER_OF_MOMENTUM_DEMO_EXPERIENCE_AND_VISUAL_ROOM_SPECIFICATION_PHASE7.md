# Chamber of Momentum™ Demo Experience and Visual Room Specification - Phase 7

## Purpose

This document defines the member-facing experience of Chamber of
Momentum™.

The goal:

Create a room that feels like a supportive place to make progress.

Chamber of Momentum™ is not a project dashboard.

It is not a collection of tools.

It is a guided environment where members discover their next best step.

------------------------------------------------------------------------

# Experience Identity

The feeling:

"I can move forward from here."

The room should feel:

-   encouraging
-   focused
-   calm
-   capable
-   supportive

The user should not feel:

-   overwhelmed
-   behind
-   confused
-   like they entered a complicated system

------------------------------------------------------------------------

# Room Entry Experience

When a member enters Chamber of Momentum™:

Do not immediately display:

-   project boards
-   task lists
-   knowledge libraries
-   multiple feature cards

Start with orientation.

------------------------------------------------------------------------

# Arrival Screen

Display:

## Welcome

"Welcome to the Chamber of Momentum™"

Supporting message:

"Let's find what will help you move forward today."

------------------------------------------------------------------------

# Primary Interaction

Ask:

"What would help you move forward today?"

Provide simple choices:

## 💡 I have an idea

Purpose:

Capture and explore ideas.

------------------------------------------------------------------------

## 🌪 I feel stuck

Purpose:

Get unstuck and find the next step.

------------------------------------------------------------------------

## 📋 I need a plan

Purpose:

Create structure.

------------------------------------------------------------------------

## 📚 I want to learn

Purpose:

Build knowledge and skills.

------------------------------------------------------------------------

## 🚀 I want to work on a project

Purpose:

Move an existing goal forward.

------------------------------------------------------------------------

# Room Layout Principles

The room should not feel like a software dashboard.

Use:

-   one primary focus area
-   calm visual hierarchy
-   limited choices
-   clear next action

------------------------------------------------------------------------

# Visible Elements

## Primary Conversation Area

Purpose:

The member's starting point.

Used for:

-   questions
-   guidance
-   reflection
-   next steps

------------------------------------------------------------------------

## Momentum Path Area

Shows current movement.

Examples:

-   First Step™
-   Easy Wins™
-   Focus Session™
-   Roadblocks™

Only show when relevant.

Do not show empty sections.

------------------------------------------------------------------------

## Progress Recognition Area

Show:

-   recent wins
-   completed steps
-   progress moments

Purpose:

Help members see movement.

------------------------------------------------------------------------

# Hidden Until Needed

Do not display by default:

-   large project lists
-   all tasks
-   all learning cards
-   all tools
-   complicated planning systems

Reveal based on need.

------------------------------------------------------------------------

# Room Components

## Momentum Builder™

Appears when the member needs:

-   clarity
-   encouragement
-   next step support

------------------------------------------------------------------------

## Momentum Institute™

Appears when the member needs:

-   learning
-   resources
-   skill building

------------------------------------------------------------------------

## Goals & Projects™

Appears when the member needs:

-   execution
-   project management
-   task organization

------------------------------------------------------------------------

# Visual Rules

The room must have:

## Correct Room Identity

The name must match:

-   image
-   route
-   manifest entry

No mismatched room names and backgrounds.

------------------------------------------------------------------------

# Navigation

The member should enter through:

🔥 Chamber of Momentum™

The internal systems should remain behind the scenes.

------------------------------------------------------------------------

# ADHD-Friendly Requirements

The room should:

-   show one next step
-   avoid overwhelming choices
-   reduce cognitive load
-   celebrate progress
-   allow easy re-entry

The member should always know:

"What do I do next?"

------------------------------------------------------------------------

# Demo Flow

## Example 1

Member enters.

System asks:

"What would help you move forward today?"

Member selects:

"I feel stuck."

Result:

Momentum Builder™ conversation begins.

------------------------------------------------------------------------

## Example 2

Member selects:

"I want to work on a project."

Result:

Project workflow opens.

------------------------------------------------------------------------

## Example 3

Member selects:

"I want to learn."

Result:

Momentum Institute™ resources open.

------------------------------------------------------------------------

# Success Test

A first-time user should understand within seconds:

"This is where I go when I need help making progress."

They should not need to understand the technology underneath.

------------------------------------------------------------------------

# Final Experience Goal

Chamber of Momentum™ should feel like:

A trusted workspace with a built-in guide.

Not:

A collection of disconnected features.
