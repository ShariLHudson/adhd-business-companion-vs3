# 314 — Universal Testing and Certification Framework

## Purpose

Define the platform-wide testing and certification standard.

## Certification levels

### Architecture certification

Validates:

- ownership
- source of truth
- contracts
- registries
- dependencies
- migration
- no duplication

### Member certification

Validates:

- knowledge
- routing
- retrieval
- conversation
- implementation
- capabilities
- Collections
- adaptation
- maintenance

### Capability certification

Validates:

- owner
- scope
- inputs
- outputs
- evidence
- dependencies
- adaptation
- provenance
- consumers

### Collection certification

Validates:

- owner
- contributors
- capability resolution
- assets
- routing
- adaptation
- coherent experience
- no duplication

### Blueprint certification

Validates:

- Work Type compatibility
- one Work ID
- depth modes
- resume
- outputs
- universal integrations
- domain quality

## Required testing layers

- schema tests
- unit tests
- contract tests
- registry tests
- integration tests
- routing tests
- retrieval tests
- conversation tests
- browser tests
- accessibility tests
- cognitive-load tests
- migration tests
- security and privacy tests
- performance tests
- regression tests
- recovery tests

## Cognitive-load certification

Verify:

- no unnecessary repeated questions
- energy and motivation remain separate
- visible options are limited by context
- low-energy mode is genuinely lighter
- exact resume works
- no punitive unfinished-work language
- recommendations explain why
- users can decline or defer
- the platform does not force navigation

## Expertise adaptation certification

Test:

- advanced knowledge/high confidence
- advanced knowledge/low confidence
- low knowledge/high confidence
- low knowledge/low confidence
- practical experience/low formal knowledge
- high formal knowledge/low application experience

## Provenance certification

Every contributed output must retain:

- capability
- owner
- contributor
- evidence
- assumptions
- version
- approval state

## Browser verification minimum

For user-facing systems verify:

1. entry from supported origins
2. existing Work resume
3. create/connect/conversation-only choices
4. adaptive depth
5. low-energy behavior
6. user correction
7. approved mutation
8. refresh
9. exact resume
10. no duplicate Work

## Certification verdicts

- PRODUCTION CERTIFIED
- CONDITIONALLY CERTIFIED
- NOT CERTIFIED
- CERTIFICATION REVOKED

## Conditional certification

Conditional certification must include:

- exact conditions
- risk
- owner
- deadline
- prohibited use
- retest requirement

## Production-ready definition

Production ready requires:

- implementation complete
- tests passing
- blockers reviewed
- accessibility passing
- privacy reviewed
- migration safe
- rollback documented
- no known source-of-truth violation
- documentation complete
- maintenance owner assigned

## Certification evidence

Store:

- test totals
- test names
- browser checklist
- blocker report
- known limitations
- commit hash
- branch
- version
- certification date
- reviewer
