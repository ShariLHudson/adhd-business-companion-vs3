# 559 — Learning Automatic Visual Intelligence Standard

## Supported visuals
- learning path
- competency ladder
- prerequisite map
- concept map
- process flow
- timeline
- decision tree
- comparison chart
- worked-example sequence
- mastery map
- practice cycle
- retention plan
- transfer map
- misconception map
- progress journey

## Visual triggers
Use a visual when steps interact, prerequisites are unclear, comparison is central, sequence matters, working-memory burden is high, progress is hard to see, or relationships matter more than prose.

## Restraint
Do not create a visual merely because one is possible. Prefer one useful visual over several decorative ones.

## Accessibility
Visuals must use readable labels, avoid crowded layouts, include text alternatives, preserve logical sequence, avoid color-only meaning, and support enlargement.

Learning owns instructional meaning, sequence, emphasis, labels, and mastery state. The shared visual engine owns rendering, interaction, and export.
