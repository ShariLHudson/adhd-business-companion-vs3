# 530 — Knowledge Capture, Validation & Curation Standard

## Purpose

Capture useful knowledge without flooding the platform with duplicates, fragments, or unsupported claims.

## Capture sources

- user-provided information
- approved business documents
- research findings
- decisions
- project lessons
- customer insights
- operating procedures
- Chamber outputs
- Board guidance
- platform usage patterns
- validated external sources

## Capture decision

Before saving, determine:

- Is this reusable?
- Is it already represented?
- Is it temporary?
- Is it an assumption?
- Is it personal preference?
- Is it sensitive?
- Who owns it?
- What evidence supports it?
- Where does it apply?
- When should it be reviewed?

## Curation actions

- accept
- merge
- refine
- split
- contextualize
- link
- quarantine
- reject
- archive
- request evidence
- request owner approval

## Validation dimensions

- source authority
- evidence quality
- consistency
- recency
- scope
- applicability
- owner approval
- conflict with canonical knowledge
- legal or safety implications

## No automatic canonization

AI-generated content, one-off user statements, or imported documents must not become canonical automatically.

## Duplication rule

Prefer:

- one canonical item
- contextual views
- aliases
- relationships
- adaptations

over multiple copies.
