# 533 — Knowledge Evidence, Memory & Lineage Registry

## Purpose

Make every important knowledge claim traceable.

## Evidence registry

Each knowledge item may reference:

- source document
- research source
- user statement
- decision record
- experiment
- analytics result
- policy
- professional guidance
- observation
- project outcome
- date
- owner
- confidence

## Evidence strength

- authoritative
- strongly supported
- supported
- limited
- anecdotal
- assumption
- disputed
- unsupported

## Memory rule

Remember:

- approved preferences
- established decisions
- recurring working patterns
- validated business facts
- accepted standards
- user corrections
- contextual exceptions

Do not remember:

- temporary emotional statements as identity
- unsupported judgments
- sensitive information without purpose
- speculative labels
- outdated assumptions as facts

## Lineage questions supported

- Where did this come from?
- Who approved it?
- What changed?
- What depends on it?
- Which outputs used it?
- What replaced it?
- What evidence contradicts it?
- When must it be reviewed?

## No detached knowledge

Canonical items must retain source, owner, scope, and history.
