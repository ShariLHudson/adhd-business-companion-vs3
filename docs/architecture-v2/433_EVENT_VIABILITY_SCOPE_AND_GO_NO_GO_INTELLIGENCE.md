# 433 — Event Viability, Scope & Go/No-Go Intelligence

## Purpose

Help users determine whether an event is the right intervention and what the smallest viable event should be.

## Viability questions

Evaluate:

- What outcome requires a live or time-bound gathering?
- Could the outcome be achieved more simply?
- Who must participate?
- What would make attendance worthwhile?
- What capacity does the user actually have?
- What decisions remain unresolved?
- What dependencies are outside the user’s control?
- What would failure cost?
- What is the minimum useful version?
- What conditions must be true before committing?

## Possible recommendations

- proceed
- proceed with reduced scope
- run a pilot
- choose a different format
- partner with someone
- postpone
- cancel
- replace the event with another creation
- gather evidence first

## Go/no-go categories

- ready_to_commit
- conditionally_viable
- pilot_recommended
- redesign_required
- insufficient_information
- not_currently_viable

## Anti-excitement overbuild rule

Do not allow excitement to substitute for:

- audience evidence
- time capacity
- budget realism
- venue or platform feasibility
- delivery capability
- follow-up capacity

## Shari language

> “The idea is strong, but the current version is carrying more complexity than the outcome requires. A two-hour pilot workshop would let you test the experience before committing to a full-day event.”
