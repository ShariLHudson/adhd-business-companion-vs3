# 584 — Momentum Automatic Visual Intelligence Standard

## Purpose

Make movement, friction, continuity, and progress easier to see without turning the user into a productivity dashboard.

## Supported visuals

- momentum path
- activation sequence
- friction map
- current-focus map
- progress trail
- work-in-progress view
- stall diagnosis map
- re-entry path
- energy-to-action map
- completion pathway
- progress timeline
- consistency pattern
- effort-versus-impact map
- celebration journey
- continuity chain

## Automatic visual triggers

Consider a visual when:

- the user cannot see where to begin
- multiple blockers interact
- work has been interrupted
- progress feels invisible
- active commitments are competing
- the user needs a return point
- completion steps are unclear
- patterns across time would be useful

## Visual restraint

Do not create:

- guilt-inducing streak displays
- red failure dashboards
- excessive completion percentages
- productivity rankings
- decorative charts without decisions

## Truth rules

Visuals must distinguish:

- planned versus completed
- active versus paused
- interruption versus abandonment
- evidence versus assumption
- user-confirmed pattern versus inferred pattern

## Ownership boundary

Momentum owns the meaning of state, friction, progress, and continuation.

The shared visual engine owns rendering, interaction, accessibility, and export.
