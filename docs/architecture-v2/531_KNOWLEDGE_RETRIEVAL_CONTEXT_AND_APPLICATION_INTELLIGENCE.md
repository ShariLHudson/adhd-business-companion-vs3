# 531 — Knowledge Retrieval, Context & Application Intelligence

## Purpose

Retrieve the right knowledge for the current user, decision, and moment rather than the largest amount of information.

## Retrieval dimensions

- user intent
- task
- Chamber member
- business context
- person or client
- project
- time
- location
- confidence
- sensitivity
- recency
- authority
- applicability
- user preference
- current energy or cognitive load

## Retrieval priority

Prefer:

1. canonical and applicable
2. current and verified
3. context-specific approved knowledge
4. relevant lessons
5. unresolved or experimental knowledge with labels
6. external information when required

## Context window behavior

Retrieve only what supports the current decision.

Do not dump the entire knowledge base into the conversation.

## Application questions

- Does this knowledge apply here?
- What assumptions differ?
- What exceptions matter?
- What changed since it was approved?
- Is the user asking for explanation, action, or decision support?
- Does another Chamber member own the interpretation?

## User-facing behavior

When uncertainty exists:

> “I found two relevant versions. One is the current approved standard; the other is an older project-specific exception. I’ll use the approved standard and show the exception only where it matters.”
