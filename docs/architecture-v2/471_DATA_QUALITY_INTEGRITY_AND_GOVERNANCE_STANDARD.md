# 471 — Data Quality, Integrity & Governance Standard

## Purpose

Prevent misleading conclusions from incomplete, inconsistent, or poorly defined data.

## Quality dimensions

- accuracy
- completeness
- consistency
- timeliness
- validity
- uniqueness
- relevance
- lineage
- accessibility
- privacy fitness

## Data-quality states

- verified
- usable_with_limits
- incomplete
- inconsistent
- stale
- untrusted
- unavailable

## Required checks

- duplicate records
- missing values
- impossible values
- inconsistent date ranges
- schema changes
- unit mismatches
- definition drift
- sample coverage
- source reconciliation
- manual-entry risk
- tracking gaps

## Governance requirements

Document:

- source owner
- metric owner
- access rules
- retention expectation
- privacy classification
- refresh cadence
- transformation logic
- correction process
- version history

## Honest-analysis rule

When data quality is weak, the response must say so before presenting conclusions.

## Privacy boundary

Use only data that is authorized, necessary, and appropriate for the decision.
Avoid exposing personally identifiable or sensitive information unnecessarily.
