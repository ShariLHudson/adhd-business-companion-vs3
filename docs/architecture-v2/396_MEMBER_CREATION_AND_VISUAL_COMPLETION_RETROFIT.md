# 396 — Member Creation & Visual Completion Retrofit

## Purpose

Apply the shared Create and visual architecture to completed member knowledge without rewriting shared systems.

## For each member define

### Creation capability catalog

For every creation:

- user-described need
- resolved artifact
- promise level
- required information
- source-of-truth owner
- professional or external completion boundary
- adaptation options
- related Project or Blueprint
- automatic visual type

### Automatic visual behavior

Choose only member-relevant maps, such as:

- diagnostic map
- strategy map
- process flow
- timeline
- stakeholder map
- decision map
- dependency view
- learning path
- financial scenario map
- market connection map

### Member boundaries

The member specifies what should appear in the map.

Visual Connection Intelligence owns:

- rendering
- layout
- editing
- deletion
- relationship truth
- versioning
- exact resume
