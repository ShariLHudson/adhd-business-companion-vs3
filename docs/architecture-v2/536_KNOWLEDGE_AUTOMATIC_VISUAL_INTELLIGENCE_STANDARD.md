# 536 — Knowledge Automatic Visual Intelligence Standard

## Purpose

Automatically build useful knowledge maps while preserving canonical truth.

## Supported visuals

- knowledge domain map
- concept map
- ownership map
- source-to-knowledge lineage
- version tree
- contradiction map
- knowledge-gap map
- decision-to-outcome map
- process knowledge map
- taxonomy tree
- evidence map
- context applicability map
- knowledge lifecycle map

## Default display

Show only:

- current knowledge item
- canonical owner
- primary source
- related items
- confidence
- current version
- one unresolved issue

## Visual truth rules

- distinguish canonical from draft
- distinguish evidence from assumption
- label outdated items
- preserve source links
- show local exceptions
- do not imply relationship equals authority
- do not hide contradictions

## Ownership

The shared visual engine owns rendering and interaction.

Knowledge owns meaning, authority, version state, and lineage.
