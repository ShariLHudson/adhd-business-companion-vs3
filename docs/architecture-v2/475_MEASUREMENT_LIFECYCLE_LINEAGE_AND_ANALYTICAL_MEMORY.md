# 475 — Measurement Lifecycle, Lineage & Analytical Memory

## Measurement lifecycle

1. decision identified
2. metric proposed
3. definition approved
4. instrumentation requested
5. data collected
6. quality checked
7. analysis performed
8. insight reviewed
9. action taken
10. outcome monitored
11. definition refreshed
12. metric retired or retained

## Analytical lineage

Every analysis should reference:

- source datasets
- date range
- filters
- transformations
- metric definitions
- assumptions
- method
- version
- analyst or system
- related decision
- resulting action

## Analytical memory

Remember:

- approved metric definitions
- known data limitations
- prior comparisons
- accepted thresholds
- past experiments
- decision outcomes
- recurring anomalies
- measurement preferences

Do not remember unsupported labels such as:

- "this campaign never works"
- "this customer group is bad"

Store evidence and conditions, not biased conclusions.

## Refresh behavior

When data changes materially, identify affected dashboards, reports, forecasts, and decisions.
