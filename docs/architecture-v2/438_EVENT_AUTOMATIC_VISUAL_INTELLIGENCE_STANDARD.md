# 438 — Event Automatic Visual Intelligence Standard

## Purpose

Automatically build the most appropriate visual map as an event is created and delivered.

## Default visual views

- event lifecycle map
- attendee journey
- event ecosystem map
- timeline
- dependency map
- stakeholder map
- asset map
- run-of-show flow
- venue or environment plan
- risk and contingency map
- follow-up and reuse map

## Automatic generation

Visuals are generated from:

- Event Record
- Event Blueprint
- Creation Workspace
- Event Asset Registry
- decisions
- dependencies
- stakeholders
- Project links
- event state

## Default display

Show only:

- current lifecycle phase
- active decision
- immediate dependencies
- next milestone
- key stakeholders

The full event ecosystem remains available but hidden until useful.

## User control

Users may:

- edit labels
- reposition
- group
- hide
- reconnect
- correct
- delete a view
- restore the generated view

Deleting the map does not delete event data or assets.
