# 582 — Momentum Memory, Pattern, Lineage & Progress Registry

## Purpose

Remember what helps the user move without turning temporary difficulty into permanent identity.

## Remember

- meaningful goals
- active momentum state
- successful activation methods
- common friction patterns
- preferred work windows
- useful environmental supports
- effective task sizes
- successful re-entry methods
- completion patterns
- celebration preferences
- current continuation points
- work intentionally paused
- conditions affecting capacity
- progress evidence

## Do not store as identity

- lazy
- inconsistent
- unmotivated
- avoidant
- incapable
- undisciplined
- chronic procrastinator

Store observations with context:

> “Large ambiguous tasks have been easier to begin after Shari helps identify the first physical action.”

## Pattern states

- observation
- repeated pattern
- likely support preference
- context-specific pattern
- user-confirmed pattern
- outdated
- contradicted
- retired

## Progress registry

Each progress record may include:

- work item
- previous state
- new state
- action completed
- date
- effort context
- friction reduced
- evidence
- next continuation point
- celebration preference
- related project or creation

## Lineage

Connect:

- intention
- activation method
- action
- progress
- interruption
- re-entry
- completion
- celebration
- learning

## Privacy rule

Momentum memory should preserve useful support context, not create surveillance or performance scoring.
