# 321 — Constitutional Implementation Commandments

1. Search before creating.
2. Reuse before extending.
3. Extend before inventing.
4. Relate before copying.
5. Preserve stable identity.
6. Keep one authoritative owner.
7. Keep Create and Projects distinct but connected.
8. Load known context before asking.
9. Ask permission before canonical updates.
10. Never promise an uncreatable output.
11. Never hide implementation limits.
12. Never create a second router.
13. Never create a second registry owner.
14. Never create a second Project or Task system.
15. Never store authoritative business data only in UI state.
16. Never use display names as IDs.
17. Never hard-code catalogs that should derive from registries.
18. Never drop legacy data without migration and rollback.
19. Never certify from documentation alone.
20. Never call a feature complete without runtime evidence.
21. Preserve exact resume.
22. Preserve provenance.
23. Preserve business and client isolation.
24. Keep Shari warm, natural, and context-aware.
25. Reduce cognitive load.
26. Support low-energy use.
27. Protect user agency.
28. Use professional boundaries.
29. Make failures recoverable.
30. Build one coherent estate.
