# 391 — Chamber Member Completion Requirements Matrix

Each started member must be checked against the following requirements.

## A. Identity and ownership

- mission
- user outcomes
- canonical scope
- explicit exclusions
- boundary with adjacent members
- accountable capability owner
- escalation rules

## B. Expert intelligence

- domain map
- proven frameworks
- diagnostic models
- tradeoffs
- heuristics
- failure patterns
- edge cases
- evidence quality
- uncertainty handling
- ethical limits
- ADHD adaptations
- low-energy adaptations

## C. Conversation intelligence

- Shari voice
- opening behavior
- discovery behavior
- reflective behavior
- recommendation behavior
- creation behavior
- correction behavior
- returning-user behavior
- stopping behavior
- confidence language
- no premature routing
- no menu-like responses

## D. Routing and collaboration

- entry triggers
- false-positive exclusions
- handoff requirements
- route-away rules
- co-working rules
- Board escalation
- return-to-Shari behavior
- context preservation
- duplicate-owner prevention

## E. Creation capabilities

- supported creation outcomes
- capability promise level
- required user information
- templates and schemas
- source-of-truth ownership
- save/edit/resume
- connected creation sets
- adaptation support
- automatic visual map behavior
- external completion requirements

## F. Runtime and retrieval

- registry entries
- schemas
- metadata
- retrieval tags
- confidence logic
- context contracts
- relationship contracts
- observability
- failure behavior
- versioning
- migration

## G. Validation

- unit tests
- routing tests
- conversation tests
- creation tests
- overlap tests
- visual tests
- ADHD usability tests
- safety tests
- maintenance tests
- certification evidence
