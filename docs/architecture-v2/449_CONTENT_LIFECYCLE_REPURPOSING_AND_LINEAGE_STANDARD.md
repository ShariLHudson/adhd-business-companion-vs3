# 449 — Content Lifecycle, Repurposing & Lineage Standard

## Purpose

Ensure content remains connected, reusable, and maintainable.

## Content lifecycle

1. need or opportunity
2. source idea
3. brief
4. research
5. architecture
6. draft
7. edit
8. approval
9. publication
10. distribution
11. performance review
12. refresh
13. repurpose
14. archive or retire

## Source-of-truth rule

Every content family should identify:

- primary source artifact
- adaptations
- excerpts
- channel versions
- updates
- performance learnings
- current canonical version

## Repurposing rule

Repurposing changes form, depth, angle, or channel while preserving the governing idea.

It should not become shallow repetition.

## Change propagation

When the source changes, identify affected adaptations.

Do not overwrite user-edited adaptations silently.

## Lineage questions supported

- Where did this post come from?
- Which guide is the source?
- Which versions are outdated?
- What content was created from this idea?
- What should be refreshed?
