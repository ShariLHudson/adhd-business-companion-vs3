# 390 — Started Chamber Member Completion Master

## Mission

Finish every Chamber member already started before beginning additional members.

## Governing rule

A Chamber member is not complete because many files exist.

A member is complete only when its:

- identity
- expert knowledge
- reasoning
- conversation behavior
- routing
- creation capabilities
- implementation
- retrieval
- visual behavior
- testing
- maintenance
- certification

work together as one production-ready member.

## Anti-duplication rule

Before creating any new file:

1. identify the capability it serves
2. identify the current canonical owner
3. search existing member and platform files
4. extend the canonical file when appropriate
5. create a new file only when a genuine missing responsibility exists

Shared platform capabilities must be referenced, not rewritten inside every member.
