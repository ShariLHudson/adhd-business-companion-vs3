# 393 — Shared Architecture Reference, Not Rewrite Standard

## Purpose

Prevent every Chamber member from receiving duplicate platform architecture.

## Shared capabilities that members should reference

- Universal Create
- Creation Promise
- Output Capability Manifest
- Creation Discovery
- Progressive Creation
- Context Reuse
- Connected Creation Sets
- Create Once, Adapt Everywhere
- Creation Strategy Intelligence
- Adaptive Creation Intelligence
- Visual Connection Intelligence
- canonical relationship model
- Projects
- Blueprints
- Shari voice authority
- canonical context
- Output Registry
- routing authority
- certification authority

## Member-owned work

Each member should define only:

- its specialized domain knowledge
- its diagnostic reasoning
- its specialized conversations
- its member-specific creation capabilities
- its member-specific routing triggers
- its member-specific safety and boundaries
- its member-specific tests
- its collaboration contracts

## Prohibited duplication

Do not create member-specific copies of:

- general Create runtime
- general visual graph engine
- general memory
- general Projects
- general Blueprint logic
- general output registry
- general certification
- general user profile
- general Shari voice rules

Member files may add specialized constraints, not parallel systems.
