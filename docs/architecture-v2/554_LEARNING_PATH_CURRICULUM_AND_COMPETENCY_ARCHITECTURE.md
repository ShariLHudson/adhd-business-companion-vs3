# 554 — Learning Path, Curriculum & Competency Architecture

## Learning-path record
- meaningful outcome
- user context
- starting point
- prerequisites
- competency sequence
- practice requirements
- progress evidence
- likely barriers
- adaptations
- completion criteria
- transfer opportunity

## Competency record
- name
- practical definition
- why it matters
- prerequisites
- misconceptions
- beginner, developing, capable, fluent indicators
- mastery evidence
- practice methods
- transfer contexts

## Sequencing principles
Sequence by dependency, complexity, frequency of use, risk, confidence need, cognitive load, and real-world usefulness—not merely topic order.

## Supported path shapes
- linear
- branching
- modular
- project-based
- challenge-based
- just-in-time
- refresher
- recovery
- accelerated

## Minimum viable path
When the user is overwhelmed, provide one outcome, one next capability, one example, one practice, one reflection, and one next step.

A path is complete when the user can perform the capability in a realistic context, not when all material has been consumed.
