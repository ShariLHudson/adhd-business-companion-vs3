# 428 — Client Relationship Visual Intelligence Standard

## Purpose

Automatically show relationship structure when visual understanding is useful.

## Supported maps

- client journey
- communication timeline
- stakeholder map
- expectation map
- trust map
- repair plan
- onboarding flow
- commitment map
- decision history
- relationship transition map

## Automatic generation

Build visual maps from:

- meetings
- messages
- commitments
- decisions
- onboarding materials
- follow-ups
- concerns
- Projects
- related creations

## Simplicity

Default to:

- current relationship
- active issue
- immediate commitments
- next checkpoint

Do not display an entire history by default.

## User control

The user may:

- edit labels
- remove connections
- hide history
- correct meaning
- delete the visual
- preserve the underlying source record
