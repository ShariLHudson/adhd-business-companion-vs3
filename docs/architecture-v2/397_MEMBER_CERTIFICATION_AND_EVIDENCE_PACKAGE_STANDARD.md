# 397 — Member Certification & Evidence Package Standard

## Required evidence

A member cannot be marked complete without:

- canonical file inventory
- ownership map
- implemented routes
- implemented schemas
- retrieval evidence
- conversation transcripts
- Create output tests
- visual behavior tests
- overlap tests
- failure tests
- ADHD usability evidence
- source files changed
- known limitations
- maintenance owner
- certification result

## Certification outcomes

- fail
- partial
- conditional pass
- pass
- exceptional pass

## Exceptional pass requirements

- no unresolved ownership conflicts
- no critical duplicate systems
- no misleading creation promises
- no broken handoffs
- strong expert reasoning
- accessible and low-friction behavior
- exact resume works
- maintenance path exists
