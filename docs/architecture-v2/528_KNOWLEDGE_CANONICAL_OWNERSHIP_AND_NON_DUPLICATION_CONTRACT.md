# 528 — Knowledge Canonical Ownership & Non-Duplication Contract

## Knowledge owns

- canonical organizational knowledge
- knowledge structure
- knowledge taxonomy
- knowledge validation state
- source relationships
- knowledge lifecycle
- versioning
- contradiction handling
- staleness detection
- retrieval rules
- contextual application
- reusable knowledge assets
- knowledge provenance
- knowledge maintenance
- knowledge ownership
- knowledge confidence

## Research owns

- external investigation
- source discovery
- evidence gathering
- research methodology
- source credibility assessment

Research finds and evaluates evidence.
Knowledge decides how validated evidence becomes reusable organizational knowledge.

## Content & Communication owns

- explanation
- writing
- editing
- narrative
- communication format
- audience adaptation

Knowledge owns truth and structure.
Content owns expression and presentation.

## Learning owns

- teaching
- learning paths
- skill development
- practice
- assessment
- knowledge transfer

Knowledge provides authoritative material.
Learning turns it into learning experiences.

## AI / Intelligence owns

- cross-domain reasoning
- synthesis
- AI workflow design
- model and agent behavior

Knowledge provides trusted context.
AI / Intelligence reasons with it.

## Create owns

- shared creation runtime
- drafting
- saving
- editing
- resuming
- adaptation
- export

Knowledge supplies knowledge-specific logic and outputs.

## File storage owns

- storage
- permissions
- folders
- file delivery

Knowledge is not merely a folder system.

## Non-duplication rule

Knowledge must not become:

- a document warehouse
- a generic search bar
- a duplicate Research engine
- a second Content member
- a Learning platform
- a file-management system
- an ungoverned memory layer
