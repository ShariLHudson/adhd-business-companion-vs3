# 474 — Dashboard, Visualization & Automatic Visual Intelligence Standard

## Purpose

Use visualizations to improve understanding, not decorate data.

## Visual selection

- trend over time → line chart
- category comparison → bar chart
- composition → stacked view when appropriate
- distribution → histogram or range view
- relationship → scatter plot
- process or lineage → flow map
- geographic pattern → map when location is meaningful
- single decision status → scorecard or indicator

## Avoid by default

- 3D charts
- overloaded dashboards
- decorative gauges
- excessive colors
- truncated axes without clear reason
- unlabeled percentages
- charts without comparison
- visuals that hide missing data

## Dashboard structure

Show:

- decision
- key outcome
- drivers
- guardrails
- notable change
- uncertainty
- next review

## Automatic visual views

Data & Analytics may automatically generate:

- KPI tree
- metric relationship map
- trend view
- segment comparison
- funnel
- cohort view
- experiment result
- forecast range
- anomaly map
- data lineage view

## Default display rule

Show the smallest view needed for the current decision.
