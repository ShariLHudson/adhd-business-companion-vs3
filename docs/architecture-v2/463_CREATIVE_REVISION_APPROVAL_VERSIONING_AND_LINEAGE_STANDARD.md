# 463 — Creative Revision, Approval, Versioning & Lineage Standard

## Purpose

Preserve creative decisions, user edits, and asset relationships.

## Asset lineage

Each asset should reference:

- source brief
- source content
- brand source
- parent concept
- adaptations
- campaign, event, project, or client context
- version
- approval state
- final export

## Revision states

- concept
- first draft
- in review
- changes requested
- approved
- production-ready
- exported
- archived
- superseded

## Revision behavior

- do not restart intake unnecessarily
- retain accepted elements
- identify what changed
- distinguish content edits from visual edits
- preserve prior versions
- provide rollback
- do not overwrite user-edited variants silently

## Approval behavior

Record:

- approver
- approval scope
- date
- conditions
- unresolved issues

## Family consistency

When the master concept changes, identify affected adaptations and request review rather than silently replacing them.
