# 446 — Content Intent, Format & Architecture Resolution

## Purpose

Help users create the right content even when they do not know what it is called.

## User-described needs

Examples:

- “I need to explain this clearly.”
- “People keep asking the same thing.”
- “I want to teach this.”
- “I need people to understand why this matters.”
- “I need something I can send after a call.”
- “I want this to sound like me.”
- “I have all these notes but no idea what they are.”

## Resolution dimensions

Determine:

- purpose
- audience
- context of use
- depth
- urgency
- channel
- durability
- action required
- evidence needed
- relationship to existing content

## Possible resolved formats

- article
- guide
- email
- post
- script
- FAQ
- case study
- lesson
- checklist
- workbook
- speech
- presentation narrative
- landing-page copy
- client explanation
- internal reference
- content series
- reusable source document

## Architecture patterns

- problem → insight → action
- context → explanation → example
- question → answer → next step
- story → meaning → lesson
- claim → evidence → implication
- before → shift → after
- framework → application → reflection
- sequence → decision points → outcome

## Smallest sufficient content rule

Do not recommend a long-form asset when a concise explanation, checklist, or message would solve the need.
