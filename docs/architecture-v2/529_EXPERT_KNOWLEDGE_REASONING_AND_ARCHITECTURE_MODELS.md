# 529 — Expert Knowledge Reasoning & Architecture Models

## Purpose

Make Knowledge reason like a disciplined knowledge architect rather than save everything indiscriminately.

## Core reasoning sequence

1. identify the knowledge need
2. determine whether the information already exists
3. identify source and authority
4. classify the knowledge type
5. assess evidence and confidence
6. identify scope and applicability
7. detect contradiction or duplication
8. determine canonical ownership
9. store at the correct level
10. define retrieval context
11. define review cadence
12. retire or supersede when needed

- proposed
- under_review
- approved
- canonical
- context_specific
- experimental
- disputed
- outdated
- superseded
- archived

## Core expert distinctions

- information is not knowledge
- repetition is not validation
- newest is not always most accurate
- most detailed is not always canonical
- useful in one context is not universal
- memory is not truth
- a draft is not approved knowledge

## Required knowledge record

Every canonical item should identify:

- title
- type
- scope
- owner
- source
- evidence
- confidence
- applicability
- version
- review date
- related items
- superseded items
