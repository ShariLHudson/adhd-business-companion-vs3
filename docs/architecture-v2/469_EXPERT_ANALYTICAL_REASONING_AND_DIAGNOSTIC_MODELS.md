# 469 — Expert Analytical Reasoning & Diagnostic Models

## Purpose

Make Data & Analytics reason like a disciplined analyst rather than produce confident-sounding summaries.

## Core reasoning sequence

1. clarify the decision
2. identify the relevant population
3. define the time period
4. define the metric
5. inspect data quality
6. establish comparison
7. identify patterns
8. test alternative explanations
9. quantify uncertainty
10. state what the evidence supports
11. state what it does not support
12. recommend the next measurement or decision

## Analytical question classes

- descriptive: what happened?
- diagnostic: why might it have happened?
- comparative: how does it differ?
- predictive: what may happen next?
- prescriptive: what action appears supported?
- causal: did one thing cause another?
- exploratory: what patterns deserve investigation?

## Common analytical failures

- unclear denominator
- cherry-picked time range
- survivorship bias
- selection bias
- small sample overconfidence
- confusing correlation with causation
- averages hiding segments
- vanity metrics
- metric definition drift
- missing-data distortion
- seasonality ignored
- lagging and leading indicators mixed
- dashboard abundance without decisions

## Expert rule

Every insight should be traceable to:

- source
- definition
- calculation
- comparison
- limitation
- decision relevance
