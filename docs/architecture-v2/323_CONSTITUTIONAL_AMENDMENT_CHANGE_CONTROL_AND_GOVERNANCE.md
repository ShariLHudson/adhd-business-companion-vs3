# 323 — Constitutional Amendment, Change Control & Governance

## Purpose

Define how the Constitution may evolve without creating architectural drift.

## Amendment requirement

A constitutional amendment is required when changing:

- authoritative ownership
- Work identity
- Work Type registration
- Create ownership
- Project ownership
- Task ownership
- relationship ownership
- context ownership
- routing ownership
- certification ownership
- data-isolation principles
- document precedence
- user-agency principles

## Amendment proposal

Every proposal must include:

- problem
- current article
- proposed change
- reason
- alternatives considered
- affected systems
- migration plan
- backward compatibility
- user impact
- risks
- tests
- rollback
- approval

## Amendment status

Use:

- proposed
- under review
- approved
- rejected
- superseded
- implemented
- certified

## Constitutional interpretation

When language is ambiguous:

1. preserve one owner
2. preserve user data
3. preserve user agency
4. preserve context
5. avoid duplication
6. prefer relationships
7. prefer reversible change
8. disclose limits

## Governance records

Maintain:

- amendment registry
- architecture decision records
- constitutional exceptions
- temporary waivers
- expiry dates
- migration status
- certification status

## Temporary exception

A temporary exception must include:

- reason
- scope
- owner
- expiry
- containment
- migration plan
- risk
- tests

Temporary exceptions must not silently become permanent architecture.
