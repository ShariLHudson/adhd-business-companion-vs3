# 519 — Concept Architecture, Selection & Value Proposition Intelligence

## Concept structure

Every innovation concept should define:

- user
- problem
- current alternative
- proposed solution
- value created
- delivery mechanism
- differentiator
- assumptions
- constraints
- evidence
- test
- success threshold
- next decision

## Concept-selection dimensions

- problem fit
- evidence
- feasibility
- desirability
- viability
- strategic fit
- adoption friction
- time to learn
- reversibility
- downside
- scalability
- defensibility

## Value proposition standard

A value proposition should explain:

- who it is for
- what meaningful problem it addresses
- what outcome it enables
- why the approach is different or better
- what evidence supports the claim

## Selection rule

Do not choose solely by score.

Use scoring to reveal tradeoffs and then document judgment.

## Concept maturity states

- idea
- framed opportunity
- concept
- testable concept
- prototype
- pilot
- validated
- scale candidate
- retired
