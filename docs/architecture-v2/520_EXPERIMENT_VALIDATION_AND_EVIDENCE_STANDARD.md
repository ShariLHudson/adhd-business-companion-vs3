# 520 — Experiment, Validation & Evidence Standard

## Purpose

Create the smallest credible test that reduces the most important uncertainty.

## Experiment design

Define:

- hypothesis
- riskiest assumption
- participant or environment
- method
- evidence
- success threshold
- failure threshold
- duration
- cost
- ethical considerations
- decision after test

## Validation categories

- problem validation
- demand validation
- usability validation
- feasibility validation
- pricing validation
- delivery validation
- retention validation
- adoption validation
- operational validation
- business-model validation

## Evidence strength

From weaker to stronger:

- opinion
- stated interest
- observed behavior
- time commitment
- data sharing
- trial use
- repeat use
- referral
- payment
- renewal
- sustained outcome

## Test-selection rule

Use the least expensive test capable of producing decision-relevant evidence.

## Invalid validation patterns

- asking friends if the idea is good
- counting compliments as demand
- treating clicks as purchase intent
- testing only with ideal supporters
- changing success criteria after results
- ignoring negative evidence
