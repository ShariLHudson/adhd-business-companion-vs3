# 178_CURSOR_CREATE_TAXONOMY_AUDIT_AND_WORKFLOW_REDESIGN.md

# Purpose

Perform a complete audit of the Spark Estate **Create** experience.

This is NOT a visual redesign.

This is an information architecture, workflow, and usability audit.

The objective is to make Create feel obvious, frictionless, and complete.

---

# Audit Every Create Option

Inventory every item currently available under Create.

For each item identify:

- Category
- Current name
- What problem the user is trying to solve
- Whether it duplicates another option
- Whether it belongs in a different category
- Whether it launches a working creation workflow
- Whether the workflow is complete
- Whether it is only a placeholder
- Whether it should be merged, renamed, moved, or removed

Return the complete inventory.

---

# Organize Around User Intent

Do NOT organize by document type alone.

Organize by what the user is trying to accomplish.

Candidate top-level categories to evaluate:

- Business
- Marketing Strategy
- Social Media
- Content
- Branding
- Sales
- Client Experience
- Finance
- Operations & Systems
- Team & HR
- Legal
- Events
- Training & Courses
- Websites
- AI & Prompts
- Communications
- Documents & Forms
- Presentations
- Personal Productivity

These are starting points—not mandatory. Recommend the best taxonomy after the audit.

---

# Duplication Audit

Identify:

- duplicate creators
- overlapping templates
- multiple names for the same outcome
- categories with too few meaningful items
- categories that are overloaded

Recommend consolidation.

Example:

Do NOT have:

- Proposal
- Client Proposal
- Sales Proposal

Instead:

Proposal

↓

"What kind of proposal?"

---

# Workflow Audit

Every visible Create option must immediately launch a guided workflow.

No dead cards.

No placeholders.

No informational-only tiles.

Every workflow should support:

- one question at a time
- pause/resume
- autosave
- review
- edit
- approve
- export/print where appropriate

List every option that fails this requirement.

---

# Category Evaluation

For each category answer:

- Does the title match how business owners naturally think?
- Would a first-time user know where to look?
- Is anything missing?
- Is anything in the wrong category?

---

# Deliverables

Return:

1. Current Create inventory
2. Duplicate analysis
3. Recommended taxonomy
4. Items to merge
5. Items to rename
6. Missing creation experiences
7. Workflow gaps
8. Prioritized implementation roadmap
9. Files that will need updating
10. Do-not-deploy recommendation until audit findings are reviewed.

Do not implement changes yet.
Stop after producing the audit and recommendations.
