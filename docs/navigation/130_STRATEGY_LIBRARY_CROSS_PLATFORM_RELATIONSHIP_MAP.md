# Strategy Library Relationship Map

## Strategy Library ↔ Chamber

Direction:

- Strategy Library asks for domain expertise.
- Chamber returns structured specialist guidance.
- Strategy Library stores the contribution.
- Shari explains it.

Use for:

- expertise
- domain-specific questions
- recommendations
- examples
- risks

Do not use for:

- owning strategy state
- duplicating strategy records
- competing visible conversations

---

## Strategy Library ↔ Board

Direction:

- Strategy Library submits a strategy for review.
- Board returns perspectives, risks, trade-offs, and challenges.
- Strategy Library stores review results.
- Shari explains the implications.

Use for:

- high-impact decisions
- major trade-offs
- risk
- values
- long-term consequences

Do not use for:

- every small strategy
- replacing Chamber expertise
- replacing the user's decision

---

## Strategy Library ↔ Visual Thinking

Direction:

- Strategy Library provides structured strategy data.
- Visual Thinking creates a useful representation.
- Strategy Library links the visual.
- Shari explains how to use it.

Use for:

- complexity
- sequencing
- relationships
- systems
- timelines

Do not use for:

- automatic first response
- simple strategies
- substituting visualization for reasoning

---

## Strategy Library ↔ Projects

Direction:

- Strategy Library sends approved execution steps.
- Project owns ongoing work.
- Project returns progress.
- Strategy Library updates status.

Use for:

- multi-step ongoing execution
- collaboration
- files and milestones

Do not use for:

- automatic project creation
- one small action

---

## Strategy Library ↔ Plan My Day

Direction:

- Strategy Library sends one or more selected next actions.
- Plan My Day fits them into today.
- Completion updates strategy progress.

Use for:

- immediate execution
- first steps
- realistic scheduling

---

## Strategy Library ↔ Reminders

Direction:

- Strategy Library sends a specific future action.
- Reminder triggers at the right time.
- Completion updates the strategy.

Use for:

- one-time follow-up
- deadlines
- future actions

---

## Strategy Library ↔ Rhythms

Direction:

- Strategy Library sends a recurring practice.
- Rhythm provides ongoing support.
- Rhythm history informs strategy adaptation.

Use for:

- weekly reviews
- recurring check-ins
- maintenance
- repeated behavior

---

## Strategy Library ↔ Shari

Shari is the orchestration and conversation layer.

She:

- recommends
- explains
- personalizes
- sequences
- summarizes contributions
- keeps one natural conversation

She does not duplicate the data ownership of each capability.
