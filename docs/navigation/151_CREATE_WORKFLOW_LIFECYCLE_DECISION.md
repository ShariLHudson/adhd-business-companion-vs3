# Create Workflow Lifecycle Decision

## Rule

A creation type may appear in the Create picker only when it has a complete working workflow.

## Required Lifecycle

Select Type
→ Guided Questions
→ Draft
→ Edit
→ Review
→ User Approval
→ Save
→ Print where appropriate
→ Reopen Later

## Clickability

Every visible creation option must be clickable and route to the correct workflow.

Inactive placeholders must not be shown.

## Guidance

Questions must appear one at a time and must be specific to the selected creation type.

## Editing

The user must be able to add, delete, edit, reorder, and revise without regenerating the entire creation.

## Approval

A generated draft is not final.

The user must explicitly approve it.

## Save and Resume

Saved creations must reopen at the exact stage and section where the user left them.

## Print

Printing is available only for appropriate creation types and only after review and approval.

## Status

- correction approved
- implementation complete (unit verified — entrance + active registry + routing)
- authenticated preview not verified
- production not approved
