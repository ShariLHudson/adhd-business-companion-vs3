# Settings Global Simplicity and Control Standard

## Standing Rule

Every Spark Estate Settings experience must minimize reading, visible choices, and decision fatigue.

## Contrast

- light surface = dark text
- dark surface = light readable text
- disabled does not mean unreadable
- selected state does not depend on color alone

## Correct Controls

- toggle for on/off
- dropdown for one choice from several
- slider for a range
- text field for user-entered values
- accordion for optional explanation

Do not use large card stacks for a single-choice preference.

## Default Structure

1. one-sentence explanation
2. setting control
3. short selected-state description
4. optional Learn More accordion

## Sound Settings

Each sound family uses one dropdown and shows only the selected sound description.

The user should never need to scan a wall of sound-option boxes.

## Save Behavior

After a change:

- show Settings saved
- remain on the page
- preserve navigation origin
- preserve drafts
- do not redirect automatically

## Quality Test

A first-time user should understand what the setting controls, its current value, and how to change it within about ten seconds.

## Status

This is a standing UX and architecture rule for all current and future Settings surfaces.
