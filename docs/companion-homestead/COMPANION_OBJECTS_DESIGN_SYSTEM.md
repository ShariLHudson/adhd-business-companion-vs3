# Companion Objects Design System
## The Visual Language Of Shari's Home

**Version:** 1.0  
**Status:** Canonical — governs all illustrated objects in the Homestead  
**Authority:** Subordinate to Manifesto → Visual Design Bible → this document  
**Code registry:** `lib/companionObjectsDesignSystem/`  
**Feature mapping:** `lib/companionUniverse/libraries/objectLibrary.ts` (workspace icons)

---

## Mission

The Companion Homestead no longer uses emojis as its primary visual language. It uses **Companion Objects** — beautifully illustrated household objects that live naturally throughout the home.

These are not icons. They are emotional symbols that belong inside Shari's Iowa home.

Over time, users should recognize these objects the same way they recognize the logo.

---

## The Recognition Test

Imagine one object on Pinterest. No logo. No text. No branding.

Someone immediately says:

> **"That's from Shari's Companion."**

That is the standard.

## Design Philosophy

Objects should feel:

| Quality | Meaning |
|---------|---------|
| **Handcrafted** | Made by human hands, not generated by a UI kit |
| **Timeless** | Could exist in this house for decades |
| **Warm** | Morning light, honey wood, cream linen |
| **Slightly imperfect** | Loved, not staged |
| **Cozy** | Invites you to stay |
| **High quality** | Pottery Barn × Magnolia × Hallmark honesty |
| **Inviting** | "Come in" not "click here" |
| **Collectible** | Worth noticing again on visit six |

### Reject

- Flat icons
- Emoji style
- Corporate UI symbols
- Glossy app icons
- Clip art
- Cartoon graphics
- Gaming aesthetics
- Plastic, chrome, neon, tech-looking surfaces

### Reference mood

Pottery Barn · Magnolia · Hallmark · Cozy farmhouse · Watercolor illustration · Hand-painted woodcut · Natural materials

## Illustration Style — Warm Homestead Realism

Canonical style name shared with `VISUAL_DESIGN_BIBLE.md` Part IV.

| Attribute | Standard |
|-----------|----------|
| **Line weight** | 1.25–1.75px equivalent at 256px — soft, never cartoon outline |
| **Perspective** | Slight three-quarter view; object grounded on implied surface |
| **Lighting** | Soft morning window light from upper left; warm ~4200K |
| **Shadows** | Soft contact shadow beneath; no harsh drop shadow |
| **Texture** | Wood grain, ceramic glaze, linen weave visible at hero size |
| **Materials** | Natural only — see Material Palette below |
| **Color palette** | Cream `#FAF7F2`, warm white `#F5F0E8`, oak honey, walnut, barn-red accent, sage, linen, aged brass, companion-teal whisper |
| **Background** | Transparent PNG/SVG for objects; mini-scenes use room gradient |
| **Depth** | Subtle edge highlight — form from light, not bevel |
| **Outline** | None — silhouette from value and material |

### Size matrix

| Token | Pixel | Use |
|-------|-------|-----|
| `xs` | 18px | Inline affordances |
| `sm` | 32px | Dense lists |
| `md` | 40px | Sidebar, toolbelt |
| `lg` | 52px | Featured cards |
| `card` | 76×54px | Navigation mini-scene |
| `hero` | 128px+ | Room headers |

## Material Palette

### Prefer

Wood · Linen · Leather · Ceramic · Stone · Cotton · Glass · Copper · Brass · Woven baskets · Handmade pottery · Wool

### Avoid

Plastic · Chrome · Neon · Tech-looking objects · Corporate icon fills

When a modern object must appear (laptop, calculator), render it in **warm homestead tones** — leather sleeve, matte ceramic body, wood accents — never glossy product photography.

## Animation Language

Movement is **life, not notification**.

### Allowed (subtle, irregular)

- Steam rising from mug or kettle
- Candle flicker (low amplitude)
- Aquarium bubbles (gentle)
- Curtain drift (barely perceptible)
- Plant leaves swaying
- Bird briefly landing, then still
- Clock pendulum (muted, optional)
- Page lift on open book

### Forbidden

- Bounce · Spin · Pulse for attention · Game particles · Celebration loops

### Reduced motion

`prefers-reduced-motion` → static beauty remains. Objects still feel alive through illustration quality.

Registry field: `animationCapability` + `animation` on each catalog entry.

## Designer Stories

**Not one the user reads. One the designers know.**

Every object carries a `designerStory` — internal backstory for illustrators, environment artists, and writers building rooms. These stories **never appear on screen**. They are not tooltips, lore cards, or discovery text.

They exist so every design decision feels **authentic instead of decorative**.

### Why this matters

| Object | Without a story | With a story |
|--------|-----------------|--------------|
| Mug | Generic coffee icon | Shari's favorite mug — chip on the handle, reached for every morning |
| Journal | Stationery prop | The journal by her chair for ideas before they disappear |
| Crochet basket | Craft décor | The project she picks up when her mind needs to settle |
| Bird feeder | Landscaping | She genuinely loves watching hummingbirds, cardinals, and goldfinches |
| Aquarium | Visual effect | Watching tropical fish helps slow racing thoughts |

### Rules

1. **Never guest-facing** — no UI, no chat, no onboarding copy may quote `designerStory`
2. **Specific beats generic** — "Shari's favorite mug" not "a ceramic beverage container"
3. **Lived-in evidence** — worn spine, chip on handle, half-finished yarn, mud on gloves
4. **Honest imperfection** — objects show use, pause, and interruption
5. **One object, one truth** — every catalog id has a story in `catalog/designerStories/`

### Code

```ts
import { designerStoryForObjectId } from "@/lib/companionObjectsDesignSystem";

// Illustrators / art briefs only — never render in UI
const story = designerStoryForObjectId("obj-shari-signature-mug");
```

**Constitutional gate:** If an object cannot be given a believable designer story, it does not belong in the Homestead.

## Seasonal Variations

Objects evolve naturally. Nothing feels decorative just because it's a holiday.

Examples:

- Winter mug with steam · Summer iced tea glass
- Spring wildflowers · Autumn wheat wreath
- Holiday candles (gathered light, not garish)
- Fresh herbs on sill · Pumpkins on porch
- Bird feeder refilled for winter

Rules:

- Max **one variant active** per object per scene
- Variants are **subtle context shifts**, not different art styles
- Registry: `seasonalVariants[]` on each entry

## Master Catalog

**Location:** `lib/companionObjectsDesignSystem/catalog/`  
**Count:** 298 objects (target 200–300)  
**API:** `companionObjectCatalogById()`, `companionObjectsByCollection()`, `companionObjectCatalogSummary()`

Each entry includes:

| Field | Purpose |
|-------|---------|
| `id` | Stable registry key (`obj-*`) |
| `name` | Human-facing object name |
| `emotionalPurpose` | Quiet meaning |
| `collection` | Collection membership |
| `primaryRoom` | Homestead place id |
| `seasonalVariants` | Allowed seasonal shifts |
| `animationCapability` | Whether motion is permitted |
| `animation` | Motion type if applicable |
| `material` | Primary material |
| `colorPalette` | Homestead palette tokens |
| `illustrationNotes` | Art direction for illustrators |
| `accessibilityDescription` | Alt text for screen readers |
| `designerStory` | **Designer-only** backstory — never shown to guests |
| `futureExpansion` | Optional 3D, hero scene, etc. |
| `featureObjectId` | Link to workspace feature registry when applicable |

## Production Quality Bar

An object ships when:

1. Readable at `sm` (32px)
2. Recognizable without label
3. Uses homestead materials palette
4. Passes Emotional Design Rules (Visual Design Bible Part VII)
5. Has `accessibilityDescription` written
6. Passes the Pinterest Recognition Test
