# 367 — Visual Connection Intelligence Master Standard

## Purpose

Provide clear, automatically generated visual understanding of how work across Spark Estate connects.

## Governing principle

> Every visual must answer a useful question. No visual should exist merely because the platform has data.

## Automatic generation rule

When a creation contains meaningful relationships, the platform should automatically generate the appropriate visual view in the background.

The user should not be required to:

- add nodes
- draw lines
- choose a graph type
- understand mapping terminology
- manually connect known related items
- design the visual layout

## Questions the visual layer may answer

- What am I creating?
- How do the pieces fit together?
- What came from this idea?
- What is still missing?
- What depends on what?
- Where did this creation come from?
- What should happen next?
- Who or what is involved?
- What changed?
- Where am I stuck?
- What can I ignore for now?
- What belongs together?

## Supported automatically generated views

### Creation Structure Map

Shows sections, components, source materials, decisions, and outputs within the current creation.

### Connection Map

Shows relationships among ideas, creations, Projects, goals, people, and decisions.

### Creation Family

Shows one source artifact and its companion or adapted artifacts.

### Project Flow

Shows purpose, milestones, active work, dependencies, and linked creations.

### Idea Evolution

Shows how a thought moved into a decision, creation, or Project.

### Decision Map

Shows question, options, considerations, decision, and resulting work.

### Timeline

Shows sequence, dates, versions, milestones, and future review points.

### Cluster View

Groups related thoughts, artifacts, or work.

### Missing-Piece View

Highlights gaps in a creation set, Blueprint, Project, or plan.

### Focus Branch

Shows only the selected branch and hides everything else.

## Default restraint

The platform must not open with the full network.

Default to:

- current item
- immediate parent
- immediate children
- one next relationship level

Let the user expand deliberately.
