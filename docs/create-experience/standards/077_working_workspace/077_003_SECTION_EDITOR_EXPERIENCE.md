# 077 — Section Editor Experience

## Required Header

Every section editor must display:

- Work title
- Section title
- Current status
- Save state
- Return to Full Workshop Map
- Previous section
- Next section

## Required Editing Behaviors

The member must be able to:

- type directly;
- select and replace text;
- add, remove, and reorder items;
- edit generated content;
- undo;
- redo;
- rename section where allowed;
- add notes;
- attach references;
- add tasks;
- insert research;
- insert ideas;
- save;
- leave and return.

## Editing Modes

- Write
- Review
- Outline
- Checklist
- Table
- Notes
- Preview

The correct modes depend on the section type.

## Empty Section Behavior

An empty section must never present only a blank page.

It should offer:

- Start with a simple draft
- Give me ideas
- Show me examples
- Help me think
- I’m not sure
- Research this
- Start from scratch

## Generated Draft Behavior

Generated content must be:

- inserted into the editor;
- fully editable;
- clearly distinguishable as a draft;
- saved only to the correct Work ID and Section ID;
- undoable;
- versioned.

## Save State

Show one of:

- Saving…
- Saved
- Not saved
- Save failed
- Offline copy protected
- Conflict needs review

Never show “Saved” before durable persistence succeeds.
