# 077 — Workshop Map and Projects Integration

## Distinction

The Workshop Map owns the structure and content of the work.

Projects owns execution.

## Turn Into Project

The member may convert:

- the entire work;
- one section;
- selected action items;
- unresolved items;
- milestones.

## Linkage

Each Project item must preserve:

- source Work ID;
- source Section ID;
- source version;
- source title;
- source context.

## Editing Boundaries

Content edits should return to the source workspace.

Execution updates belong in Projects.

The member should not have to guess where to edit.

## From Projects

Opening a linked source item from Projects must return to the exact relevant section in the Workshop Map.

## Completion Synchronization

Project completion does not automatically make source content complete unless explicitly configured and confirmed.
