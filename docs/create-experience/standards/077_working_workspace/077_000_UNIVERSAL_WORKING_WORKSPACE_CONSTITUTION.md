# 077 — Universal Working Workspace Constitution

## Purpose

This standard defines exactly how a member opens, starts, edits, saves, revisits, completes, reopens, and continues every part of a created item in Spark Estate™.

It applies to:

- Create
- Projects
- Creative Studio
- Events
- Marketing
- SOPs
- Plans
- Strategies
- Campaigns
- Frameworks
- Checklists
- Courses
- Presentations
- Reports
- Founder Studio
- Business Estate
- Chamber-created work
- Board-supported work
- all future workspaces

## Constitutional Promise

Every visible section, stage, card, status, step, or map item that represents work must be actionable.

A member must always be able to:

- click it;
- open it;
- start it;
- edit it;
- save it;
- leave it;
- return to it;
- revise it after completion;
- ask for ideas;
- say “I’m not sure”;
- ask Shari;
- ask a Chamber Member;
- research it;
- mark it complete;
- reopen it.

No status label may behave like a locked display unless the platform explicitly explains why.

## Core Rule

> Complete means complete for now. It never means permanently locked.

## One Workspace Experience

Every work item uses the same underlying interaction model:

1. Open work.
2. See the full working map.
3. Click any section.
4. Start, continue, review, or edit.
5. Save automatically and manually.
6. Use contextual support.
7. return to the map.
8. complete or reopen sections at any time.

## Non-Negotiable Behaviors

- Every map item is clickable.
- Every section has an editable workspace.
- Every status is accurate.
- “Not Started” opens a starter state.
- “In Progress” opens the latest saved state.
- “Complete” opens in review/edit mode.
- “Needs Review” opens with review guidance.
- “Skipped for Now” remains reopenable.
- No section silently fails to open.
- No generated content is permanently read-only.
- Autosave never replaces visible save status.
- Leaving and returning restores the correct section and cursor location when possible.
