# 077 — Complete, Reopen, and Revision

## Complete for Now

The member-facing label should be “Complete for Now” where space permits.

Completion is a milestone, not a lock.

## Opening Completed Work

A completed section must open normally.

The member can:

- read;
- edit;
- review history;
- compare versions;
- ask for ideas;
- research;
- request review;
- reopen;
- keep complete after minor edits.

## Meaningful Edit Detection

A meaningful edit may:

- add new content;
- remove important content;
- change a decision;
- change a date, budget, audience, or scope;
- alter a requirement.

Formatting-only changes should not automatically reopen the section.

## Revision Versions

When revising a completed section:

- preserve the previous completed version;
- create a new revision;
- show revision history;
- allow restore;
- preserve completion dates and notes.

## Whole Work Completion

Marking the full work complete must not lock its sections.

It should:

- create a whole-work checkpoint;
- show any incomplete or skipped sections;
- allow completion with acknowledged exceptions;
- preserve future editing;
- support reopening the whole work.
