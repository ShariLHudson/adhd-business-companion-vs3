# 377 — Visual Map Versioning, Deletion & Recovery Standard

## Purpose

Make visual maps safe to change without fear.

## View versioning

Track:

- original auto-generated view
- user-edited versions
- layout changes
- hidden items
- removed relationships
- alternate views
- current preferred view

## Reset behavior

Offer:

- reset layout
- restore hidden items
- restore suggested connections
- return to original auto-generated view

## Delete behavior

The user may delete:

- a single visual view
- an alternate saved view
- a visual-only relationship
- a canonical relationship, when authorized
- the underlying source artifact, only through a separate explicit action

## Recovery

Deleted visual views should be recoverable for an appropriate retention period when platform policy permits.

## No accidental source deletion

Deleting the map must never delete the creation, Project, Blueprint, thought, or file it represents.
