# 364 — Spark Estate Adaptive Creation & Visual Connection Intelligence Master Standard

## Architecture ID

`architecture.adaptive_creation_visual_connection`

## Mission

Make Spark Estate responsive to how the user is thinking in the moment while also making the platform's relationships understandable through automatically generated visual connection when useful.

## Core principle

> Spark Estate should not only help users create individual things. It should help them understand how their ideas, decisions, creations, people, Projects, Blueprints, goals, and next steps belong together.

## Automatic visual principle

> When a creation would benefit from visual understanding, Spark Estate should automatically build the most appropriate visual map from the work already being created. The user should never have to construct the map manually before receiving value.

The user may then:

- edit
- relabel
- reconnect
- disconnect
- reorder
- expand
- collapse
- hide
- restore
- delete visual relationships
- delete the visual view
- correct inaccurate relationships
- create alternate views

## Adaptive Creation Intelligence

Adaptive Creation Intelligence determines:

- whether the user is thinking aloud
- whether the user wants something created
- whether the user is uncertain
- whether the user is overwhelmed
- whether the user wants Shari to choose
- whether reflection or action is more helpful
- whether a first version would help more than another question
- whether the user is exploring, deciding, creating, revising, or finishing

## Visual Connection Intelligence

Visual Connection Intelligence determines:

- whether a visual should be built
- which visual type is most useful
- when it should be generated
- what relationships should appear
- how much detail to show initially
- what should remain hidden
- how changes to the work update the visual
- how the user can correct or customize it

## Experience rule

The visual layer is automatic when useful, optional in display, and always editable.

Spark Estate remains:

- companion first
- workspace second
- visual map when helpful

It must not become a permanent dense dashboard.

## Required outcomes

A user must be able to:

- create from vague or incomplete thoughts
- receive help matched to their current state
- receive a useful visual without building it
- understand why things are connected
- open any source item from the visual
- change or remove visual relationships
- hide complexity
- focus on one branch
- return to the whole
- see what is missing
- see what is blocked
- see what changed
- use the visual to choose the next helpful step
