# 378 — Adaptive Creation & Visual Connection Certification and Cursor Handoff

## Certification ID

`certification.adaptive_creation_visual_connection_12_10`

## Required certification gates

### Adaptive behavior

- thinking aloud is not mistaken for create authorization
- ready-to-create state produces progress
- overwhelm reduces options
- low-energy mode simplifies
- hesitation rescue works
- perfection loops are gently interrupted
- returning users receive one clear resume step

### Automatic visual generation

- useful maps are generated without user construction
- appropriate visual type is selected
- simple creations do not receive unnecessary visual interruption
- maps may be built quietly and displayed later
- underlying changes update the map
- user layout changes are preserved

### User control

- edit works
- relabel works
- connect and disconnect work
- hide and restore work
- delete-view works
- reset works
- alternate views work
- source deletion remains separate and explicit

### Relationship truth

- one canonical relationship model
- no visual-only duplicate truth
- inferred relationships are labeled
- user corrections persist appropriately
- source objects remain authoritative

### Visual simplicity

- initial views are focused
- large networks are summarized
- no more than seven initial nodes by default
- accessibility does not rely only on color
- visual silence is respected

### Cross-platform connection

- Create links to Projects without duplication
- Blueprints show stages and outputs
- idea lineage works
- decisions connect to resulting work
- visual nodes open source items

## Required Cursor deliverables

Cursor must provide:

- current graph and relationship audit
- canonical relationship ownership map
- auto-generation trigger implementation
- view-selection logic
- map update logic
- user edit and deletion flows
- recovery behavior
- exact visual resume
- accessibility tests
- performance tests
- source files changed
- migration notes
- certification results

## Final principle

> Spark Estate should do the connecting work for the user. The visual should arrive already useful, remain simple, and stay fully under the user’s control.
