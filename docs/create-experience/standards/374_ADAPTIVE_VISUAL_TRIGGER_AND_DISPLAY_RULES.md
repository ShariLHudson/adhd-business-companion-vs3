# 374 — Adaptive Visual Trigger & Display Rules

## Purpose

Determine when a visual should be built, shown, offered, or withheld.

## Automatic build triggers

Build a visual automatically when:

- the creation has three or more meaningful components
- several source materials contribute
- the creation includes stages or dependencies
- multiple companion artifacts are generated
- decisions lead to downstream work
- a creation connects to a Project or Blueprint
- the user has scattered thoughts being organized
- relationships will improve future resume or navigation

## Automatic display triggers

Show the visual automatically when:

- the user asks how things connect
- the user is confused about structure
- the user asks what is missing
- the user asks for the big picture
- the visual clarifies a complex creation better than text
- the user is returning to multi-part work

## Build but do not display

The platform may generate the visual quietly and keep it available when:

- the creation is simple
- the user is in low-energy mode
- the user asked for direct creation
- showing it would interrupt momentum
- the map is useful for later lineage but not now

## Display modes

- companion explanation first
- visual first
- focused visual
- full map

The visual should match the user’s current need, not the maximum available data.
