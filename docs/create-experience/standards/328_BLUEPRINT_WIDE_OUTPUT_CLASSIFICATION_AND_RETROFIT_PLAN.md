# 328 — Blueprint-Wide Output Classification & Retrofit Plan

## Purpose

Classify every promised output in every existing Blueprint and correct any wording, implementation, or save-path gap.

## Scope

Apply retroactively to all Blueprint bundles and future Blueprints.

## Step 1 — Extract promised outputs

Search every Blueprint file for phrases such as:

- create
- build
- generate
- produce
- prepare
- develop
- design
- write
- draft
- plan
- publish
- submit
- manage
- complete
- calculate
- evaluate
- review
- deliver

Create one output inventory.

## Step 2 — Normalize duplicate output names

Examples:

- marketing strategy
- marketing plan
- campaign strategy

Determine whether these are:

- one shared output type
- separate templates
- distinct artifacts

Use one stable output ID when the underlying artifact is the same.

## Step 3 — Assign creation level

Assign one of the four required levels.

Base the level on actual runtime capability, not desired future capability.

## Step 4 — Map artifact creation

For every output define:

- what artifact appears
- where it is saved
- how it is edited
- how it is resumed
- how it is reused
- whether it exports
- whether it links to Project
- whether it requires professional review
- what happens if required inputs are missing

## Step 5 — Correct Blueprint language

Replace overstated promises.

Examples:

Replace:

> “Create your official property valuation.”

With:

> “Create a property-pricing preparation brief using your verified market information.”

Replace:

> “Manage your entire transaction.”

With:

> “Create the communication, decision, preparation, and working materials that support your transaction.”

Replace:

> “Publish your campaign.”

With:

> “Create and organize your campaign content; publishing depends on your connected channels and authorization.”

## Step 6 — Identify implementation gaps

Classify each gap:

- output named but no template
- template exists but no save path
- save exists but no resume
- artifact exists but no edit
- export promised but unsupported
- external action implied as complete
- professional boundary missing
- required data unavailable
- capability unavailable
- duplicate output identity
- output has no accountable owner

## Step 7 — Remediate

Choose:

- implement full capability
- lower the creation level
- clarify wording
- add user-input requirement
- add external-completion step
- add professional-review requirement
- remove unsupported promise
- mark future
- block release

## Required retrofit outputs

Cursor must generate:

- Blueprint Output Inventory
- Output Normalization Map
- Creation-Level Classification Matrix
- Output Capability Manifest Registry
- Unsupported Promise Report
- Missing Artifact Implementation Report
- External Completion Matrix
- Professional Review Matrix
- Wording Correction Report
- Retrofit Completion Report
