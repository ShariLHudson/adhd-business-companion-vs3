# Migration & Adoption

Defines the controlled migration of Create, Projects, Chamber Members, Board Directors, Founder Studio, Business Estate, Research, Knowledge, and future consumers into the Universal Creation architecture.

Migration sequence:
1. inventory current behavior;
2. identify the canonical capability;
3. protect member data;
4. create adapters where necessary;
5. migrate active records;
6. certify browser behavior;
7. retire conflicting paths;
8. monitor adoption.

No legacy path may be removed until data continuity and resume behavior are verified.
