# Platform Monitoring & Incidents

Defines operational monitoring, alerting, incident severity, containment, recovery, communication, and post-incident review.

Monitor:
- routing failures;
- failed saves;
- duplicate Work IDs;
- stale-session capture;
- broken handoffs;
- permission failures;
- output failures;
- integration drift;
- dead controls;
- unexplained latency.

Incident communication must clearly state what failed, whether member work is safe, and what remediation is underway.
