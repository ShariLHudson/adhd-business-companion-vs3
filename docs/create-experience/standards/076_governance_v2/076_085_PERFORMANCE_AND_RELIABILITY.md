# Performance & Reliability

Defines performance budgets, availability expectations, latency targets, save reliability, recovery behavior, and graceful degradation.

The platform must prioritize:
- fast opening of active work;
- reliable autosave;
- visible save state;
- bounded AI wait states;
- resilient routing;
- recovery after interruption;
- protection from duplicate mutations;
- usable fallback behavior when connected services are unavailable.
