# 079 — Accessibility and Cognitive Usability

## Purpose

Define accessibility as both technical compliance and reduced cognitive friction.

## Visual Readability

Spark Estate workspace targets should follow established platform readability requirements, including large titles, readable body text, generous spacing, and clear action labels.

## Keyboard

All interactive elements must:

- receive focus;
- show visible focus;
- operate by keyboard;
- follow logical order;
- avoid traps.

## Screen Readers

Provide:

- semantic headings;
- labeled regions;
- descriptive controls;
- status announcements;
- save-state announcements;
- error summaries;
- meaningful link text.

## Cognitive Usability

The interface should:

- show one clear next step;
- avoid excessive simultaneous choices;
- preserve context during transitions;
- use consistent labels;
- explain unfamiliar actions;
- show examples;
- allow safe pause;
- avoid shame-based urgency;
- distinguish required from optional.

## Motion and Sound

- animation must not block work;
- reduced-motion preferences must be honored;
- celebration sounds remain optional;
- status changes must not rely on sound alone.
