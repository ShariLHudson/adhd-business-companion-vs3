# 079 — Universal Workspace Interface Constitution

## Purpose

This standard defines the reusable interface system that surrounds every Universal Working Workspace in Spark Estate™.

It governs:

- navigation;
- dialogs;
- drawers;
- side panels;
- inspectors;
- status displays;
- contextual assistance;
- command surfaces;
- return paths;
- responsive behavior;
- accessibility;
- interruption recovery.

It applies to Create, Projects, Creative Studio, Events, Marketing, SOPs, Plans, Strategies, Campaigns, Frameworks, Founder Studio, Business Estate, Chamber-created work, Board-supported work, and every future workspace.

## Constitutional Promise

The interface around the work must never become harder to understand than the work itself.

A member should always know:

- where they are;
- what they are working on;
- what is saved;
- what section is open;
- how to return to the Full Workshop Map;
- what the next useful action is;
- how to ask for help;
- how to leave safely;
- how to return later.

## Non-Negotiable Rules

1. One shared interface model is used across every workspace.
2. Navigation must preserve the current Work ID and Section ID.
3. No panel may cover essential content without a clear close action.
4. No modal may trap the member.
5. Every dialog must explain the consequence of the action.
6. The interface must work with keyboard, touch, mouse, and assistive technology.
7. Mobile and desktop experiences must preserve the same capabilities.
8. Completed work remains editable.
9. Hidden controls may not be required for core actions.
10. Every visible control must be functional and certifiable.
