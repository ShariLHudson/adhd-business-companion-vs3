# 079 — Responsive and Touch Standard

## Purpose

Ensure the complete workspace remains usable across desktop, tablet, and mobile.

## Desktop

May support:

- editor plus side panel;
- persistent Full Workshop Map navigation;
- resizable panels;
- keyboard shortcuts;
- wider tables and timelines.

## Tablet

Should support:

- editor-first layout;
- collapsible map;
- touch-sized controls;
- optional split view in landscape;
- drawer-based assistance.

## Mobile

Must preserve:

- open section;
- edit;
- save;
- status;
- contextual help;
- return to map;
- previous and next;
- complete and reopen;
- notes;
- history;
- Project linkage.

Mobile may simplify layout, but not remove essential capabilities.

## Touch Requirements

- minimum touch target appropriate for accessibility;
- no hover-only controls;
- swipe actions require visible alternatives;
- destructive gestures require confirmation;
- drag-and-drop must have non-drag alternatives.

## Small-Screen Map

The Full Workshop Map may use:

- status filters;
- collapsible groups;
- search;
- recommended-next card;
- clear progress summary.

It must not hide completed sections permanently.
