# 080 — Universal Work Type and Section Schema Constitution

## Purpose

This standard defines how every createable work type in Spark Estate™ receives:

- a canonical Work Type ID;
- a Full Workshop Map;
- actionable sections;
- section-specific prompts;
- section-specific contextual assistance;
- status rules;
- validation;
- outputs;
- Project conversion;
- completion and reopening behavior.

This standard ensures that an Event, SOP, marketing campaign, strategic plan, checklist, course, report, framework, presentation, or future work type is not merely a title with generic fields. Each work type must provide a meaningful, expert-designed working structure while inheriting the same Universal Working Workspace behavior from 077 and the same interface system from 079.

## Constitutional Rule

> Every work type must define what the member can work on, and every defined section must open, edit, save, complete, reopen, and receive contextual help.

## Required Work Type Definition

Every work type must include:

- Work Type ID;
- human-readable name;
- purpose;
- common use cases;
- default Full Workshop Map;
- required sections;
- optional sections;
- conditional sections;
- section order;
- dependency rules;
- contextual-help configuration;
- completion logic;
- output formats;
- Project-conversion behavior;
- browser certification scenarios.

## Prohibited Patterns

- generic work types with no meaningful structure;
- display-only section lists;
- sections without Section IDs;
- shared labels with different hidden meanings;
- completion rules that permanently lock content;
- one fixed map for all work types;
- templates that cannot be edited;
- work types that bypass shared save, version, and recovery behavior.
