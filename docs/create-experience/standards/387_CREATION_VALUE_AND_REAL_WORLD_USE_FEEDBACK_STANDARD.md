# 387 — Creation Value & Real-World Use Feedback Standard

## Purpose

Learn whether created assets actually helped.

## Value questions

When appropriate, ask later:

- Did you use it?
- Did it make the situation easier?
- Did it reduce repeated questions?
- Did it save time?
- Did it improve consistency?
- What still did not work?
- Should this be reused?

## Possible outcome states

- not_used
- used_once
- reused
- partially_helpful
- solved_need
- needs_revision
- replaced
- became_standard
- became_knowledge_candidate

## No productivity scoring

Do not assign moral value to the user based on whether an artifact was used.

The purpose is to improve the creation and future recommendations.

## Strategy learning

Use outcome evidence to improve:

- create versus reuse recommendations
- scope recommendations
- artifact structure
- adaptation suggestions
- reusable-asset detection

Do not create rigid user labels.
