# 370 — Visual Create, Project & Blueprint Connection Standard

## Purpose

Make the relationship between Create, Projects, and Blueprints clear without merging them.

## Create

Create remains the source of truth for content.

Visuals may show:

- source idea
- purpose
- internal structure
- related artifacts
- adaptations
- review status
- intended use
- linked Project

## Projects

Projects manage execution.

Visuals may show:

- milestones
- tasks
- dependencies
- dates
- blockers
- linked Create artifacts
- related decisions

Projects must not duplicate creation content.

## Blueprints

Blueprints guide broader outcomes.

Visuals may show:

- Blueprint stages
- current stage
- artifacts created
- decisions made
- missing outputs
- related Projects
- contributing capabilities

## Required visual distinction

The user should be able to tell:

- this is something I created
- this is work I am executing
- this is a guided path I am following

without learning technical architecture.
