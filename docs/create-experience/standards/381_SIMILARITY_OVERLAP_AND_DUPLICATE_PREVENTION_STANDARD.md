# 381 — Similarity, Overlap & Duplicate Prevention Standard

## Purpose

Prevent clutter while respecting meaningful differences.

## Similarity signals

Compare:

- intended outcome
- audience
- topic
- structure
- required behavior
- source context
- repeated language
- related Project
- related Blueprint
- real-world use
- creation level

## Similarity is not identity

Two artifacts may share content but serve different purposes.

Examples:

- full procedure versus quick checklist
- internal version versus client-facing version
- source guide versus printable sign
- policy versus friendly expectations
- proposal versus sales presentation

Do not merge artifacts solely because text overlaps.

## Duplicate-risk levels

- none
- low
- moderate
- high
- near_duplicate

## User-facing behavior

When duplication risk is high, present no more than three actions:

- update the existing item
- create a connected variation
- create something separate

Include one recommended choice.

## No silent merging

Never merge, overwrite, archive, or delete existing work without explicit user approval.

## Alias handling

Similar titles must resolve through the Output Registry and artifact metadata rather than creating duplicate output types.
