# 382 — Creation Scope & Asset Value Standard

## Purpose

Match creation effort to likely usefulness.

## Scope questions

Evaluate silently:

- How often will this be used?
- Who will use it?
- What problem will it solve?
- What risk will it reduce?
- Will it improve consistency?
- Will it save repeated explanation?
- Can it support other creations?
- Is the need temporary or recurring?
- Is a smaller artifact sufficient?

## Scope levels

### Quick utility

A small, immediate artifact.

### Reusable working asset

Likely to be used more than once.

### Connected asset set

Several reusable artifacts serving one outcome.

### Strategic business asset

Core knowledge, framework, method, or recurring operating material.

## Value signals

- repeated use
- time saved
- reduced mistakes
- clearer decisions
- improved experience
- consistent delivery
- transferable knowledge
- training usefulness
- revenue support
- risk reduction

## Anti-overbuilding rule

The system must not recommend a high-scope asset merely because it could be valuable.

The recommendation must match the user's actual need, energy, and expected use.
