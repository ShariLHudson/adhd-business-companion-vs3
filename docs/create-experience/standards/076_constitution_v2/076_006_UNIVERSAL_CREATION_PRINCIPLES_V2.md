# 076 — Universal Creation Principles, Version 2

## Principle 1 — The Member Owns the Work
Implementation: editable content, explicit acceptance, no silent overwrite.

## Principle 2 — One Work Item, One Identity
Implementation: immutable Work ID across all surfaces.

## Principle 3 — Present Intent Before Continuity
Implementation: route current meaning before sticky state.

## Principle 4 — Classification Is Authoritative
Implementation: downstream enrichment cannot reinterpret work type.

## Principle 5 — Value Before Discovery
Implementation: starter draft plus minimal questions.

## Principle 6 — Everything Applicable Is Editable
Implementation: generated content is not locked.

## Principle 7 — Save Must Be Durable
Implementation: database verification before “Saved.”

## Principle 8 — No Dead Controls
Implementation: complete, clarify, or explain.

## Principle 9 — “I Don’t Know” Adapts
Implementation: options, examples, recommendation, skip, research.

## Principle 10 — Ideas Are Contextual
Implementation: use Work ID, section, audience, goals, constraints.

## Principle 11 — Research Writes Back
Implementation: insert, cite, note, task, Knowledge.

## Principle 12 — Structure Is Adaptive
Implementation: checklist, retreat, SOP, strategy, and budget remain distinct.

## Principle 13 — Conversation and Workspace Agree
Implementation: no claimed mutation without persisted change.

## Principle 14 — Continuity Is Earned
Implementation: unrelated questions exit the work owner.

## Principle 15 — Progress Is Visible
Implementation: clear state, next step, save status.

## Principle 16 — Members May Work Nonlinearly
Implementation: reorder, skip, branch, return.

## Principle 17 — Recovery Is Built In
Implementation: undo, versions, restore, crash recovery.

## Principle 18 — One Capability, Many Consumers
Implementation: shared services and contracts.

## Principle 19 — Specialized Intelligence Enriches
Implementation: Marketing, Finance, Events, Leadership, and others add expertise without replacing architecture.

## Principle 20 — Projects Own Execution
Implementation: source Work ID remains authoritative for content.

## Principle 21 — Outputs Preserve Lineage
Implementation: exports and derivatives reference source.

## Principle 22 — Accessibility Is Structural
Implementation: keyboard, contrast, screen readers, readable typography, cognitive accessibility.

## Principle 23 — Privacy Is Contextual
Implementation: minimum necessary context and permissions.

## Principle 24 — Errors Preserve Trust
Implementation: explain what failed, what is safe, and next action.

## Principle 25 — Reversible by Default
Implementation: soft delete, branch, version restore.

## Principle 26 — No Completion Theater
Implementation: meaningful status, not arbitrary progress percentages.

## Principle 27 — Low-Energy Use Is First-Class
Implementation: smaller scope and fewer decisions.

## Principle 28 — Recommendations Reduce Burden
Implementation: do not present equal choices without guidance.

## Principle 29 — Browser Evidence Determines Completion
Implementation: real persistence, refresh, resume, actions.

## Principle 30 — Future Features Inherit by Default
Implementation: new creation surfaces register as consumers.

## Certification
Each principle must map to:
- implementation owner;
- automated test;
- browser scenario;
- telemetry signal;
- failure remediation.
