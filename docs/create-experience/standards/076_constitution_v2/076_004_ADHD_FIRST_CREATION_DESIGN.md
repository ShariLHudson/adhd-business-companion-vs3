# 076 — ADHD-First Creation Design

## 1. Purpose

Translate adult ADHD and executive-function realities into platform architecture.

## 2. Common Friction

- initiation difficulty;
- working-memory overload;
- time blindness;
- interruption loss;
- too many directions;
- perfectionism;
- ambiguity avoidance;
- inconsistent energy;
- novelty-driven switching;
- difficulty locating prior work;
- underestimated complexity;
- shame after unfinished work.

## 3. Architectural Responses

### Initiation

Provide:

- starter drafts;
- templates;
- one-click examples;
- recommended starting points;
- “make the first part for me.”

### Working Memory

Keep visible:

- current goal;
- current section;
- key decisions;
- unresolved questions;
- next step;
- save state.

### Decision Fatigue

Use:

- two to four options;
- recommendations;
- “best starting point” language;
- progressive disclosure;
- reversible defaults.

### Interruption Recovery

Persist:

- active section;
- last meaningful action;
- stopping point;
- next useful step;
- unresolved decisions.

### Perfectionism

Support:

- rough draft;
- “good enough for now”;
- versioning;
- private draft states;
- easy revision.

### Time Blindness

Show:

- estimated effort ranges when useful;
- small-session options;
- milestones;
- pause points;
- scheduled review.

## 4. Therapy-Dog Lens

The platform should be:

- present;
- nonjudgmental;
- steady;
- observant;
- reassuring;
- supportive during transitions;
- able to help the member regulate and reconnect.

This is expressed through architecture, not cutesy language.

## 5. Overwhelm Behavior

When overwhelm is detected:

- do not route automatically to a relaxation destination;
- determine whether the member needs task help;
- reduce visible scope;
- offer one small next step;
- preserve the rest;
- offer pause.

## 6. Low-Energy Mode

Low-energy mode may:

- hide secondary panels;
- produce a starter answer;
- reduce decisions;
- offer a five-minute task;
- allow voice input;
- save stopping notes.

## 7. No Shame

Avoid:

- “You still haven’t…”
- “You failed to complete…”
- “Overdue” without useful context;
- forced streaks;
- infantilizing praise.

## 8. ADHD Certification

Browser scenarios must prove:

- start with minimal input;
- “I don’t know” adaptation;
- interruption recovery;
- low-energy flow;
- no repeated questions;
- clear save state;
- safe switching;
- return summary.
