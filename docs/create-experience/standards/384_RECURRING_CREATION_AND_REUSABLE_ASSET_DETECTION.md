# 384 — Recurring Creation & Reusable Asset Detection

## Purpose

Recognize when repeated work should become a reusable asset.

## Detection signals

- the same explanation is created repeatedly
- similar emails are drafted often
- several checklists share a stable core
- the user repeatedly rebuilds the same preparation
- several client-specific artifacts differ only in details
- the same process appears across Projects
- repeated decisions use the same criteria

## Possible recommendations

- reusable template
- standard guide
- approved response
- checklist
- framework
- decision aid
- content source
- connected creation set
- reusable business knowledge candidate

## Timing restraint

Do not interrupt active creation with organizational suggestions unless the opportunity is highly valuable.

Prefer surfacing after the immediate need is solved.

## User control

Nothing becomes a reusable standard or business-wide asset without approval.
