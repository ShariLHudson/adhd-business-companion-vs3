# 369 — Visual Simplicity & Decision-Fatigue Guardrails

## Purpose

Prevent automatic visual connection from becoming another source of overwhelm.

## Default limits

- show no more than seven visible nodes initially
- emphasize one primary item
- show no more than three relationship categories at once
- group low-priority items
- hide archived and completed branches by default
- avoid crossing lines where possible
- use plain-language labels
- offer one suggested visual action

## Progressive expansion

Support:

- show more
- focus here
- collapse branch
- hide for now
- show only active
- show only what affects this
- return to whole view

## No forced interpretation

Do not rely only on color, shape, or line style.

Every relationship must have readable text available.

## Visual mode choices

Use:

- Show how this connects
- Show the steps
- Show what is missing
- Show where this came from
- Show only what matters now

## Visual silence

The map may be built in the background without being placed in front of the user.

If displaying it would not help, keep it available but hidden.
