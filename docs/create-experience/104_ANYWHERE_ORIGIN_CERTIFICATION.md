# 104 — Anywhere-Origin Work Certification Standard

## Purpose

Certify that meaningful work can begin, continue, connect, and be supported from anywhere in Spark Estate while preserving one canonical Work item, one source of truth, one history, and one coherent relationship graph.

This is a certification phase. Do not add Marketing Plan.

## Certification target

Run certification against Event as the reference Work Type and the five registered Event Spark Blueprints:

- Business Luncheon
- Online Workshop
- One-Day Workshop
- Three-Day Retreat
- Book Signing

Use the Universal Work Engine, Universal Blueprint Framework, Blueprint interface, durable Blueprint storage, and Anywhere-Origin routing already implemented.

Certify:

- Create

- Shari conversation
- Chamber

- Clear My Mind

- Welcome Home

## Certification levels

### Level 1 — Architecture Certified

Prove:

- one authoritative Universal Work Engine
- one authoritative Work identity owner
- one authoritative Work Type registry
- one authoritative Blueprint registry
- approved durable repository boundaries
- no origin-specific creation engine
- no shadow save path
- no legacy template fallthrough
- no Work Type direct database access
- no Chamber, Board, Cartography, Body Doubling, Strategy, or Research direct persistence path

### Level 2 — Automated Certified

All unit, integration, boundary, migration, and end-to-end tests pass.

### Level 3 — Experience Certified

Real browser verification confirms correct user-visible behavior.

### Level 4 — ADHD Experience Certified

The experience reduces friction, preserves progress, avoids overload, and supports low-energy continuation.

### Level 5 — Domain Certified

Event-specific guidance, sections, tasks, risks, and deliverables remain appropriate for each Blueprint.

### Level 6 — Production Certified

No release-blocking defect remains, observability is present, and rollback/recovery paths are verified.

## Core certification scenario

Run this scenario from every origin:

1. Begin meaningful Event work in the origin.
2. Confirm the origin passes context through the universal launch contract.
3. Confirm duplicate detection runs.
4. Continue an existing Event or deliberately create a new one.
5. Confirm one canonical Work ID.
6. Confirm the intended Work Type and Blueprint.
7. Select or retain a depth mode.
8. Answer a domain-appropriate question.
9. Save durably.
10. Refresh.
11. Resume at the exact section and question.
12. Change depth and confirm the same Work ID.
13. Add a research question inside a section.
14. Save findings.
15. Review findings.
16. Approve one finding.
17. Apply it and verify version/history recording.
18. Create or connect a Project.
19. Create a task and milestone.
20. Attach a Cartography relationship.
21. Ask a Chamber member for help.
22. Request a Board review.
23. Talk with Shari in talk-only mode and verify no Work mutation.
24. Use Shari work-on-this mode and approve one change.
25. Start or attach a Body Doubling session.
26. Leave the experience.
27. Reopen from another origin.
28. Confirm the same Work ID, Blueprint, depth, content, section, tasks, research, relationships, history, and current focus.
29. Confirm no duplicate Work, Project, task, research item, or Blueprint was created.
30. Confirm the item can be archived, restored, and reopened.

## Identity and duplication certification

Verify:

- all newly created master Work IDs use the canonical format
- legacy Event records are adopted without duplication
- cross-origin reopen resolves the same Work
- similar titles do not automatically merge unrelated Work
- identical intent does not automatically create a duplicate
- explicit new-copy intent creates a new related Work item
- source and child relationships are preserved

## Data integrity certification

Verify:

- autosave
- manual save
- refresh recovery
- crash/interruption recovery where supported
- schema version handling
- Blueprint version handling
- archive
- restore
- delete/Trash behavior
- history integrity
- provenance integrity
- relationship integrity
- no orphan tasks, milestones, research, or Cartography references

## Security and privacy certification

Verify:

- Personal Blueprints remain user-isolated
- Company Blueprints remain company-scoped
- confidential information is not silently reused
- Chamber and Board contexts do not expose unauthorized Work
- research sources and files respect access rules
- direct repository access is blocked outside approved owners

## ADHD experience certification

Observe and document:

- number of simultaneous primary choices
- clarity of the recommended next step
- save reassurance
- skip and return-later behavior
- "I don't know" behavior
- low-energy continuation
- error recovery
- no forced restart
- no loss when changing depth
- no confusing duplicate work
- question wording and explanation quality

Release-blocking ADHD failures include:

- losing progress
- forcing the user to restart
- showing an overwhelming catalogue without progressive disclosure
- unclear whether work saved
- duplicating work after changing origin
- trapping the user in a depth mode
- preventing return to skipped questions

## Performance and observability

Verify:

- launch and resume are acceptably responsive
- registry lookup does not scan uncontrolled legacy sources
- duplicate detection is bounded and observable
- errors include traceable context without leaking private data
- failed apply operations do not corrupt Work
- retry behavior is safe
- metrics/logging distinguish origin, Work Type, Blueprint, and outcome

## Allowed final verdicts

Use exactly one:

- WORK TYPE PRODUCTION CERTIFIED
- WORK TYPE CONDITIONALLY CERTIFIED
- WORK TYPE NOT CERTIFIED

Conditional certification must name every condition, owner, and required follow-up.

## Release blockers

The following automatically prevent production certification:

- duplicate master Work
- shadow workspace
- wrong Work reopened
- lost content
- Blueprint or depth change creates a new Work ID
- origin bypasses Universal Work Engine
- direct unauthorized durable access
- research silently overwrites Work
- Cartography duplicates Work content
- Chamber or Board contribution applies without approval
- Shari talk-only mode mutates Work
- refresh cannot restore current state
- Event regression failure
- unresolved security or privacy defect
- undocumented migration risk

## Completion rule

Do not begin Marketing Plan unless the final verdict is:

`WORK TYPE PRODUCTION CERTIFIED`

and there are no release blockers.

Do not commit or push until certification artifacts are complete and reviewed.
