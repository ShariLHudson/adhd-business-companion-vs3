# 266 — Multi-Location Business Blueprint

## Blueprint ID

`business.multi_location`

## Purpose

Help businesses with multiple offices, stores, studios, branches, clinics, service areas, or operating locations create a shared operating model while preserving local visibility, accountability, and adaptation.

## Best-fit users

- multi-location retail
- service businesses
- studios
- agencies
- clinics operating within scope
- restaurants
- education centers
- branches
- regional offices
- home-service companies with multiple territories
- owner-operators expanding beyond one location

## Primary outcomes

The user can:

- define the role of headquarters and locations
- standardize core operations
- preserve appropriate local flexibility
- create location scorecards
- plan openings
- coordinate staffing and inventory
- manage regional leadership
- share knowledge
- compare performance
- intervene when locations struggle
- decide when and where to expand

## Shared architecture requirements

The implementation must preserve:

- one canonical business network
- one canonical location record type
- shared standards with controlled local variations
- location-level and network-level Work
- roll-up reporting
- cross-location access controls
- clear decision rights
- historical location performance
- source and revision provenance
