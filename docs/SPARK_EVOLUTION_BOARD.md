# Spark Evolution Board™

**Observation Mode** · [SPARK_OBSERVATION_MODE.md](./SPARK_OBSERVATION_MODE.md) · [Learning Log](./CONVERSATION_LEARNING_LOG.md)

Patterns graduate here **only after evidence** — not because an idea sounds good.

**The Rule of Three™:** A row moves toward implementation only when the same weakness appears in **three or more unrelated conversations** (linked in Evidence).

## Column guide

| Column | Purpose |
|--------|---------|
| **New Observation** | Short name of the pattern (e.g. "Turn 1 templates on SOP request") |
| **Evidence** | Links to [CONV-…](./CONVERSATION_LEARNING_LOG.md) entries — must be unrelated conversations |
| **Occurrences** | Count · must be ≥ 3 before implementation |
| **Recommendation** | Minimal change — usually one prompt line or one wisdom-loop rule · not a feature |
| **Approved?** | Human yes/no — member experience authority |
| **Implemented?** | Change landed · link commit or file |
| **Validation complete?** | Re-run relevant CT tests · scorecard attached |
| **Legacy Rule OK?** | More Shari, not more AI — warmth/trust preserved? |

## Workflow

```
Learning Log (1–2 hits)     →  Status: Observed / Watching
Learning Log (3+ hits)      →  Add row here
Approved + implemented      →  Re-validate CT-11 (+ related tests)
Validation complete         →  Resolved · archive row or mark done
```

---

## Not allowed on this board

- New features
- New specifications
- UI redesigns
- "Wouldn't it be cool if…" without three conversation receipts
