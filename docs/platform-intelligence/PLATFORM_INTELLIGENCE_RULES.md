# Platform Intelligence Rules™

**Status:** Shared operating rules for major Spark Estate destinations.  
**Does not replace:** Platform Governance Constitution (`docs/architecture-v2/315_…`), Estate Adaptive Intelligence, Conversation Guardrails (Spec 106), Simplicity Constitution (128).

## Rules every major destination should follow

1. Ask only what is not already known.  
2. Reuse existing work — do not create duplicates.  
3. Preserve context across intentional moves.  
4. Explain recommendations in plain language.  
5. Recommend one primary next step (≤2 secondary).  
6. Limit visible choices; offer More Options when truncated.  
7. Ask before creating or updating work in another destination.  
8. Preserve source-of-truth ownership per domain.  
9. Return useful contributions to the originating work when appropriate.  
10. Keep the member’s central question visible during guided work.  
11. Support pause and return without punishment.  
12. Respect active Adaptive Companion presentation preferences.  
13. Allow direct user requests to override stored presentation for that response.  
14. Do not silently change permanent settings.  
15. Do not pretend an unfinished integration works.  
16. Do not hide important intelligence — use progressive disclosure.  
17. Do not route unnecessarily; prefer guidance over menus.  
18. Protect user agency.  
19. Make uncertainty visible.  
20. Preserve provenance of returned contributions.
