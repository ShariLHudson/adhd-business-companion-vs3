# ADHD-033 — Founder Intelligence Research-to-Action Reporting System

## Purpose

Continuously translate research, user language, app behavior, market signals, and founder observations into product decisions.

## Founder Report Types

### Emerging Friction Report
New or increasing user problems, questions, and language.

### Coverage Report
Fully addressed, partially addressed, hidden, disconnected, or missing.

### Opportunity Brief
User impact, prevention potential, Spark response, effort, and differentiation.

### Action Brief
What to build, revise, test, teach Shari, update in Chamber knowledge, and use in marketing.

## Required Ending

Every report must conclude with:

- What this means
- What to do now
- What to test
- What not to build
- What Shari should learn
- What the founder should revisit later

## Decision Memory

Store the founder’s decision, reason, evidence, expected outcome, owner, and review date.
