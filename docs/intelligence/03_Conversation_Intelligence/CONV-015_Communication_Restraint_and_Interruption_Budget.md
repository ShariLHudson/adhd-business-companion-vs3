# CONV-015 — Communication Restraint and Interruption Budget

## Purpose

Prevent helpful communication from becoming noise.

## Core Principle

Every interruption should earn its place.

## Communication Types

- Essential reminder
- Time-sensitive alert
- User-requested follow-up
- Contextual suggestion
- Relationship check-in
- Discovery Key™ invitation
- System message

## Priority Order

1. Safety or critical account issue
2. User-requested reminder
3. Time-sensitive calendar support
4. Important follow-up
5. Contextual suggestion
6. Relationship question
7. Discovery

## Default Restraint Rules

- Do not stack multiple nonessential prompts.
- Do not repeat declined suggestions.
- Do not interrupt focus sessions unless the user approved it.
- Do not use sound for every message.
- Do not show several badges or counters.
- Prefer combining related messages.

## Suggested Support Levels

### Essential Only

Only reminders and important alerts.

### Light Support

Essential items plus occasional high-value suggestions.

### Balanced

Reminders, discoveries, and contextual support.

### Active Companion

More frequent suggestions and check-ins, still within limits.

## Session Rule

By default, no more than one unsolicited nonessential suggestion should appear in a single short session.

## Success Criteria

The user experiences support without notification fatigue.
