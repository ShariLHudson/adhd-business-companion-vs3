# CONV-035 — Conversation Pattern Quality Standard

Every conversation pattern should answer:

- What is happening?
- What is the user really asking?
- What problem is underneath?
- What does Spark retrieve?
- What does Spark ignore?
- What does Shari say?
- What action occurs?
- What gets remembered?
- How does Spark reduce friction?
- How is trust strengthened?

Conversation patterns become the executable behavior of Spark Estate™.
