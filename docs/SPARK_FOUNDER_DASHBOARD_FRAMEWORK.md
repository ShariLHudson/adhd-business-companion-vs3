# Spark Founder Dashboard™ — Founder-Only Intelligence

| Field | Value |
|-------|-------|
| **Spec** | Founder tooling (post-architecture) |
| **Status** | **Documented** — not member-facing · not built in Observation Mode |
| **Visibility** | Founders / operators only — **never members** |
| **Purpose** | Guide Spark's evolution with evidence, not intuition |
| **Mode** | [Observation Mode](./SPARK_OBSERVATION_MODE.md) · [Conversation Coach](./SPARK_CONVERSATION_COACH.md) |

---

## Why this exists

Observation Mode produces conversation evidence. The Founder Dashboard aggregates it so you can answer:

> *What deserves attention next — based on real conversations, not speculation?*

This does **not** change Spark for members. It changes how **you** coach Spark's evolution.

---

## Daily questions the dashboard should answer

| # | Question |
|---|----------|
| 1 | What are the **top 10 conversation patterns** this week? |
| 2 | Where are members **getting stuck**? |
| 3 | Which Spark **workspaces are suggested** most often? |
| 4 | Which workspaces are **almost never** suggested? |
| 5 | What **hidden intents** appear most frequently? |
| 6 | What **desired futures** (hoped success) appear most frequently? |
| 7 | What questions does Shari answer **exceptionally well**? |
| 8 | Which conversations received the highest **"felt like Shari"** scores? |
| 9 | What are the top **recurring opportunities** on the [Evolution Board](./SPARK_EVOLUTION_BOARD.md)? |
| 10 | What patterns are **Watching** vs ready for **Implement** (Rule of Three)? |

## Principles

- **Intelligence Paradox** — dashboard is deep; members never see it
- **Not a member feature** — no dashboards in Spark Alpha member UI
- **Evidence before change** — dashboard informs 1% refinements, not reinventions
- **Legacy Rule** — metrics include warmth/trust, not only correctness

---

## Build gate

Implement only when:

1. Observation Mode has **meaningful log volume** (real conversations, not tests only)
2. [Rule of Three](./SPARK_OBSERVATION_MODE.md) workflow is in active use
3. Founder explicitly approves build — **after** apprenticeship has produced data

Until then: manual review of Learning Log + Evolution Board.
