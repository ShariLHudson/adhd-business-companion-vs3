# User Control and Consent

## Always available to the member

- Review current Adaptive Companion prefs  
- Change a preference  
- Turn an adaptation off  
- Use a different presentation for one situation (session or turn)  
- Return to platform defaults  
- Reject a suggested preference (future observation phase)  
- Ask for more / less detail in conversation  

## Consent rules

| Change type | Requires |
|-------------|----------|
| Session-only override | Implicit (this conversation / this visit) |
| Conversational override (“give me the full detail”) | Immediate for that response; not permanent |
| Permanent preference write | Explicit approval |
| Observation-based suggestion (future) | Yes / Not now / No / Use only here |

## Language

Support · preference · access · capacity · working style · choice.  
Never diagnosis, deficit, or patronizing copy.

## This phase

No automatic write of observed behavior into permanent prefs.  
Hooks may exist for a future approval prompt only.
