## Principle

Google Drive becomes the **master storage location**.

Founder Knowledge Vault **indexes** these documents.  
Google Drive **stores** them.  
NotebookLM **researches** across them.

## Rules

1. **One authoritative file** per document — no duplicates across folders.
2. Vault and Master Library entries should include `googleDrivePath` when indexed.
3. NotebookLM libraries mirror Drive collections — not separate storage.
4. Recovery documents live in `Founder Studio/Recovery/` for fast orientation.

---

## When adding new documents

1. Save to the correct Drive folder first.
2. Register path in Knowledge Vault and Master Library sample data.
3. Add row to `MASTER_LIBRARY_INDEX.md`.
4. Add to NotebookLM library if large-collection research applies.
