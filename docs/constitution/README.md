## Order of authority (from 113)

When documents conflict:

1. User trust and safety  
2. **This Product Constitution (113)**  
3. **Simplicity & Cognitive Load (128)** on mental-load, architecture exposure, and ADHD/simplicity certification gates  
4. Specialized Constitutions **114–117**  
5. **Design Bible (125)** — [`docs/design-bible/`](../design-bible/README.md)  
6. Certified Product Specifications (including **112**)  
7. Design System and UX Standards (**118–124**) — [`docs/design-system/`](../design-system/README.md)  
8. Implementation documentation  
9. Local feature convenience  

> **Numeral note:** Create Blueprint track `docs/create-experience/128_webinar_event/` is a different series. Constitution **128** (this index) governs Estate-wide simplicity and cognitive load.
