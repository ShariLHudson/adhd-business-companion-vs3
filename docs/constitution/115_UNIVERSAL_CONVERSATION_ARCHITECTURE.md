
# 115 — Universal Conversation Architecture

Purpose:
Every conversation follows one architecture regardless of room.

Conversation lifecycle:
Intent → Context → Clarify (only if needed) → Progress → Completion → Continuity

Rules:
- Preserve active Work.
- Preserve return location.
- Do not repeat known information.
- Navigation changes location without losing conversation.
- Related capabilities join the conversation rather than replacing it.
- End every interaction with either clear completion or the next meaningful step.
