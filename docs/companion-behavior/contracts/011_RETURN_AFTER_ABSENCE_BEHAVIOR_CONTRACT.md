# Return After Absence Behavior Contract

## User Expectation

Returning should not create shame or an overwhelming backlog.

## Required Behavior

- welcome the person naturally
- determine the absence window without dramatizing it
- identify what remains relevant
- separate active, outdated, and optional work
- recommend one re-entry point
- allow pause, remove, reschedule, or resume
- summarize missed changes only when useful
- restore context without forcing a full review

## Default No-Setup Behavior

Even without a completed profile, use:

- saved active work
- recent confirmed commitments
- current user message
- safe re-entry defaults

## Prohibited Behavior

- list every missed reminder
- imply failure
- require re-onboarding
- assume previous priorities are still current
- force the user to clean up before receiving help

## Definition of Success

The person can return and begin again with dignity, clarity, and minimal effort.
