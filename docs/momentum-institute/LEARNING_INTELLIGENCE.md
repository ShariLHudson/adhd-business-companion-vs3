## Purpose

The Momentum Institute™ should **not** simply recommend lessons.

**Learning Intelligence™** decides what kind of experience will help the member **most at this moment**:

| Question | Possible answer |
|----------|-----------------|
| Should we **teach**? | Knowledge Card™, Business Mastery Minute™, Deep Lesson™ |
| Should we **coach**? | Discuss with Shari™ · Coaching Conversation™ |
| Should we **practice**? | Simulation™ · Challenge™ · Thinking Gym™ |
| Should we **create**? | Business Lab™ · Creative Studio™ · Make It Mine™ |
| Should we **reflect**? | Reflection™ · Journal™ · Conservatory™ |
| Should we **pause**? | Peaceful Places™ · Music Room™ · stay in conversation |
| Should we **celebrate**? | Acknowledge progress · Evidence Vault™ (permission) |

The answer depends on **member context** — not catalog availability alone.

**Transformation Map™** answers *what journey*. **Learning Intelligence™** answers *what mode right now*.

---

## The Golden Rule

> **The goal is NOT to maximize learning. The goal is to maximize entrepreneurial progress.**

Sometimes the best recommendation is **not another lesson**.

| Situation | Wrong instinct | Right instinct |
|-----------|----------------|----------------|
| Three hours of marketing study today | Another marketing card | Apply in Creative Studio™ or Business Lab™ |
| Exhausted member | Apprenticeship enrollment | Rest, reflection, gentle conversation |
| Same lesson didn't land | Repeat the same card | Different analogy, simulation, or coaching |
| Stuck on implementation | More theory | Make It Mine™ or Try It This Week™ |
| Natural win just happened | Next topic immediately | Celebrate · optional Evidence Vault™ |

Success is measured by **entrepreneurial growth** — capability, confidence, implementation, better decisions — not cards opened or minutes in the Institute.

## Learning states

Learning Intelligence™ tracks and responds to **where the member is in the learning arc** — not just what room they are in.

| State | Member signal | Intelligence bias |
|-------|---------------|-------------------|
| **Discovering** | Curious · exploring · "what is…?" | Light BMM · one card · no commitment |
| **Learning** | Engaged · wants the idea | Knowledge Card™ · guided lesson |
| **Understanding** | "I think I get it" | Confirm · one clarifying question |
| **Practicing** | Ready to try · "how would I…?" | Simulation™ · Challenge™ · Thinking Gym™ |
| **Applying** | Building something real | Business Lab™ · Creative Studio™ · Make It Mine™ |
| **Reflecting** | Processing · journaling | Reflection™ · Journal™ · Conservatory™ |
| **Experimenting** | Testing in the wild | Try It This Week™ · small challenge |
| **Building** | Artifact in progress | Resume paused work · don't interrupt with new lessons |
| **Teaching others** | Sharing what they learned | Evidence Vault™ · Return & Share · Growth Profile™ |
| **Celebrating** | Win just happened | Celebrate member · optional capture |
| **Resting** | Low energy · overwhelm · exhaustion | **No curriculum** · restoration rooms |
| **Reviewing** | Returning after apply | "What worked?" · adjust · not new content |

States are **invisible to members**. Shari responds naturally — never labels the state.

**Alignment:** [Spark Learning Model™](./SPARK_TEACHING_STANDARDS.md#the-spark-learning-model) · Institute lifecycle (`discover` → `learn` → … → `growth_profile`).

## Decision factors

Before recommending any Institute experience, Learning Intelligence™ weighs:

| Factor | Source | Weight when high |
|--------|--------|------------------|
| **Current conversation** | Session memory · digest | Match tone and immediate need |
| **Current project** | Artifact state · paused work | Prefer apply/build over teach |
| **Current competency levels** | Growth Profile™ | Don't re-teach mastered basics |
| **Current confidence** | Emotional thread · expressed doubt | Coach before lecture |
| **Current momentum** | Journey engine · recent wins | Apply or celebrate — don't brake |
| **Current energy** | Mood/energy in journey engine | Rest if low; learn if focused |
| **Projects in progress** | Paused work · active artifact | Resume > new lesson |
| **Recent learning** | Journey `learning` · topic study | Spaced learning — block binge |
| **Time available** | Expressed or inferred | BMM if short; apprenticeship only if sustained interest |
| **Current obstacle** | Stated stuck point | Simulation, coaching, or different modality |
| **Estate rooms recently visited** | Journey room history | Avoid ping-pong; natural next room only |
| **Current business stage** | Brain · profile | Match depth to idea/launch/growth/scale |

**Precedence when factors conflict:**

1. **Safety & restoration** — exhaustion, overwhelm → rest, not curriculum  
2. **Implementation in flight** — building beats teaching  
3. **Spaced learning** — enough recent study → practice  
4. **Transformation match** — Transformation Map™ journey  
5. **Modality fit** — teach / coach / practice / create / reflect

## Recommendation modes

### Teach

**When:** New concept needed · member asks to learn · understanding gap with energy to learn.

**Experiences:** Knowledge Card™ · Business Mastery Minute™ · Deep Lesson™ · Apprenticeship™ (only when sustained arc is appropriate).

**Avoid when:** Recent binge on same topic · exhaustion · active build in progress.

---

### Coach

**When:** Confusion is personal · decision pending · emotion blocks action · "talk this through."

**Experiences:** Discuss with Shari™ · Coaching Conversation™ · one question at a time.

**Avoid when:** Member asked for a quick factual answer and confidence is high.

---

### Practice

**When:** Concept understood but not embodied · fear of real-world try · needs rehearsal.

**Experiences:** Simulation™ · Challenge™ · Thinking Gym™.

**Avoid when:** Member hasn't grasped the idea yet — teach or coach first.

---

### Create

**When:** Ready to produce · apply to business · artifact would help.

**Experiences:** Business Lab™ · Creative Studio™ · Make It Mine™ (permission).

**Avoid when:** Member is exploring — don't force a deliverable.

---

### Reflect

**When:** Processing · integration · after apply · journal-worthy moment.

**Experiences:** Reflection™ · Journal™ · Conservatory™ · quiet conversation.

**Avoid when:** Member wants action now — don't therapize.

---

### Pause

**When:** Cognitive load maxed · life happened · no shame return.

**Experiences:** Stay in conversation · Peaceful Places™ · Music Room™ · Coffee House™ · **no** new Institute enrollment.

**Avoid when:** Member is energized and asking to continue learning.

---

### Celebrate

**When:** Win · completion · brave step · returned after absence.

**Experiences:** Warm acknowledgment · optional Evidence Vault™ · Growth Profile™ quiet update.

**Avoid when:** Member is in crisis — celebrate only what is true.

## Worked examples

### Example 1 — Enough learning for now

**Member:** *"I've spent three hours learning marketing."*

**Learning Intelligence™**

- Recent learning: heavy marketing topic study → **block new marketing lessons**
- State: Learning → shift to **Applying**
- Recommendation mode: **Create** + practice invitation

**Spark:** *"You've put real time into this — I think the best move now is to put it into practice. Would you like to try something in Creative Studio™, or a short Business Lab™ to shape one piece for your business?"*

**Not:** Another marketing Knowledge Card™ or Apprenticeship™ pitch.

---

### Example 2 — Exhaustion

**Member:** *"I'm exhausted."*

**Learning Intelligence™**

- Energy: low · emotional thread: depleted
- State: **Resting**
- Recommendation mode: **Pause** — zero curriculum pressure

**Spark:** *"I'm glad you said that. Let's not pile on anything else right now. The Conservatory™ or Music Room™ might be peaceful if you want a change of air — or we can just sit here quietly. What would feel kindest?"*

**Not:** Apprenticeship™ · multi-step learning path · guilt for "not finishing."

---

### Example 3 — Still don't understand

**Member:** *"I still don't understand pricing."*

**Learning Intelligence™**

- Obstacle: same topic · prior lesson didn't land
- State: stuck between Learning and Understanding
- **Do not repeat** the same card
- Recommendation mode: rotate modality — **Coach** or **Practice** or different **Teach** angle

**Spark:** *"Let's try a different angle — not the same lesson again. Would you like to talk it through with me, walk a quick pricing simulation, or see the idea through a real example from your business?"*

**Options (max 3):** Coaching conversation · Simulation™ · Business Lab™ with their numbers.

---

### Example 4 — Building in progress

**Member returns · paused workshop in journey engine**

**Learning Intelligence™**

- Projects in progress: workshop paused
- State: **Building**
- Recommendation mode: **resume** — not new curriculum

**Spark:** *"Welcome back. Last time we were building your workshop. Would you like to continue where we left off?"*

---

### Example 5 — Celebration after apply

**Member:** *"I sent the email with my new price — and she said yes."*

**Learning Intelligence™**

- State: **Celebrating** → optional **Teaching others** / Evidence
- Block: immediate next lesson

**Spark:** *"That's huge — you did the hard part. How does it feel? If you'd like, we can capture this in your Evidence Vault™ for the day you forget how capable you are."*

## Spaced learning

Learning Intelligence™ enforces **spaced learning** — the antidote to binge culture.

```
Learn → Practice → Reflect → Apply → Return later
```

| Rule | Implementation |
|------|----------------|
| **No binge encouragement** | After N learning events same topic/session → switch to apply |
| **Return later is valid** | Cabinet save · pause · no reminders that shame |
| **Same topic cooldown** | Don't recommend identical card within cooldown window |
| **Session budget** | Soft cap on Institute learning per session — Shari names it warmly |
| **Topic study weeks** | Journey engine `topicStudy` — sustained study triggers apply invitation (see Estate Journey Intelligence) |

**Shari lines (permission to stop):**

- *"I think you've learned enough for today."*
- *"You've made wonderful progress."*
- *"I'd love to see you try this before we learn something new."*
- *"Let's pause the learning and put it into practice."*

## Shari's role

Shari **quietly guides** — never performs as a learning management system.

| Shari is | Shari is not |
|----------|--------------|
| Mentor who knows when to stop teaching | Catalog that always suggests more |
| Partner who notices exhaustion | Engagement optimizer |
| Coach who rotates modalities | Repeater of failed lessons |
| Celebrator of the member | Streak tracker |

Every recommendation passes the [Shari test](../RELATIONSHIP_CONSTITUTION.md) and [Spark Teaching Standards™](./SPARK_TEACHING_STANDARDS.md).

## Estate connections

Learning Intelligence™ moves members across the Estate as **one continuous journey** — not module hopping.

```
Momentum Institute™  — teach · discover · understand
        ↓
Creative Studio™     — create · apply · Make It Mine™
        ↓
Journal™             — reflect · integrate
        ↓
Coffee House™        — connection · informal thinking
        ↓
Evidence Vault™      — celebrate · proof of growth (permission)
        ↓
Growth Profile™      — competency deepens quietly
```

**Restoration loop (low energy):**

```
Conservatory™ · Peaceful Places™ · Music Room™
        ↓

        ↓

```

**Outputs:** Estate Intelligence™ room invitation · workspace offer · in-conversation next step — **one primary invitation**.

## Decision pipeline (internal)

```
Member turn
    ↓
Estate Journey Engine (context)
    ↓
Transformation Map™ (which journey — if any)
    ↓
Learning Intelligence™ (which mode + experience)
    ↓
Confidence & permission gates
    ↓
One invitation (teach · coach · practice · create · reflect · pause · celebrate)
```

### Mode selection matrix (simplified)

| Energy | Recent learning (same topic) | Active build | Likely mode |
|--------|------------------------------|--------------|-------------|
| Low | Any | Any | **Pause** / reflect |
| Medium | High | No | **Practice** / **Create** |
| Medium | Low | Yes | **Resume build** |
| High | Low | No | **Teach** / **Coach** |
| High | High | No | **Apply** — not teach |
| Any | Any | Stuck on concept | **Coach** or alternate **Teach** modality |

### Block rules (hard)

| Condition | Block |
|-----------|--------|
| `overwhelmed` or `exhausted` | Apprenticeships · new multi-step paths |
| 3+ learning events same topic today | Another lesson same topic |
| Paused artifact exists | New unrelated curriculum (offer resume first) |
| `prefers-reduced-motion` + restoration | Intense challenge or simulation without consent |
| Permission not granted | Labs · Make It Mine™ · Evidence Vault™ |

## Quality gates

Before any educational recommendation ships:

1. Does this maximize **progress**, not consumption?
2. Is this the right **mode** for the learning state?
3. Would **another lesson** be worse than apply/coach/rest?
4. Is spaced learning respected?
5. One primary invitation only?
6. Permission where required?
7. [Transformation Map™](./TRANSFORMATION_MAP.md) journey named when relevant?
8. Passes Shari test?

If any gate fails — change the recommendation, not the member.
